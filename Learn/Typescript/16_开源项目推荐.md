# 16 开源项目推荐

> **难度：了解即可** | 本章用于 TypeScript 源码精读。建议带着“它如何建模类型、组织模块、暴露 API”这些问题阅读。

---

## 1. TypeScript Compiler

**标注：核心高频**

- **它是什么**：TypeScript 官方编译器与语言服务
- **核心特性**：AST、类型检查、增量编译、语言服务
- **值得学什么**：大型编译器项目如何组织类型、节点和诊断信息
- **阅读切入点**：先看 Compiler API 示例，再看 `Program`、`SourceFile`、diagnostic 相关代码

**为什么这样设计、这样设计带来的好处：**

TypeScript 自身是理解类型系统边界的最好材料，尤其适合进阶学习类型检查器和语言工具。

---

## 2. VS Code

**标注：重要**

- **它是什么**：微软开源编辑器
- **核心特性**：插件系统、工作台架构、语言服务集成
- **值得学什么**：大型桌面应用的模块边界、事件系统和扩展 API
- **阅读切入点**：从 extension sample 和贡献点配置开始，再看工作台模块

**为什么这样设计、这样设计带来的好处：**

VS Code 展示了 TypeScript 在超大型应用中的工程化实践。

---

## 3. Deno

**标注：重要**

- **它是什么**：现代 JavaScript/TypeScript 运行时
- **核心特性**：权限模型、内置 TS 支持、标准库、工具链
- **值得学什么**：运行时 API 类型设计、安全边界、工具链集成
- **阅读切入点**：CLI 命令、权限系统、标准库类型定义

**为什么这样设计、这样设计带来的好处：**

Deno 把运行时、包管理、格式化、测试整合到一个工具链，适合学习工程取舍。

---

## 4. Vite

**标注：核心高频**

- **它是什么**：现代前端构建工具
- **核心特性**：开发服务器、插件系统、预构建、HMR
- **值得学什么**：插件接口、配置解析、开发与生产构建的差异
- **阅读切入点**：`createServer`、插件钩子、配置解析流程

**为什么这样设计、这样设计带来的好处：**

Vite 是学习 TypeScript 工具链项目的优秀入口，业务复杂度适中但工程价值高。

---

## 5. pnpm

**标注：重要**

- **它是什么**：高性能包管理器
- **核心特性**：内容寻址存储、workspace、依赖解析、锁文件
- **值得学什么**：复杂 CLI、monorepo、文件系统操作、错误提示
- **阅读切入点**：命令入口、workspace 处理、依赖解析模块

**为什么这样设计、这样设计带来的好处：**

pnpm 展示了如何用 TypeScript 构建严肃的开发者基础设施工具。

---

## 6. tRPC

**标注：核心高频**

- **它是什么**：端到端类型安全 RPC 框架
- **核心特性**：类型推导、路由器、过程定义、客户端类型自动生成
- **值得学什么**：高级泛型、条件类型、API 类型设计
- **阅读切入点**：router 定义、procedure builder、client 类型推导

**为什么这样设计、这样设计带来的好处：**

tRPC 把 TypeScript 类型推导用于真实框架，是学习高级类型工程价值的典型项目。

---

## 7. TanStack Query

**标注：重要**

- **它是什么**：前端异步状态管理库
- **核心特性**：缓存、请求状态、失效机制、框架适配
- **值得学什么**：泛型 API、状态机、跨框架核心包设计
- **阅读切入点**：core 包、query cache、observer 模型

**为什么这样设计、这样设计带来的好处：**

它把异步请求抽象为缓存和观察者模型，适合学习库设计与类型友好 API。

---

## 8. Zod

**标注：核心高频**

- **它是什么**：TypeScript-first schema 校验库
- **核心特性**：运行时校验、类型推导、组合式 schema
- **值得学什么**：如何让运行时对象和静态类型保持一致
- **阅读切入点**：基础 schema、`infer` 类型、错误格式化

**为什么这样设计、这样设计带来的好处：**

Zod 解决了 TypeScript 类型擦除后的运行时边界问题，是后端和前端都常用的项目。

---

## 9. NestJS

**标注：了解即可**

- **它是什么**：Node.js 企业级后端框架
- **核心特性**：模块、依赖注入、装饰器、管道、守卫
- **值得学什么**：框架分层、元数据反射、企业应用架构
- **阅读切入点**：模块注册、DI 容器、控制器调用链

**为什么这样设计、这样设计带来的好处：**

NestJS 适合学习 TypeScript 在大型后端项目中的架构化使用。

---

## 源码精读路线

| 阶段 | 目标 |
|------|------|
| 第一遍 | 跑 examples，理解项目解决什么问题 |
| 第二遍 | 找入口、配置解析和核心数据结构 |
| 第三遍 | 读类型定义，理解 API 如何约束用户 |
| 第四遍 | 看测试，理解维护者关心哪些边界 |
| 第五遍 | 尝试修文档或补一个小测试 |

---

## 动手练习

### 练习 1：精读一个工具链项目

选择 Vite 或 pnpm，画出 CLI 启动到核心执行的流程图。

**思路提示：** 先找 package.json 的 bin 字段，再追踪入口文件。

### 练习 2：分析一个泛型 API

选择 tRPC、TanStack Query 或 Zod，整理一个 API 的类型推导链路。

**思路提示：** 从用户调用处反向追踪类型定义。

### 练习 3：阅读错误模型

找一个项目的错误格式化逻辑，记录错误如何面向用户展示。

**思路提示：** 好的工具不仅要报错，还要告诉用户如何修复。

### 练习 4：提交一次低风险贡献

为示例、文档或测试补充一个小改动。

**思路提示：** 第一次贡献优先选文档和测试，不要直接改核心逻辑。
