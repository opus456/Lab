# TypeScript 工程化配置

## 1. tsconfig.json 完全指南

`tsconfig.json` 是 TypeScript 项目的核心配置文件，它告诉编译器如何处理你的代码。

### 1.1 compilerOptions 核心选项详解

#### target - 编译目标

`target` 决定 TypeScript 编译输出的 JavaScript 版本：

```json
{
  "compilerOptions": {
    "target": "ES2020"
  }
}
```

**各选项对比：**

| target | 适用场景 | 说明 |
|--------|---------|------|
| ES5 | 需要兼容 IE11 | 会编译箭头函数、class、async/await 等 |
| ES6/ES2015 | 现代浏览器 | 保留箭头函数、class，编译 async/await |
| ES2020 | Node.js 14+ / 现代浏览器 | 保留可选链、空值合并 |
| ES2022 | Node.js 18+ | 保留 top-level await、class fields |
| ESNext | 始终最新 | 几乎不做语法转换，依赖运行时支持 |

**选择建议：**
- 前端项目：通常选 `ES2020`（配合 Babel/esbuild 做进一步降级）
- Node.js 项目：根据最低支持的 Node 版本选择（Node 18 → ES2022）
- 库开发：选 `ES2020` 或更低，确保兼容性

```json
// 前端项目推荐
{ "compilerOptions": { "target": "ES2020" } }

// Node.js 18+ 项目推荐
{ "compilerOptions": { "target": "ES2022" } }
```

#### module - 模块系统

`module` 决定编译输出使用哪种模块系统：

```json
{
  "compilerOptions": {
    "module": "ESNext"
  }
}
```

**各选项对比：**

| module | 输出格式 | 适用场景 |
|--------|---------|---------|
| commonjs | `require()/module.exports` | Node.js 传统项目 |
| esnext | `import/export` | 前端项目（由打包工具处理） |
| node16/nodenext | 根据 package.json 的 type 字段决定 | Node.js 现代项目 |
**选择建议：**
- 前端项目（Vite/Webpack）：`ESNext`，让打包工具处理模块转换
- Node.js（使用 ESM）：`node16` 或 `nodenext`
- Node.js（使用 CJS）：`commonjs`
- 库开发：`ESNext`（配合打包工具输出多种格式）

```json
// 前端项目
{ "compilerOptions": { "module": "ESNext" } }

// Node.js ESM 项目
{ "compilerOptions": { "module": "node16" } }
```

#### lib - 类型声明库

`lib` 指定编译时可用的内置类型声明。如果不配置，TypeScript 会根据 `target` 自动选择。

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "lib": ["ES2020", "DOM", "DOM.Iterable"]
  }
}
```

**为什么需要手动配置 lib？**

1. **Node.js 项目不需要 DOM 类型**：默认会包含 DOM，导致 `document`、`window` 等全局变量可用，这在 Node.js 中是错误的
2. **需要更新的 API 类型**：比如 target 是 ES2020 但想用 ES2022 的 `Array.at()` 类型
3. **Web Worker 环境**：需要 `WebWorker` 而非 `DOM`

```json
// Node.js 项目 - 不要 DOM
{
  "compilerOptions": {
    "target": "ES2022",
    "lib": ["ES2022"]
  }
}

// 前端项目 - 需要 DOM
{
  "compilerOptions": {
    "target": "ES2020",
    "lib": ["ES2020", "DOM", "DOM.Iterable"]
  }
}
```

#### strict - 严格模式

`strict: true` 是一个总开关，启用所有严格类型检查选项：

```json
{
  "compilerOptions": {
    "strict": true
    // 等价于同时开启以下所有选项：
    // "noImplicitAny": true,
    // "strictNullChecks": true,
    // "strictFunctionTypes": true,
    // "strictBindCallApply": true,
    // "strictPropertyInitialization": true,
    // "noImplicitThis": true,
    // "alwaysStrict": true,
    // "useUnknownInCatchVariables": true
  }
}
```

**各子选项详解：**

| 选项 | 作用 | 示例 |
|------|------|------|
| `noImplicitAny` | 禁止隐式 any | `function fn(x) {}` 报错，必须标注类型 |
| `strictNullChecks` | null/undefined 不能赋给其他类型 | `let s: string = null` 报错 |
| `strictFunctionTypes` | 函数参数类型严格检查（逆变） | 防止不安全的函数赋值 |
| `strictBindCallApply` | 严格检查 bind/call/apply 参数 | 确保参数类型正确 |
| `strictPropertyInitialization` | class 属性必须初始化 | 未在构造函数中赋值的属性报错 |
| `noImplicitThis` | 禁止隐式 any 类型的 this | 必须显式标注 this 类型 |
| `useUnknownInCatchVariables` | catch 的 error 是 unknown 而非 any | 必须类型收窄后使用 |

**强烈建议：** 新项目始终开启 `strict: true`。对于迁移项目，可以逐步开启子选项。

#### moduleResolution - 模块解析策略

决定 TypeScript 如何查找 `import` 的模块：

```json
{
  "compilerOptions": {
    "moduleResolution": "bundler"
  }
}
```

| 策略 | 适用场景 | 说明 |
|------|---------|------|
| `node` | 旧版 Node.js（CJS） | 按 Node.js 的 require 规则解析 |
| `node16`/`nodenext` | 现代 Node.js（ESM） | 严格遵循 Node.js ESM 规则，需要文件扩展名 |
| `bundler` | 前端项目（Vite/Webpack） | 类似 node，但支持 package.json exports 且不要求扩展名 |
**选择建议：**
- 使用 Vite/Webpack 等打包工具：`bundler`
- Node.js ESM 项目：`node16` 或 `nodenext`
- 旧 Node.js CJS 项目：`node`

#### esModuleInterop - ES 模块互操作

```json
{
  "compilerOptions": {
    "esModuleInterop": true
  }
}
```

**为什么需要这个？**

很多 npm 包使用 CommonJS 导出（`module.exports = ...`），但我们想用 ES Module 语法导入：

```typescript
// 没有 esModuleInterop 时，必须这样写：
import * as express from 'express';

// 开启 esModuleInterop 后，可以这样写：
import express from 'express';
```

它会在编译时添加辅助函数 `__importDefault` 和 `__importStar`，确保 CJS 和 ESM 之间的互操作正确。

**建议：** 始终开启。

#### skipLibCheck - 跳过声明文件检查

```json
{
  "compilerOptions": {
    "skipLibCheck": true
  }
}
```

**什么时候开：**
- 大多数项目都应该开启
- 加快编译速度（不检查 node_modules 中的 .d.ts）
- 避免第三方库类型声明之间的冲突

**什么时候关：**
- 你在开发类型声明库（如 @types/xxx）
- 需要确保自己的 .d.ts 文件完全正确

#### declaration - 生成类型声明文件

```json
{
  "compilerOptions": {
    "declaration": true,
    "declarationDir": "./dist/types",
    "declarationMap": true
  }
}
```

- `declaration`：生成 `.d.ts` 文件
- `declarationDir`：声明文件输出目录
- `declarationMap`：生成 `.d.ts.map`，支持"跳转到源码"而非声明文件

**适用场景：** 库/包开发时必须开启，应用项目通常不需要。

## 2. 不同场景的推荐配置

### 前端项目（React + Vite）

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "jsx": "react-jsx",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "skipLibCheck": true,
    "noEmit": true,
    "baseUrl": ".",
    "paths": { "@/*": ["src/*"] }
  },
  "include": ["src"]
}
```

### Node.js 后端项目

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "Node16",
    "moduleResolution": "Node16",
    "strict": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "declaration": true,
    "sourceMap": true,
    "esModuleInterop": true
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist"]
}
```

### 库开发

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "outDir": "./dist"
  },
  "include": ["src"]
}
```

## 3. 构建工具集成

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| tsc | 官方编译器，类型检查+编译 | 库开发、简单项目 |
| Vite | 开发快（esbuild），只做类型检查 | 前端应用 |
| esbuild/swc | 极快编译，不做类型检查 | 需要配合 tsc --noEmit |
| Webpack | 生态丰富，配置复杂 | 大型遗留项目 |

**为什么 Vite/esbuild 更快？**

它们只做语法转换（擦除类型），不做类型检查。类型检查由 IDE 和 CI 中的 `tsc --noEmit` 负责。这种分离让开发体验极快。

## 6. 从 JavaScript 迁移到 TypeScript

**渐进式迁移策略：**

1. 添加 tsconfig.json，设置 `allowJs: true`
2. 将 `.js` 文件逐个重命名为 `.ts`
3. 先用 `any` 消除错误，再逐步添加精确类型
4. 开启 `strict: true`（最后一步）

```json
// 迁移期间的 tsconfig
{
  "compilerOptions": {
    "allowJs": true,
    "checkJs": false,
    "strict": false,
    "noImplicitAny": false
  }
}
```
