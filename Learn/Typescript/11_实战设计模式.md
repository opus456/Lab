# TypeScript 实战设计模式

## 概述

设计模式是软件工程中经过验证的解决方案模板。TypeScript 的类型系统为经典设计模式带来了编译时安全性，使得模式的约束可以在代码编写阶段就被强制执行，而非依赖运行时检查或文档约定。

本章将每个模式放在真实工程场景中讲解，重点关注：
- 模式解决什么问题
- TypeScript 类型系统如何增强模式的安全性
- 实际项目中的典型应用

## 一、创建型模式

### 1.1 单例模式 (Singleton)

**为什么用：** 确保全局只有一个实例，常用于配置管理、数据库连接池、日志服务等需要共享状态的场景。

**工程场景：** 应用配置管理器——整个应用共享同一份配置，避免重复读取和不一致。

```typescript
class AppConfig {
  private static instance: AppConfig;
  private config: Map<string, unknown> = new Map();

  // 私有构造函数阻止外部 new
  private constructor() {}

  static getInstance(): AppConfig {
    if (!AppConfig.instance) {
      AppConfig.instance = new AppConfig();
    }
    return AppConfig.instance;
  }

  set<T>(key: string, value: T): void {
    this.config.set(key, value);
  }

  get<T>(key: string): T | undefined {
    return this.config.get(key) as T | undefined;
  }
}

// 使用
const config = AppConfig.getInstance();
config.set('dbHost', 'localhost');
config.set('dbPort', 5432);

// 任何地方获取的都是同一个实例
const sameConfig = AppConfig.getInstance();
console.log(sameConfig.get<string>('dbHost')); // 'localhost'
```

**TypeScript 增强点：** 私有构造函数在编译期阻止 `new AppConfig()`，泛型方法保证类型安全的存取。

**数据库连接池单例：**

```typescript
interface PoolOptions {
  host: string;
  port: number;
  maxConnections: number;
}

class DatabasePool {
  private static instance: DatabasePool;
  private connections: Array<{ id: number; busy: boolean }> = [];

  private constructor(private options: PoolOptions) {
    for (let i = 0; i < options.maxConnections; i++) {
      this.connections.push({ id: i, busy: false });
    }
  }

  static initialize(options: PoolOptions): DatabasePool {
    if (!DatabasePool.instance) {
      DatabasePool.instance = new DatabasePool(options);
    }
    return DatabasePool.instance;
  }

  static getInstance(): DatabasePool {
    if (!DatabasePool.instance) {
      throw new Error('DatabasePool not initialized. Call initialize() first.');
    }
    return DatabasePool.instance;
  }

  acquire(): { id: number; busy: boolean } | null {
    const conn = this.connections.find(c => !c.busy);
    if (conn) {
      conn.busy = true;
      return conn;
    }
    return null;
  }

  release(id: number): void {
    const conn = this.connections.find(c => c.id === id);
    if (conn) conn.busy = false;
  }
}
```

### 1.2 工厂模式 (Factory)

**为什么用：** 将对象创建逻辑集中管理，调用方无需关心具体实现类，便于扩展和替换。

**工程场景：** 通知服务——根据渠道类型创建不同的通知发送器。

```typescript
// 产品接口
interface Notification {
  send(to: string, message: string): Promise<boolean>;
  readonly channel: string;
}

// 具体产品
class EmailNotification implements Notification {
  readonly channel = 'email';

  async send(to: string, message: string): Promise<boolean> {
    console.log(`Sending email to ${to}: ${message}`);
    return true;
  }
}

class SmsNotification implements Notification {
  readonly channel = 'sms';

  async send(to: string, message: string): Promise<boolean> {
    console.log(`Sending SMS to ${to}: ${message}`);
    return true;
  }
}

class PushNotification implements Notification {
  readonly channel = 'push';

  async send(to: string, message: string): Promise<boolean> {
    console.log(`Sending push to ${to}: ${message}`);
    return true;
  }
}

// 工厂
type NotificationChannel = 'email' | 'sms' | 'push';

class NotificationFactory {
  private static creators: Map<NotificationChannel, () => Notification> = new Map([
    ['email', () => new EmailNotification()],
    ['sms', () => new SmsNotification()],
    ['push', () => new PushNotification()],
  ]);

  static create(channel: NotificationChannel): Notification {
    const creator = this.creators.get(channel);
    if (!creator) {
      throw new Error(`Unknown channel: ${channel}`);
    }
    return creator();
  }

  // 支持注册新渠道（开闭原则）
  static register(channel: NotificationChannel, creator: () => Notification): void {
    this.creators.set(channel, creator);
  }
}

// 使用
const notification = NotificationFactory.create('email');
notification.send('user@example.com', 'Hello!');
```

**TypeScript 增强点：** 联合类型 `NotificationChannel` 限制了合法的渠道值，`Notification` 接口确保所有产品实现统一契约。

### 1.3 建造者模式 (Builder)

**为什么用：** 分步构建复杂对象，避免构造函数参数爆炸，支持链式调用使代码更具可读性。

**工程场景：** HTTP 请求构建器、SQL 查询构建器。

```typescript
interface HttpRequest {
  url: string;
  method: 'GET' | 'POST' | 'PUT' | 'DELETE';
  headers: Record<string, string>;
  body?: unknown;
  timeout: number;
  retries: number;
}

class RequestBuilder {
  private request: Partial<HttpRequest> = {};

  setUrl(url: string): this {
    this.request.url = url;
    return this;
  }

  setMethod(method: HttpRequest['method']): this {
    this.request.method = method;
    return this;
  }

  setHeader(key: string, value: string): this {
    if (!this.request.headers) {
      this.request.headers = {};
    }
    this.request.headers[key] = value;
    return this;
  }

  setBody(body: unknown): this {
    this.request.body = body;
    return this;
  }

  build(): HttpRequest {
    if (!this.request.url || !this.request.method) {
      throw new Error('URL and method are required');
    }
    return this.request as HttpRequest;
  }
}

// 使用：链式调用，类型安全
const request = new RequestBuilder()
  .setUrl('/api/users')
  .setMethod('POST')
  .setHeader('Content-Type', 'application/json')
  .setBody({ name: 'Alice' })
  .build();
```

## 二、结构型模式

### 适配器模式 (Adapter)

将不兼容的接口转换为期望的接口：

```typescript
// 旧的支付接口
interface LegacyPayment {
  makePayment(amount: number, currency: string): boolean;
}

// 新的统一支付接口
interface PaymentProcessor {
  pay(request: { amount: number; currency: string; metadata?: Record<string, string> }): Promise<{ success: boolean; transactionId: string }>;
}

// 适配器：让旧接口符合新接口
class LegacyPaymentAdapter implements PaymentProcessor {
  constructor(private legacy: LegacyPayment) {}

  async pay(request: { amount: number; currency: string }) {
    const success = this.legacy.makePayment(request.amount, request.currency);
    return { success, transactionId: `legacy-${Date.now()}` };
  }
}
```

### 观察者模式 (Observer)

```typescript
type Listener<T> = (data: T) => void;

class EventBus<Events extends Record<string, any>> {
  private listeners = new Map<keyof Events, Set<Function>>();

  on<K extends keyof Events>(event: K, listener: Listener<Events[K]>): () => void {
    if (!this.listeners.has(event)) this.listeners.set(event, new Set());
    this.listeners.get(event)!.add(listener);
    return () => this.listeners.get(event)?.delete(listener);
  }

  emit<K extends keyof Events>(event: K, data: Events[K]) {
    this.listeners.get(event)?.forEach(fn => fn(data));
  }
}

// 类型安全的事件总线
interface AppEvents {
  'cart:add': { productId: string; quantity: number };
  'cart:remove': { productId: string };
  'checkout:complete': { orderId: string; total: number };
}

const bus = new EventBus<AppEvents>();
const unsubscribe = bus.on('cart:add', (data) => {
  console.log(`Added ${data.quantity} of ${data.productId}`);
});
```

## 三、行为型模式

### 策略模式 (Strategy)

```typescript
interface SortStrategy<T> {
  sort(data: T[]): T[];
}

class QuickSort<T> implements SortStrategy<T> {
  sort(data: T[]): T[] { return [...data].sort(); }
}

class MergeSort<T> implements SortStrategy<T> {
  sort(data: T[]): T[] { /* merge sort implementation */ return data; }
}

class Sorter<T> {
  constructor(private strategy: SortStrategy<T>) {}
  setStrategy(strategy: SortStrategy<T>) { this.strategy = strategy; }
  sort(data: T[]): T[] { return this.strategy.sort(data); }
}
```

## 四、函数式模式

### Result/Either 模式（类型安全的错误处理）

```typescript
type Result<T, E = Error> =
  | { ok: true; value: T }
  | { ok: false; error: E };

function ok<T>(value: T): Result<T, never> {
  return { ok: true, value };
}

function err<E>(error: E): Result<never, E> {
  return { ok: false, error };
}

// 使用：不用 try-catch，错误成为类型的一部分
function parseJSON<T>(input: string): Result<T, string> {
  try {
    return ok(JSON.parse(input));
  } catch (e) {
    return err(`Invalid JSON: ${input}`);
  }
}

const result = parseJSON<{ name: string }>('{"name": "Alice"}');
if (result.ok) {
  console.log(result.value.name); // 类型安全
} else {
  console.error(result.error);    // 类型安全
}
```

### 管道模式 (Pipe)

```typescript
function pipe<A, B>(fn1: (a: A) => B): (a: A) => B;
function pipe<A, B, C>(fn1: (a: A) => B, fn2: (b: B) => C): (a: A) => C;
function pipe<A, B, C, D>(fn1: (a: A) => B, fn2: (b: B) => C, fn3: (c: C) => D): (a: A) => D;
function pipe(...fns: Function[]) {
  return (input: any) => fns.reduce((acc, fn) => fn(acc), input);
}

const processUser = pipe(
  (name: string) => name.trim(),
  (name: string) => name.toLowerCase(),
  (name: string) => ({ name, createdAt: new Date() })
);

const user = processUser("  ALICE  "); // { name: "alice", createdAt: Date }
```
