# 第四章：泛型 (Generics)

泛型是 TypeScript 中最强大的类型系统特性之一。它允许我们编写可复用、类型安全的代码，同时保持灵活性。本章将从基础到高级，全面讲解泛型的方方面面。

---

## 1. 泛型基础

### 1.1 为什么需要泛型 - 从一个实际问题出发

假设我们需要编写一个函数，它接收一个值并原样返回（identity 函数）：

```typescript
// 方案一：使用具体类型 - 不够通用
function identityNumber(arg: number): number {
  return arg;
}

function identityString(arg: string): string {
  return arg;
}

// 方案二：使用 any - 丢失类型信息
function identityAny(arg: any): any {
  return arg;
}

const result = identityAny("hello");
// result 的类型是 any，我们丢失了它是 string 的信息
// result.toFixed() 不会报错，但运行时会崩溃
```

**问题所在：**
- 方案一需要为每种类型写一个函数，代码重复
- 方案二虽然通用，但完全丢失了类型安全性

**泛型的解决方案：**

```typescript
function identity<T>(arg: T): T {
  return arg;
}

// 显式指定类型参数
const result1 = identity<string>("hello"); // 类型为 string
const result2 = identity<number>(42);       // 类型为 number

// 类型推断 - TypeScript 自动推断 T
const result3 = identity("hello"); // 类型为 "hello"（字面量类型）
const result4 = identity(42);      // 类型为 42（字面量类型）
```

泛型用 `<T>` 声明一个**类型变量**，它在函数被调用时才确定具体类型。这样既保持了通用性，又不丢失类型信息。

### 1.2 泛型函数

泛型函数可以有一个或多个类型参数：

```typescript
// 单个类型参数
function firstElement<T>(arr: T[]): T | undefined {
  return arr[0];
}
const first = firstElement([1, 2, 3]);       // number | undefined
const firstStr = firstElement(["a", "b"]);   // string | undefined

// 多个类型参数
function pair<T, U>(first: T, second: U): [T, U] {
  return [first, second];
}

const p = pair("hello", 42); // [string, number]

// 类型参数之间可以有关系
function map<Input, Output>(
  arr: Input[],
  fn: (item: Input) => Output
): Output[] {
  return arr.map(fn);
}

const lengths = map(["hello", "world"], (s) => s.length); // number[]
const parsed = map(["1", "2", "3"], (s) => parseInt(s));  // number[]
```

**泛型函数的类型签名：**

```typescript
// 函数类型表达式
type GenericFn = <T>(arg: T) => T;

// 调用签名
type GenericFnObj = {
  <T>(arg: T): T;
};

// 两者等价
const myIdentity: GenericFn = identity;
const myIdentity2: GenericFnObj = identity;
```

### 1.3 泛型接口

接口也可以使用泛型参数：

```typescript
// 泛型接口 - 描述一个容器
interface Container<T> {
  value: T;
  getValue(): T;
  setValue(newValue: T): void;
}

// 实现泛型接口
class NumberContainer implements Container<number> {
  constructor(public value: number) {}
  getValue(): number {
    return this.value;
  }
  setValue(newValue: number): void {
    this.value = newValue;
  }
}
// 泛型接口 - 描述数据仓库
interface Repository<T> {
  findById(id: string): Promise<T | null>;
  findAll(): Promise<T[]>;
  create(item: Omit<T, "id">): Promise<T>;
  update(id: string, item: Partial<T>): Promise<T>;
  delete(id: string): Promise<boolean>;
}

// 使用泛型接口
interface User {
  id: string;
  name: string;
  email: string;
}

interface Product {
  id: string;
  title: string;
  price: number;
}

// 同一个接口，不同的类型实例
class UserRepository implements Repository<User> {
  async findById(id: string): Promise<User | null> {
    // 实际数据库查询逻辑
    return null;
  }
  async findAll(): Promise<User[]> { return []; }
  async create(item: Omit<User, "id">): Promise<User> {
    return { id: "1", ...item };
  }
  async update(id: string, item: Partial<User>): Promise<User> {
    return { id, name: "", email: "", ...item };
  }
  async delete(id: string): Promise<boolean> { return true; }
}
```

### 1.4 泛型类

类可以在类名后声明类型参数：

```typescript
// 泛型栈实现
class Stack<T> {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }
  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  isEmpty(): boolean {
    return this.items.length === 0;
  }

  size(): number {
    return this.items.length;
  }
}

const numberStack = new Stack<number>();
numberStack.push(1);
numberStack.push(2);
numberStack.pop(); // number | undefined

const stringStack = new Stack<string>();
stringStack.push("hello");
// stringStack.push(42); // 错误！不能将 number 赋给 string

// 泛型类 - 链表节点
class LinkedListNode<T> {
  constructor(
    public value: T,
    public next: LinkedListNode<T> | null = null
  ) {}
}

class LinkedList<T> {
  private head: LinkedListNode<T> | null = null;

  append(value: T): void {
    const newNode = new LinkedListNode(value);
    if (!this.head) {
      this.head = newNode;
      return;
    }
    let current = this.head;
    while (current.next) {
      current = current.next;
    }
    current.next = newNode;
  }

  toArray(): T[] {
    const result: T[] = [];
    let current = this.head;
    while (current) {
      result.push(current.value);
      current = current.next;
    }
    return result;
  }
}
```

**注意：** 静态成员不能使用类的类型参数：

```typescript
class MyClass<T> {
  // static defaultValue: T; // 错误！静态成员不能引用类型参数
  static instanceCount: number = 0; // 这是可以的
  value: T; // 实例成员可以使用类型参数

  constructor(value: T) {
    this.value = value;
    MyClass.instanceCount++;
  }
}
```

### 1.5 泛型类型别名

类型别名同样支持泛型：

// 简单泛型类型别名
type Nullable<T> = T | null;

// 使用

const fileTree: TreeNode<string> = {
  value: "root",

## 2. 泛型约束 (Constraints)

### 2.1 extends 关键字

没有约束的泛型参数可以是任何类型，这有时会导致问题：

```typescript
// 问题：T 可以是任何类型，不一定有 length 属性
function logLength<T>(arg: T): T {
  // console.log(arg.length); // 错误！T 上不存在 length
  return arg;
}

// 解决方案：使用 extends 添加约束
interface HasLength {
  length: number;
}

function logLength<T extends HasLength>(arg: T): T {
  console.log(arg.length); // OK！T 一定有 length 属性
  return arg;
}

logLength("hello");       // OK，string 有 length
logLength([1, 2, 3]);     // OK，数组有 length
logLength({ length: 10 }); // OK，对象有 length 属性
// logLength(123);         // 错误！number 没有 length 属性
```

**约束的本质：** `T extends Constraint` 意味着 T 必须是 Constraint 的子类型（至少包含 Constraint 的所有属性）。

```typescript
// 约束为对象类型
function getProperty<T extends object, K extends keyof T>(
  obj: T,
  key: K
): T[K] {
  return obj[key];
}
const user = { name: "Alice", age: 30 };
const name = getProperty(user, "name");  // string
const age = getProperty(user, "age");    // number
// getProperty(user, "email");            // 错误！"email" 不在 keyof typeof user 中
```

### 2.2 keyof 约束

`keyof` 操作符获取一个类型的所有键的联合类型，常与泛型约束配合使用：

```typescript
// keyof 基础
type UserKeys = keyof User; // "id" | "name" | "email"

// 泛型中使用 keyof 约束
function pick<T, K extends keyof T>(obj: T, keys: K[]): Pick<T, K> {
  const result = {} as Pick<T, K>;
  keys.forEach((key) => {
    result[key] = obj[key];
  });
  return result;
}

const user = { id: "1", name: "Alice", email: "alice@example.com", age: 30 };
const nameAndEmail = pick(user, ["name", "email"]);
// 类型为 Pick<typeof user, "name" | "email">
// 即 { name: string; email: string }

// 动态属性访问
function setProperty<T, K extends keyof T>(
  obj: T,
  key: K,
  value: T[K]
): void {
  obj[key] = value;
}

setProperty(user, "name", "Bob");    // OK
// setProperty(user, "name", 123);   // 错误！number 不能赋给 string
// setProperty(user, "foo", "bar");  // 错误！"foo" 不是有效的键
```

### 2.3 多重约束

TypeScript 不支持 `T extends A, B` 的语法，但可以通过交叉类型实现多重约束：

```typescript
interface Serializable {
  serialize(): string;
}

interface Loggable {
  log(): void;
}

// 多重约束：T 必须同时满足 Serializable 和 Loggable
function process<T extends Serializable & Loggable>(item: T): string {
  item.log();
  return item.serialize();
}
class LogEntry implements Serializable, Loggable {
  constructor(private message: string) {}
  serialize(): string {
    return JSON.stringify({ message: this.message });
  }
  log(): void {
    console.log(this.message);
  }
}

process(new LogEntry("hello")); // OK

// 约束中使用构造函数签名
interface Constructable<T> {
  new (...args: any[]): T;
}

function createInstance<T>(ctor: Constructable<T>): T {
  return new ctor();
}

class MyService {
  name = "MyService";
}

const service = createInstance(MyService); // MyService 类型
```

### 2.4 为什么需要约束 - 类型安全与灵活性的平衡

泛型约束的设计哲学是在**最大灵活性**和**最小必要类型安全**之间找到平衡：

```typescript
// 过于宽松 - 没有约束，无法安全操作
function badMerge<T>(target: T, source: T): T {
  // 我们想做 {...target, ...source}，但 T 可能不是对象
  return { ...target, ...source }; // 可能运行时出错
}

// 过于严格 - 失去泛型的意义
function tooStrict(target: Record<string, any>, source: Record<string, any>) {
  return { ...target, ...source }; // 返回类型丢失了具体信息
}

// 恰到好处 - 约束为对象，保留具体类型
function merge<T extends object, U extends object>(
  target: T,
  source: U
): T & U {
  return { ...target, ...source };
}

const merged = merge(
  { name: "Alice" },
  { age: 30 }
);
// 类型为 { name: string } & { age: number }
// 即 { name: string; age: number }
```

## 3. 泛型推断

### 3.1 TypeScript 如何推断泛型参数
TypeScript 编译器会根据传入的参数自动推断泛型类型参数，无需显式指定：

```typescript
// 从参数推断
function wrap<T>(value: T): { value: T } {
  return { value };
}

const wrapped = wrap("hello"); // T 被推断为 string
// wrapped 的类型为 { value: string }

// 从多个参数推断
function combine<T>(a: T, b: T): T[] {
  return [a, b];
}

const nums = combine(1, 2);       // T 推断为 number
const strs = combine("a", "b");   // T 推断为 string
// combine(1, "a");                // 错误！number 和 string 不兼容

// 从返回值上下文推断
function createArray<T>(): T[] {
  return [];
}

// 需要显式指定，因为没有参数可供推断
const arr: string[] = createArray<string>();

// 从回调函数推断
function transform<T, U>(items: T[], fn: (item: T) => U): U[] {
  return items.map(fn);
}

// T 从 items 推断为 number，U 从回调返回值推断为 string
const result = transform([1, 2, 3], (n) => n.toString());
// result: string[]
```

**推断的优先级规则：**

```typescript
// 当多个位置提供推断信息时，TypeScript 会尝试找到最佳公共类型
function longest<T extends { length: number }>(a: T, b: T): T {
  return a.length >= b.length ? a : b;
}

// 两个参数都是 string，推断 T = string
const longerStr = longest("hello", "hi"); // string

// 两个参数都是数组，推断 T = number[]
const longerArr = longest([1, 2, 3], [4, 5]); // number[]

// 注意：不会推断为联合类型
// longest("hello", [1, 2]); // 错误！string 和 number[] 不兼容
```

### 3.2 推断失败的情况和解决方案
```typescript
// 情况1：没有足够的推断上下文
function createState<T>(): [T, (newValue: T) => void] {
  let state: T;
  return [state!, (newValue: T) => { state = newValue; }];
}

// 推断失败，T 为 unknown
// const [value, setValue] = createState(); // value: unknown

// 解决方案：显式指定类型参数
const [value, setValue] = createState<number>(); // value: number

// 情况2：推断结果过于宽泛
function fetchData<T>(url: string): Promise<T> {
  return fetch(url).then((res) => res.json());
}

// T 无法从 string 参数推断，必须显式指定
const userData = await fetchData<User>("/api/users/1");

// 情况3：多个类型参数部分推断
function assign<T, U>(target: T, source: U): T & U {
  return { ...target, ...source };
}

// 如果只想指定其中一个，需要全部指定
// TypeScript 不支持部分类型参数推断（截至 5.x 版本）
const result = assign<{ name: string }, { age: number }>(
  { name: "Alice" },
  { age: 30 }
);

// 解决方案：使用柯里化拆分类型参数
function assignCurried<T>(target: T) {
  return function <U>(source: U): T & U {
    return { ...target, ...source };
  };
}

const result2 = assignCurried({ name: "Alice" })({ age: 30 });
// T 和 U 分别从各自的参数推断
```

### 3.3 infer 关键字（条件类型中的推断）

`infer` 是条件类型中的强大工具，允许我们从类型中"提取"部分类型：

```typescript
// 基本语法：在 extends 子句中使用 infer 声明待推断的类型变量
type ReturnTypeOf<T> = T extends (...args: any[]) => infer R ? R : never;

// 使用
type Fn1 = () => string;
type Fn2 = (x: number) => boolean;

type R1 = ReturnTypeOf<Fn1>; // string
type R2 = ReturnTypeOf<Fn2>; // boolean
// 提取函数参数类型
type ParametersOf<T> = T extends (...args: infer P) => any ? P : never;

type P1 = ParametersOf<(x: string, y: number) => void>; // [string, number]

// 提取 Promise 内部类型
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T;

type A = UnwrapPromise<Promise<string>>;  // string
type B = UnwrapPromise<Promise<number>>;  // number
type C = UnwrapPromise<string>;           // string（不是 Promise，原样返回）

// 递归解包嵌套 Promise
type DeepUnwrapPromise<T> = T extends Promise<infer U>
  ? DeepUnwrapPromise<U>
  : T;

type D = DeepUnwrapPromise<Promise<Promise<Promise<number>>>>; // number

// 提取数组元素类型
type ElementOf<T> = T extends (infer E)[] ? E : never;

type E1 = ElementOf<string[]>;   // string
type E2 = ElementOf<number[]>;   // number

// 提取对象值类型
type ValueOf<T> = T extends Record<any, infer V> ? V : never;

type V1 = ValueOf<{ a: string; b: number }>; // string | number

// infer 在元组中的应用
type First<T extends any[]> = T extends [infer F, ...any[]] ? F : never;
type Last<T extends any[]> = T extends [...any[], infer L] ? L : never;
type Rest<T extends any[]> = T extends [any, ...infer R] ? R : never;

type F = First<[1, 2, 3]>;  // 1
type L = Last<[1, 2, 3]>;   // 3
type R = Rest<[1, 2, 3]>;   // [2, 3]

// infer 配合模板字面量类型
type ExtractRouteParams<T extends string> =
  T extends `${string}:${infer Param}/${infer Rest}`
    ? Param | ExtractRouteParams<`/${Rest}`>
    : T extends `${string}:${infer Param}`
      ? Param
      : never;

type Params = ExtractRouteParams<"/users/:userId/posts/:postId">;
// "userId" | "postId"
```

## 4. 泛型默认值

### 4.1 默认类型参数
类似函数参数的默认值，泛型类型参数也可以有默认类型：

```typescript
// 基本语法
interface EventEmitter<T = Record<string, any>> {
  on<K extends keyof T>(event: K, handler: (payload: T[K]) => void): void;
  emit<K extends keyof T>(event: K, payload: T[K]): void;
}

// 使用默认类型 - 不指定类型参数
const emitter: EventEmitter = {
  on(event, handler) { /* ... */ },
  emit(event, payload) { /* ... */ },
};

// 指定具体类型 - 覆盖默认值
interface AppEvents {
  login: { userId: string };
  logout: void;
  error: Error;
}

const appEmitter: EventEmitter<AppEvents> = {
  on(event, handler) { /* ... */ },
  emit(event, payload) { /* ... */ },
};

// appEmitter.on("login", (payload) => {
//   payload.userId; // string，类型安全！
// });

// 默认值可以引用前面的类型参数
interface ApiClient<
  TResponse = any,
  TError = Error,
  TConfig extends object = Record<string, any>
> {
  get(url: string, config?: TConfig): Promise<TResponse>;
  post(url: string, data: any, config?: TConfig): Promise<TResponse>;
  handleError(error: TError): void;
}

// 带约束的默认值
type Stringify<T extends { toString(): string } = string> = `${T}`;
```

### 4.2 使用场景

```typescript
// 场景1：组件 Props 默认值
interface TableProps<T = Record<string, any>> {
  data: T[];
  columns: Array<{
    key: keyof T;
    title: string;
    render?: (value: T[keyof T], record: T) => string;
  }>;
  onRowClick?: (record: T) => void;
}
// 场景2：工厂函数的默认配置类型
interface CacheOptions<TKey = string, TValue = any> {
  maxSize: number;
  ttl: number;
  serialize?: (value: TValue) => string;
  deserialize?: (raw: string) => TValue;
  keyTransform?: (key: TKey) => string;
}

class Cache<TKey = string, TValue = any> {
  private store = new Map<string, { value: TValue; expires: number }>();

  constructor(private options: CacheOptions<TKey, TValue>) {}

  set(key: TKey, value: TValue): void {
    const strKey = this.options.keyTransform
      ? this.options.keyTransform(key)
      : String(key);
    this.store.set(strKey, {
      value,
      expires: Date.now() + this.options.ttl,
    });
  }

  get(key: TKey): TValue | undefined {
    const strKey = this.options.keyTransform
      ? this.options.keyTransform(key)
      : String(key);
    const entry = this.store.get(strKey);
    if (!entry || entry.expires < Date.now()) return undefined;
    return entry.value;
  }
}

// 使用默认类型
const simpleCache = new Cache({ maxSize: 100, ttl: 60000 });
simpleCache.set("key", { any: "value" }); // TKey=string, TValue=any

// 指定具体类型
const typedCache = new Cache<number, User>({
  maxSize: 50,
  ttl: 300000,
  keyTransform: (id) => `user_${id}`,
});
typedCache.set(1, { id: "1", name: "Alice", email: "a@b.com" });
```

## 5. 泛型工具类型实现原理

TypeScript 内置了许多基于泛型的工具类型。理解它们的实现原理有助于我们编写自己的工具类型。

### 5.1 Partial\<T\> 的实现

将所有属性变为可选：
```typescript
// 源码实现
type MyPartial<T> = {
  [P in keyof T]?: T[P];
};

// 原理解析：
// 1. keyof T - 获取 T 的所有键的联合类型
// 2. [P in keyof T] - 映射类型，遍历每个键
// 3. ? - 将每个属性标记为可选
// 4. T[P] - 索引访问类型，获取原始属性的值类型

// 使用示例
interface Config {
  host: string;
  port: number;
  debug: boolean;
}

type PartialConfig = MyPartial<Config>;
// 等价于：
// {
//   host?: string;
//   port?: number;
//   debug?: boolean;
// }

// 实际应用：更新函数只需要部分字段
function updateConfig(current: Config, updates: Partial<Config>): Config {
  return { ...current, ...updates };
}

updateConfig(
  { host: "localhost", port: 3000, debug: false },
  { debug: true } // 只更新 debug
);

// 深度 Partial（递归版本）
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface NestedConfig {
  server: {
    host: string;
    port: number;
  };
  database: {
    url: string;
    pool: {
      min: number;
      max: number;
    };
  };
}

type DeepPartialConfig = DeepPartial<NestedConfig>;
// server?.host?, server?.port?, database?.url?, database?.pool?.min?, ...
```

### 5.2 Required\<T\> 的实现

将所有属性变为必需（移除可选标记）：
```typescript
// 源码实现
type MyRequired<T> = {
  [P in keyof T]-?: T[P];
};

// 原理解析：
// -? 是"移除可选修饰符"的语法
// 类似地，-readonly 可以移除 readonly 修饰符

interface PartialUser {
  name?: string;
  email?: string;
  age?: number;
}

type CompleteUser = MyRequired<PartialUser>;
// {
//   name: string;
//   email: string;
//   age: number;
// }

// 实际应用：确保配置完整
function startServer(config: Required<Partial<Config>>): void {
  // config 的所有属性都是必需的
  console.log(`Starting on ${config.host}:${config.port}`);
}
```

### 5.3 Pick\<T, K\> 的实现

从类型中选取部分属性：

```typescript
// 源码实现
type MyPick<T, K extends keyof T> = {
  [P in K]: T[P];
};

// 原理解析：
// 1. K extends keyof T - K 必须是 T 的键的子集
// 2. [P in K] - 只遍历 K 中的键（而非 T 的所有键）
// 3. T[P] - 保留原始值类型

interface FullUser {
  id: string;
  name: string;
  email: string;
  password: string;
  createdAt: Date;
  updatedAt: Date;
}

// 只暴露安全的字段给前端
type PublicUser = MyPick<FullUser, "id" | "name" | "email">;
// { id: string; name: string; email: string }

// 实际应用：API 响应只返回需要的字段
function toPublicUser(user: FullUser): PublicUser {
  return {
    id: user.id,
    name: user.name,
    email: user.email,
  };
}
```

### 5.4 Omit\<T, K\> 的实现

从类型中排除部分属性（Pick 的反操作）：
```typescript
// 源码实现
type MyOmit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>;

// 拆解：
// 1. keyof T - 获取 T 的所有键
// 2. Exclude<keyof T, K> - 从 T 的键中排除 K 中的键
// 3. Pick<T, ...> - 选取剩余的键

// Exclude 的实现（分布式条件类型）
type MyExclude<T, U> = T extends U ? never : T;

// 示例
type ExcludeDemo = MyExclude<"a" | "b" | "c", "a" | "c">; // "b"

// Omit 使用
type UserWithoutPassword = MyOmit<FullUser, "password">;
// { id: string; name: string; email: string; createdAt: Date; updatedAt: Date }

// 实际应用：创建实体时不需要 id 和时间戳
type CreateUserInput = Omit<FullUser, "id" | "createdAt" | "updatedAt">;
// { name: string; email: string; password: string }
```

### 5.5 Record\<K, V\> 的实现

构造一个属性键为 K、属性值为 V 的对象类型：

```typescript
// 源码实现
type MyRecord<K extends keyof any, T> = {
  [P in K]: T;
};

// 原理解析：
// keyof any = string | number | symbol（所有可能的属性键类型）
// [P in K] - 遍历 K 中的每个键
// 每个键的值类型都是 T

// 使用示例
type Roles = "admin" | "user" | "guest";

interface Permission {
  read: boolean;
  write: boolean;
  delete: boolean;
}

type RolePermissions = Record<Roles, Permission>;
// {
//   admin: Permission;
//   user: Permission;
//   guest: Permission;
// }

const permissions: RolePermissions = {
  admin: { read: true, write: true, delete: true },
  user: { read: true, write: true, delete: false },
  guest: { read: true, write: false, delete: false },
};

// 实际应用：状态机
type State = "idle" | "loading" | "success" | "error";
type StateHandlers = Record<State, () => void>;
```

### 5.6 ReturnType\<T\> 的实现
获取函数返回值的类型：

```typescript
// 源码实现
type MyReturnType<T extends (...args: any) => any> =
  T extends (...args: any) => infer R ? R : any;

// 原理解析：
// 1. T extends (...args: any) => any - 约束 T 必须是函数类型
// 2. T extends (...args: any) => infer R - 用 infer 推断返回值类型
// 3. ? R : any - 如果匹配成功返回 R，否则返回 any

// 使用示例
function createUser(name: string, age: number) {
  return { id: Math.random().toString(), name, age, createdAt: new Date() };
}

type CreateUserReturn = ReturnType<typeof createUser>;
// { id: string; name: string; age: number; createdAt: Date }

// 实际应用：从函数推导类型，避免重复定义
const fetchUsers = async () => {
  const response = await fetch("/api/users");
  return response.json() as Promise<User[]>;
};

type FetchUsersResult = Awaited<ReturnType<typeof fetchUsers>>; // User[]
```

### 5.7 Parameters\<T\> 的实现

获取函数参数的类型（元组）：

```typescript
// 源码实现
type MyParameters<T extends (...args: any) => any> =
  T extends (...args: infer P) => any ? P : never;

// 使用示例
function updateUser(id: string, data: Partial<User>, notify: boolean) {
  // ...
}

type UpdateUserParams = Parameters<typeof updateUser>;
// [id: string, data: Partial<User>, notify: boolean]

// 实际应用：包装函数保持相同签名
function withLogging<T extends (...args: any[]) => any>(fn: T) {
  return function (...args: Parameters<T>): ReturnType<T> {
    console.log("Calling with:", args);
    const result = fn(...args);
    console.log("Result:", result);
    return result;
  };
}

const loggedUpdateUser = withLogging(updateUser);
// loggedUpdateUser 的类型与 updateUser 完全相同

// ConstructorParameters - 获取构造函数参数
type MyConstructorParameters<T extends abstract new (...args: any) => any> =
  T extends abstract new (...args: infer P) => any ? P : never;

class HttpClient {
  constructor(baseUrl: string, timeout: number) {}
}

type HttpClientParams = ConstructorParameters<typeof HttpClient>;
// [baseUrl: string, timeout: number]
```

### 6.1 递归泛型

泛型类型可以引用自身，实现递归类型定义：

```typescript
// 递归类型：深度只读
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object
    ? T[P] extends Function
      ? T[P]  // 函数保持原样
      : DeepReadonly<T[P]>  // 对象递归处理
    : T[P];  // 原始类型保持原样
};

interface AppState {
  user: {
    name: string;
    preferences: {
      theme: "light" | "dark";
      language: string;
    };
  };
  posts: Array<{ id: string; title: string }>;
}

type ReadonlyAppState = DeepReadonly<AppState>;
// 所有嵌套属性都是 readonly

// 递归类型：JSON 类型
type JsonValue =
  | string
  | number
  | boolean
  | null
  | JsonValue[]
  | { [key: string]: JsonValue };

// 递归类型：深度路径
type DeepKeys<T> = T extends object
  ? {
      [K in keyof T & string]: T[K] extends object
        ? K | `${K}.${DeepKeys<T[K]>}`
        : K;
    }[keyof T & string]
  : never;

interface Form {
  user: {
    name: string;
    address: {
      city: string;
      zip: string;
    };
  };
  settings: {
    theme: string;
  };
}

type FormPaths = DeepKeys<Form>;
// "user" | "user.name" | "user.address" | "user.address.city"
// | "user.address.zip" | "settings" | "settings.theme"

// 根据路径获取值类型
type DeepValue<T, Path extends string> =
  Path extends `${infer Key}.${infer Rest}`
    ? Key extends keyof T
      ? DeepValue<T[Key], Rest>
      : never
    : Path extends keyof T
      ? T[Path]
      : never;
type CityType = DeepValue<Form, "user.address.city">; // string
type ThemeType = DeepValue<Form, "settings.theme">;    // string

// 类型安全的深度取值函数
function getDeepValue<T, P extends DeepKeys<T>>(
  obj: T,
  path: P
): DeepValue<T, P> {
  const keys = (path as string).split(".");
  let result: any = obj;
  for (const key of keys) {
    result = result[key];
  }
  return result;
}
```

### 6.2 泛型与条件类型结合

条件类型与泛型结合可以实现强大的类型级编程：

```typescript
// 条件类型基础
type IsString<T> = T extends string ? true : false;

type A = IsString<string>;  // true
type B = IsString<number>;  // false

// 根据类型选择不同的返回类型
type TypeName<T> =
  T extends string ? "string" :
  T extends number ? "number" :
  T extends boolean ? "boolean" :
  T extends undefined ? "undefined" :
  T extends Function ? "function" :
  "object";

type T1 = TypeName<string>;    // "string"
type T2 = TypeName<() => void>; // "function"

// 条件类型实现类型过滤
type FilterByType<T, U> = {
  [K in keyof T as T[K] extends U ? K : never]: T[K];
};

interface Mixed {
  name: string;
  age: number;
  isActive: boolean;
  email: string;
  score: number;
}

type StringFields = FilterByType<Mixed, string>;
// { name: string; email: string }

type NumberFields = FilterByType<Mixed, number>;
// { age: number; score: number }

// 条件类型实现函数重载的替代方案
type ProcessResult<T> =
  T extends string ? string[] :
  T extends number ? number :
  never;

function process<T extends string | number>(input: T): ProcessResult<T> {
  if (typeof input === "string") {
    return input.split("") as ProcessResult<T>;
  }
  return (input * 2) as ProcessResult<T>;
}

const strResult = process("hello"); // string[]
const numResult = process(42);      // number
```
### 6.3 分布式条件类型

当条件类型作用于联合类型时，会自动"分布"到联合的每个成员上：

```typescript
// 分布式条件类型的触发条件：
// 1. 类型参数是裸类型（naked type parameter）
// 2. 被检查的类型是联合类型

type ToArray<T> = T extends any ? T[] : never;

// 分布式行为：联合类型的每个成员分别应用条件类型
type Result = ToArray<string | number>;
// = ToArray<string> | ToArray<number>
// = string[] | number[]
// 注意：不是 (string | number)[]

// 如果不想要分布式行为，用方括号包裹
type ToArrayNonDist<T> = [T] extends [any] ? T[] : never;

type Result2 = ToArrayNonDist<string | number>;
// = (string | number)[]

// 分布式条件类型的实际应用

// Extract - 从联合类型中提取匹配的成员
type MyExtract<T, U> = T extends U ? T : never;

type Numbers = MyExtract<string | number | boolean, number | boolean>;
// = number | boolean

// Exclude - 从联合类型中排除匹配的成员
type MyExclude<T, U> = T extends U ? never : T;

type WithoutNumber = MyExclude<string | number | boolean, number>;
// = string | boolean

// NonNullable - 排除 null 和 undefined
type MyNonNullable<T> = T extends null | undefined ? never : T;

type SafeString = MyNonNullable<string | null | undefined>; // string

// 分布式条件类型 + 映射类型 = 强大的类型变换
type EventHandlerMap<T> = {
  [K in keyof T as `on${Capitalize<string & K>}Change`]: (
    newValue: T[K],
    oldValue: T[K]
  ) => void;
};

interface UserState {
  name: string;
  age: number;
  active: boolean;
}

type UserHandlers = EventHandlerMap<UserState>;
// {
//   onNameChange: (newValue: string, oldValue: string) => void;
//   onAgeChange: (newValue: number, oldValue: number) => void;
//   onActiveChange: (newValue: boolean, oldValue: boolean) => void;
// }
```
### 6.4 协变与逆变

协变和逆变描述了类型参数在子类型关系中的行为方向：

```typescript
// 基础概念：
// 如果 A extends B（A 是 B 的子类型）
// 协变（Covariant）：Container<A> extends Container<B>（方向相同）
// 逆变（Contravariant）：Container<B> extends Container<A>（方向相反）
// 不变（Invariant）：两者都不成立

// TypeScript 中的规则：
// - 函数返回值位置：协变
// - 函数参数位置：逆变
// - 可变属性（读写）：不变

// 协变示例（返回值位置）
type Producer<T> = () => T;

type Animal = { name: string };
type Dog = { name: string; breed: string };
// Dog extends Animal

type ProducerAnimal = Producer<Animal>;
type ProducerDog = Producer<Dog>;

// ProducerDog extends ProducerAnimal（协变）
// 能产出 Dog 的地方一定能产出 Animal
const dogProducer: ProducerDog = () => ({ name: "Rex", breed: "Labrador" });
const animalProducer: ProducerAnimal = dogProducer; // OK

// 逆变示例（参数位置）
type Consumer<T> = (item: T) => void;

type ConsumerAnimal = Consumer<Animal>;
type ConsumerDog = Consumer<Dog>;

// ConsumerAnimal extends ConsumerDog（逆变，方向反转）
// 能消费 Animal 的函数一定能消费 Dog
const animalConsumer: ConsumerAnimal = (animal) => console.log(animal.name);
const dogConsumer: ConsumerDog = animalConsumer; // OK

// 不变示例（同时读写）
interface MutableBox<T> {
  value: T; // 既可读又可写
}

// MutableBox<Dog> 不能赋给 MutableBox<Animal>，反之亦然
// 因为如果允许，可能导致类型不安全：
// const dogBox: MutableBox<Dog> = { value: { name: "Rex", breed: "Lab" } };
// const animalBox: MutableBox<Animal> = dogBox; // 如果允许...
// animalBox.value = { name: "Cat" }; // 写入一个没有 breed 的 Animal
// dogBox.value.breed; // 运行时错误！

// TypeScript 4.7+ 支持显式标注变型
interface ReadonlyBox<out T> {  // out = 协变
  readonly value: T;
}

interface WriteOnlyBox<in T> {  // in = 逆变
  setValue(value: T): void;
}

interface InvariantBox<in out T> {  // in out = 不变
  value: T;
}
```
**为什么理解协变逆变很重要？**

```typescript
// 实际场景：事件处理器的类型安全
interface BaseEvent {
  type: string;
  timestamp: number;
}

interface ClickEvent extends BaseEvent {
  type: "click";
  x: number;
  y: number;
}

interface KeyEvent extends BaseEvent {
  type: "key";
  key: string;
}

// 事件处理器是逆变的（参数位置）
type EventHandler<T extends BaseEvent> = (event: T) => void;

// 通用处理器可以处理具体事件
const baseHandler: EventHandler<BaseEvent> = (e) => console.log(e.type);
const clickHandler: EventHandler<ClickEvent> = baseHandler; // OK（逆变）

// 但反过来不行
// const specificAsGeneral: EventHandler<BaseEvent> = clickHandler; // 错误！
```

### 7.1 通用 API 请求函数

```typescript
// 定义 API 响应的基础结构
interface ApiResponse<T> {
  success: boolean;
  data: T;
  error?: {
    code: string;
    message: string;
  };
  meta?: {
    page: number;
    total: number;
    pageSize: number;
  };
}

// HTTP 方法类型
type HttpMethod = "GET" | "POST" | "PUT" | "PATCH" | "DELETE";

// 请求配置
interface RequestConfig<TBody = unknown> {
  method: HttpMethod;
  headers?: Record<string, string>;
  body?: TBody;
  params?: Record<string, string | number | boolean>;
  timeout?: number;
}

// 类型安全的 API 路由定义
interface ApiRoutes {
  "/users": {
    GET: { response: User[]; params: { page?: number; limit?: number } };
    POST: { response: User; body: Omit<User, "id"> };
  };
  "/users/:id": {
    GET: { response: User; params: { id: string } };
    PUT: { response: User; body: Partial<User> };
    DELETE: { response: void };
  };
  "/posts": {
    GET: { response: Post[]; params: { authorId?: string } };
    POST: { response: Post; body: Omit<Post, "id" | "createdAt"> };
  };
}
interface Post {
  id: string;
  title: string;
  content: string;
  authorId: string;
  createdAt: Date;
}

// 提取路由参数
type ExtractPathParams<T extends string> =
  T extends `${string}:${infer Param}/${infer Rest}`
    ? Param | ExtractPathParams<Rest>
    : T extends `${string}:${infer Param}`
      ? Param
      : never;

// 类型安全的请求函数
class TypedApiClient {
  private baseUrl: string;

  constructor(baseUrl: string) {
    this.baseUrl = baseUrl;
  }

  async request<
    TPath extends keyof ApiRoutes,
    TMethod extends keyof ApiRoutes[TPath] & HttpMethod
  >(
    path: TPath,
    method: TMethod,
    config?: {
      params?: Record<string, string | number>;
      body?: ApiRoutes[TPath][TMethod] extends { body: infer B } ? B : never;
    }
  ): Promise<
    ApiResponse<
      ApiRoutes[TPath][TMethod] extends { response: infer R } ? R : never
    >
  > {
    let url = `${this.baseUrl}${path as string}`;

    // 替换路径参数
    if (config?.params) {
      Object.entries(config.params).forEach(([key, value]) => {
        url = url.replace(`:${key}`, String(value));
      });
    }

    const response = await fetch(url, {
      method,
      headers: { "Content-Type": "application/json" },
      body: config?.body ? JSON.stringify(config.body) : undefined,
    });

    return response.json();
  }
}

// 使用
const api = new TypedApiClient("https://api.example.com");

// 完全类型安全的 API 调用
async function demo() {
  // response.data 的类型自动推断为 User[]
  const users = await api.request("/users", "GET");

  // response.data 的类型自动推断为 User
  const newUser = await api.request("/users", "POST", {
    body: { name: "Alice", email: "alice@example.com" },
  });

  // 路径参数 + 类型安全
  const user = await api.request("/users/:id", "GET", {
    params: { id: "123" },
  });
}
```
### 7.2 通用表单验证

```typescript
// 验证规则类型
type ValidationRule<T> = {
  validate: (value: T) => boolean;
  message: string;
};

// 表单字段验证配置
type FormValidation<T> = {
  [K in keyof T]?: ValidationRule<T[K]>[];
};

// 验证结果
type ValidationErrors<T> = {
  [K in keyof T]?: string[];
};

// 通用表单验证器
class FormValidator<T extends Record<string, any>> {
  private rules: FormValidation<T>;

  constructor(rules: FormValidation<T>) {
    this.rules = rules;
  }

  validate(data: T): { valid: boolean; errors: ValidationErrors<T> } {
    const errors: ValidationErrors<T> = {};
    let valid = true;

    for (const key in this.rules) {
      const fieldRules = this.rules[key];
      if (!fieldRules) continue;

      const fieldErrors: string[] = [];
      for (const rule of fieldRules) {
        if (!rule.validate(data[key])) {
          fieldErrors.push(rule.message);
          valid = false;
        }
      }

      if (fieldErrors.length > 0) {
        errors[key] = fieldErrors;
      }
    }

    return { valid, errors };
  }
}

// 常用验证规则工厂
const Validators = {
  required<T>(): ValidationRule<T> {
    return {
      validate: (value) => value !== null && value !== undefined && value !== "",
      message: "此字段为必填项",
    };
  },

  minLength(min: number): ValidationRule<string> {
    return {
      validate: (value) => value.length >= min,
      message: `最少需要 ${min} 个字符`,
    };
  },
  maxLength(max: number): ValidationRule<string> {
    return {
      validate: (value) => value.length <= max,
      message: `最多允许 ${max} 个字符`,
    };
  },

  email(): ValidationRule<string> {
    return {
      validate: (value) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value),
      message: "请输入有效的邮箱地址",
    };
  },

  min(minValue: number): ValidationRule<number> {
    return {
      validate: (value) => value >= minValue,
      message: `不能小于 ${minValue}`,
    };
  },

  max(maxValue: number): ValidationRule<number> {
    return {
      validate: (value) => value <= maxValue,
      message: `不能大于 ${maxValue}`,
    };
  },

  pattern(regex: RegExp, message: string): ValidationRule<string> {
    return {
      validate: (value) => regex.test(value),
      message,
    };
  },
};

// 使用示例
interface RegisterForm {
  username: string;
  email: string;
  password: string;
  age: number;
}

const registerValidator = new FormValidator<RegisterForm>({
  username: [
    Validators.required(),
    Validators.minLength(3),
    Validators.maxLength(20),
    Validators.pattern(/^[a-zA-Z0-9_]+$/, "只允许字母、数字和下划线"),
  ],
  email: [
    Validators.required(),
    Validators.email(),
  ],
  password: [
    Validators.required(),
    Validators.minLength(8),
    Validators.pattern(
      /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/,
      "密码必须包含大小写字母和数字"
    ),
  ],
  age: [
    Validators.required(),
    Validators.min(18),
    Validators.max(120),
  ],
});

// 使用验证器
const result = validate(schema, {
  username: "ab",
  email: "invalid",
  password: "weak",
  age: 15,
});
// result 的类型精确包含每个字段的错误信息
```
