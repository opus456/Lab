# 声明文件与类型定义

## 1. 声明文件基础

### 什么是 .d.ts 文件

`.d.ts` 文件是 TypeScript 的声明文件（Declaration File），它只包含类型信息，不包含任何实现代码。编译后不会产生任何 JavaScript 输出。

```typescript
// math-utils.d.ts —— 只有类型，没有实现
declare function add(a: number, b: number): number;
declare function subtract(a: number, b: number): number;
declare const PI: number;
```

声明文件的作用是为已有的 JavaScript 代码提供类型描述，让 TypeScript 编译器能够进行类型检查和智能提示。

### 为什么需要声明文件

TypeScript 的生态中存在大量纯 JavaScript 编写的库（如 jQuery、Lodash 早期版本）。没有声明文件时：

```typescript
// 没有声明文件，TypeScript 不认识 $
$('#app').show(); // Error: Cannot find name '$'

// 没有声明文件，无法获得任何类型提示
import _ from 'lodash'; // Error: Could not find a declaration file for module 'lodash'
```

声明文件解决了三个核心问题：
- **类型安全**：让编译器能检查对第三方库的调用是否正确
- **开发体验**：提供自动补全、参数提示、跳转定义
- **文档作用**：类型本身就是最好的 API 文档

### declare 关键字的含义

`declare` 告诉编译器："这个东西存在于运行时环境中，但不是在这里定义的，你只需要知道它的类型。"

```typescript
// declare 不会生成任何 JS 代码
declare var jQuery: (selector: string) => any;

// 对比普通声明 —— 会生成 JS 代码
var myVar: string = "hello";

// declare 用于描述"外部存在"的东西
declare function require(module: string): any;
declare var process: { env: Record<string, string | undefined> };
```

关键理解：`declare` 是一种"承诺"——你告诉编译器这个值在运行时一定存在，编译器信任你但不验证。如果承诺是假的，运行时会报错。

## 2. 编写声明文件

### 全局变量声明 (declare var/let/const)

当 JavaScript 通过 `<script>` 标签引入全局变量时，需要用 `declare` 声明：

```typescript
// globals.d.ts
declare var __DEV__: boolean;          // 可读写
declare let currentUser: string;       // 可读写
declare const VERSION: string;         // 只读

// 使用时
if (__DEV__) {
  console.log(`Version: ${VERSION}`);
}
```

`var` vs `let` vs `const` 的选择：
- `const`：值不会被重新赋值（推荐，大多数全局变量都是常量）
- `let`：值可能被重新赋值
- `var`：与 `let` 类似，但语义上更老旧

### 全局函数声明 (declare function)

```typescript
// globals.d.ts
declare function greet(name: string): void;

// 支持重载
declare function createElement(tag: 'div'): HTMLDivElement;
declare function createElement(tag: 'span'): HTMLSpanElement;
declare function createElement(tag: string): HTMLElement;

// 带可选参数和默认值描述
declare function ajax(url: string, options?: {
  method?: 'GET' | 'POST' | 'PUT' | 'DELETE';
  body?: string;
  headers?: Record<string, string>;
  timeout?: number;
}): Promise<any>;
```

### 全局类声明 (declare class)

```typescript
// globals.d.ts
declare class Animal {
  name: string;
  constructor(name: string);
  speak(): void;
  static create(name: string): Animal;
}

// 更复杂的例子：声明一个事件发射器
declare class EventEmitter {
  on(event: string, listener: (...args: any[]) => void): this;
  off(event: string, listener: (...args: any[]) => void): this;
  emit(event: string, ...args: any[]): boolean;
  once(event: string, listener: (...args: any[]) => void): this;
  removeAllListeners(event?: string): this;
}
```

### 全局枚举声明 (declare enum)

```typescript
// globals.d.ts
declare enum Direction {
  Up,
  Down,
  Left,
  Right
}

// 常量枚举（编译时内联）
declare const enum Color {
  Red = '#ff0000',
  Green = '#00ff00',
  Blue = '#0000ff'
}

// 使用
const dir: Direction = Direction.Up;
const color: Color = Color.Red;
```

注意：`declare enum` 假设运行时存在对应的枚举对象。如果实际 JS 中没有，使用 `declare const enum` 更安全（编译时内联，不依赖运行时对象）。

### 命名空间声明 (declare namespace)

命名空间用于描述通过全局对象暴露的复杂 API：

```typescript
// globals.d.ts
declare namespace MyLib {
  // 嵌套的接口
  interface Config {
    debug: boolean;
    version: string;
  }

  // 嵌套的函数
  function init(config: Config): void;
  function destroy(): void;

  // 嵌套的类
  class Plugin {
    constructor(name: string);
    activate(): void;
  }

  // 嵌套的命名空间
  namespace Utils {
    function formatDate(date: Date): string;
    function parseJSON(text: string): any;
  }
}

// 使用
MyLib.init({ debug: true, version: '1.0.0' });
const plugin = new MyLib.Plugin('analytics');
MyLib.Utils.formatDate(new Date());
```

### 模块声明 (declare module)

用于描述通过模块系统（CommonJS/ESM）导入的库：

```typescript
// lodash.d.ts
declare module 'lodash' {
  export function chunk<T>(array: T[], size: number): T[][];
  export function debounce<T extends (...args: any[]) => any>(
    func: T,
    wait: number
  ): T;
  export function cloneDeep<T>(value: T): T;
  export function get(object: any, path: string, defaultValue?: any): any;
}

// 使用
import { chunk, debounce } from 'lodash';
const chunks = chunk([1, 2, 3, 4], 2); // number[][]
```

模块声明也可以用通配符匹配：

```typescript
// 为所有 .vue 文件提供类型
declare module '*.vue' {
  import { DefineComponent } from 'vue';
  const component: DefineComponent<{}, {}, any>;
  export default component;
}

// 为所有 .yaml 文件提供类型
declare module '*.yaml' {
  const content: Record<string, any>;
  export default content;
}
```

## 3. 第三方库的类型

### DefinitelyTyped (@types/xxx)

[DefinitelyTyped](https://github.com/DefinitelyTyped/DefinitelyTyped) 是一个巨大的社区仓库，为数千个 JavaScript 库提供类型定义。安装方式：

```bash
# 安装 lodash 的类型
npm install --save-dev @types/lodash

# 安装 express 的类型
npm install --save-dev @types/express

# 安装 node 环境类型
npm install --save-dev @types/node

# 安装 react 相关类型
npm install --save-dev @types/react @types/react-dom
```

TypeScript 编译器会自动从 `node_modules/@types/` 目录加载类型。可以在 `tsconfig.json` 中控制：

```json
{
  "compilerOptions": {
    // 只加载指定的 @types 包（不写则加载所有）
    "types": ["node", "jest"],
    // 指定额外的类型搜索路径
    "typeRoots": ["./typings", "./node_modules/@types"]
  }
}
```

### 库自带类型

现代库通常自带类型定义，无需额外安装 `@types` 包：

```json
// axios 的 package.json
{
  "name": "axios",
  "types": "./index.d.ts",  // 指向类型入口
  // ...
}
```

判断一个库是否自带类型：
- 查看 `package.json` 中是否有 `types` 或 `typings` 字段
- 查看包根目录是否有 `.d.ts` 文件
- npm 页面上是否有 "TS" 标记（蓝色小图标表示自带，DT 图标表示需要 @types）

### 如何查找和安装类型包

```bash
# 方法1：直接尝试安装，不存在会报错
npm install --save-dev @types/library-name

# 方法2：使用 TypeSearch 网站搜索
# https://www.typescriptlang.org/dt/search

# 方法3：查看编译器错误提示
# TypeScript 会提示 "Try npm install @types/xxx"
```

### 类型版本与库版本的对应

`@types` 包的版本号通常与库的主版本号对应：

```bash
# express 4.x 对应 @types/express 4.x
npm install express@4
npm install --save-dev @types/express@4

# react 18.x 对应 @types/react 18.x
npm install react@18
npm install --save-dev @types/react@18
```

版本不匹配可能导致类型错误。最佳实践是在 `package.json` 中锁定主版本号一致。

## 4. 为无类型库编写声明

### 快速声明（any 大法）

当你急需使用一个没有类型的库时，最快的方式：

```typescript
// src/typings/untyped-lib.d.ts

// 方法1：整个模块声明为 any
declare module 'untyped-lib';
// 使用时：import anything from 'untyped-lib'; // anything: any

// 方法2：声明特定导出为 any
declare module 'legacy-utils' {
  export const doSomething: any;
  export default any;
}
```

### 逐步完善类型

```typescript
// 第一步：基本结构
declare module 'chart-lib' {
  export interface ChartOptions {
    width: number;
    height: number;
    title?: string;
  }

  export class Chart {
    constructor(element: HTMLElement, options: ChartOptions);
    render(data: number[]): void;
    destroy(): void;
  }
}

// 随着使用深入，逐步添加更多类型
```

## 5. 模块声明技巧

### 扩展已有模块

```typescript
// 给 express 的 Request 添加自定义属性
import 'express';

declare module 'express' {
  interface Request {
    user?: { id: string; role: string };
    requestId: string;
  }
}

// 声明非 JS 模块
declare module '*.css' {
  const classes: Record<string, string>;
  export default classes;
}

declare module '*.png' {
  const src: string;
  export default src;
}
```

### 声明全局变量

```typescript
// global.d.ts
declare global {
  interface Window {
    __APP_VERSION__: string;
    gtag: (...args: any[]) => void;
  }

  // 全局函数
  function __DEV__(): boolean;
}

export {}; // 必须有 export 使文件成为模块
```

```json
// package.json
{
  "name": "my-lib",
  "main": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {

      "import": "./dist/index.mjs",
      "require": "./dist/index.cjs"
    }
  }
}
```

## 7. 实际工程案例

```typescript
// 为公司内部 SDK 写声明文件
// company-sdk.d.ts
declare module '@company/sdk' {
  export interface SDKConfig {
    appId: string;
    secret: string;
    environment: 'dev' | 'staging' | 'prod';
  }

  export class SDK {
    constructor(config: SDKConfig);
    init(): Promise<void>;
    track(event: string, properties?: Record<string, any>): void;
    identify(userId: string, traits?: Record<string, any>): void;
  }

  export function createSDK(config: SDKConfig): SDK;
}
```
