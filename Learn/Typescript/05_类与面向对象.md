# 第五章：类与面向对象

TypeScript 为 JavaScript 的类系统添加了完整的类型支持，包括访问修饰符、抽象类、接口实现等面向对象特性。本章将系统讲解 TypeScript 中类与面向对象编程的核心概念。

---

## 1. 类的基础

### 1.1 类的定义和实例化

```typescript
class Person {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet(): string {
    return `你好，我是${this.name}，今年${this.age}岁`;
  }
}

const person = new Person("张三", 25);
console.log(person.greet()); // 你好，我是张三，今年25岁
```

### 1.2 构造函数

构造函数在实例化时自动调用，用于初始化对象状态：

```typescript
class Database {
  private connection: string;
  private maxRetries: number;

  constructor(host: string, port: number, maxRetries = 3) {
    this.connection = `${host}:${port}`;
    this.maxRetries = maxRetries;
    console.log(`连接到数据库: ${this.connection}`);
  }
}

// 使用默认参数
const db = new Database("localhost", 5432);
// 覆盖默认参数
const prodDb = new Database("prod-server", 5432, 5);
```

### 1.3 参数属性（Parameter Properties）简写

TypeScript 提供了一种简写方式，在构造函数参数前加上访问修饰符，自动声明并赋值属性：

```typescript
// 传统写法
class UserVerbose {
  public name: string;
  private email: string;
  protected role: string;

  constructor(name: string, email: string, role: string) {
    this.name = name;
    this.email = email;
    this.role = role;
  }
}

// 参数属性简写 - 效果完全相同
class User {
  constructor(
    public name: string,
    private email: string,
    protected role: string
  ) {}
  // 构造函数体可以为空，属性已自动声明和赋值
}

// 混合使用：部分参数属性 + 部分手动赋值
class Config {
  public readonly computedValue: string;

  constructor(
    public readonly host: string,
    public readonly port: number,
    private secret: string
  ) {
    this.computedValue = `${host}:${port}`;
  }
}
```

> **设计原因**：参数属性是 TypeScript 独有的语法糖，减少了大量样板代码。在实际项目中，大多数类都使用这种简写形式。

## 2. 访问修饰符

### 2.1 public, private, protected

```typescript
class Animal {
  public name: string;        // 任何地方都可访问（默认）
  private _health: number;    // 仅在类内部可访问
  protected speed: number;    // 在类内部和子类中可访问

  constructor(name: string, health: number, speed: number) {
    this.name = name;
    this._health = health;
    this.speed = speed;
  }

  // private 方法只能在类内部调用
  private regenerate() {
    this._health += 10;
  }

  // public 方法可以在任何地方调用
  public heal() {
    this.regenerate();
  }
}

class Dog extends Animal {
  run() {
    console.log(`${this.name} runs at speed ${this.speed}`); // OK: protected
    // console.log(this._health); // Error: private 不能在子类访问
  }
}

const dog = new Dog("Rex", 100, 5);
dog.name;    // OK: public
// dog._health; // Error: private
// dog.speed;   // Error: protected 不能在外部访问
```

### 2.2 #私有字段 vs private 关键字

```typescript
class BankAccount {
  // TypeScript 的 private：编译时检查，运行时仍可访问
  private balance1: number = 0;

  // ES2022 的 #私有字段：运行时真正私有
  #balance2: number = 0;

  deposit(amount: number) {
    this.#balance2 += amount;
  }
}

const account = new BankAccount();
// (account as any).balance1; // 运行时可以访问（private 只是编译时约束）
// (account as any).#balance2; // 运行时也无法访问（真正的私有）

// 选择建议：
// - 大多数场景用 private 就够了（编译时保护）
// - 需要运行时安全性（如库开发）时用 #
```

## 3. 继承

```typescript
class Shape {
  constructor(public color: string) {}

  area(): number {
    return 0;
  }

  describe(): string {
    return `A ${this.color} shape with area ${this.area()}`;
  }
}

class Circle extends Shape {
  constructor(color: string, public radius: number) {
    super(color); // 必须调用 super
  }

  // 方法重写
  area(): number {
    return Math.PI * this.radius ** 2;
  }
}

class Rectangle extends Shape {
  constructor(color: string, public width: number, public height: number) {
    super(color);
  }

  area(): number {
    return this.width * this.height;
  }
}

// 多态：父类引用指向子类实例
const shapes: Shape[] = [
  new Circle("red", 5),
  new Rectangle("blue", 3, 4),
];

shapes.forEach(s => console.log(s.describe()));
```

## 4. 抽象类

```typescript
// 抽象类不能直接实例化，只能被继承
abstract class Logger {
  // 抽象方法：子类必须实现
  abstract log(message: string): void;
  abstract getPrefix(): string;

  // 具体方法：子类可以直接使用
  info(message: string) {
    this.log(`[INFO] ${this.getPrefix()} ${message}`);
  }

  error(message: string) {
    this.log(`[ERROR] ${this.getPrefix()} ${message}`);
  }
}

class ConsoleLogger extends Logger {
  getPrefix() { return new Date().toISOString(); }
  log(message: string) { console.log(message); }
}

class FileLogger extends Logger {
  constructor(private filePath: string) { super(); }
  getPrefix() { return `[${this.filePath}]`; }
  log(message: string) { /* 写入文件 */ }
}

// const logger = new Logger(); // Error: 不能实例化抽象类
const logger = new ConsoleLogger();
logger.info("Server started"); // [INFO] 2024-01-01T00:00:00.000Z Server started
```

**抽象类 vs 接口的选择**：
- 抽象类：有共享的实现逻辑时使用（模板方法模式）
- 接口：只定义契约，不包含实现时使用

## 5. 接口实现 (implements)

```typescript
interface Serializable {
  serialize(): string;
  deserialize(data: string): void;
}

interface Validatable {
  validate(): boolean;
  getErrors(): string[];
}

// 一个类可以实现多个接口
class UserForm implements Serializable, Validatable {
  constructor(public name: string, public email: string) {}

  serialize(): string {
    return JSON.stringify({ name: this.name, email: this.email });
  }

  deserialize(data: string): void {
    const parsed = JSON.parse(data);
    this.name = parsed.name;
    this.email = parsed.email;
  }

  validate(): boolean {
    return this.name.length > 0 && this.email.includes("@");
  }

  getErrors(): string[] {
    const errors: string[] = [];
    if (!this.name) errors.push("Name is required");
    if (!this.email.includes("@")) errors.push("Invalid email");
    return errors;
  }
}
```

## 6. 静态成员与单例模式

```typescript
// 单例模式
class AppConfig {
  private static instance: AppConfig;
  private settings = new Map<string, string>();

  private constructor() {} // 私有构造函数防止外部 new

  static getInstance(): AppConfig {
    if (!AppConfig.instance) {
      AppConfig.instance = new AppConfig();
    }
    return AppConfig.instance;
  }

  set(key: string, value: string) { this.settings.set(key, value); }
  get(key: string) { return this.settings.get(key); }
}

const config = AppConfig.getInstance();
config.set("apiUrl", "https://api.example.com");
```

## 7. 装饰器 (Decorators)

装饰器是一种特殊的声明，可以附加到类、方法、属性上，用于修改行为。

```typescript
// 方法装饰器：日志记录
function Log(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
  const original = descriptor.value;
  descriptor.value = function (...args: any[]) {
    console.log(`Calling ${propertyKey} with args:`, args);
    const result = original.apply(this, args);
    console.log(`${propertyKey} returned:`, result);
    return result;
  };
}

// 类装饰器：自动注册
function Controller(path: string) {
  return function (target: Function) {
    Reflect.defineMetadata("path", path, target);
  };
}

// NestJS 风格的使用
@Controller("/users")
class UserController {
  @Log
  findAll(): string[] {
    return ["Alice", "Bob"];
  }
}
```

## 8. Mixin 模式

```typescript
// TypeScript 不支持多继承，但可以用 Mixin 实现类似效果
type Constructor<T = {}> = new (...args: any[]) => T;

function Timestamped<TBase extends Constructor>(Base: TBase) {
  return class extends Base {
    createdAt = new Date();
    updatedAt = new Date();
  };
}

function Activatable<TBase extends Constructor>(Base: TBase) {
  return class extends Base {
    isActive = false;
    activate() { this.isActive = true; }
    deactivate() { this.isActive = false; }
  };
}

class BaseUser {
  constructor(public name: string) {}
}

// 组合多个 Mixin
const TimestampedActivatableUser = Timestamped(Activatable(BaseUser));
const user = new TimestampedActivatableUser("Alice");
user.activate();
console.log(user.isActive);   // true
console.log(user.createdAt);  // Date
```

## 9. 实际工程案例：依赖注入

```typescript
// 简化版 IoC 容器
class Container {
  private services = new Map<string, any>();

  register<T>(token: string, factory: () => T): void {
    this.services.set(token, factory);
  }

  resolve<T>(token: string): T {
    const factory = this.services.get(token);
    if (!factory) throw new Error(`Service ${token} not registered`);
    return factory();
  }
}

// 使用
interface IUserRepository {
  findById(id: string): Promise<User | null>;
}

class PostgresUserRepository implements IUserRepository {
  async findById(id: string) { /* 查询数据库 */ return null; }
}

const container = new Container();
container.register<IUserRepository>("UserRepo", () => new PostgresUserRepository());

const repo = container.resolve<IUserRepository>("UserRepo");
```
