# 第六章：高级类型

TypeScript 的类型系统远不止基础类型注解。高级类型特性让我们能够构建精确、灵活且类型安全的抽象，从而在编译期捕获更多错误。本章将深入探讨这些强大的类型工具。

---

## 1. 类型收窄 (Type Narrowing)

类型收窄是指 TypeScript 通过代码中的条件判断，将变量的类型从宽泛类型缩小为更具体类型的过程。这是 TypeScript 控制流分析的核心能力。

### 1.1 typeof 守卫

`typeof` 操作符是最常用的类型收窄手段，适用于原始类型的判断。

```typescript
function formatValue(value: string | number | boolean): string {
  if (typeof value === "string") {
    // 此处 value 被收窄为 string
    return value.toUpperCase();
  }
  if (typeof value === "number") {
    // 此处 value 被收窄为 number
    return value.toFixed(2);
  }
  // 此处 value 被收窄为 boolean
  return value ? "是" : "否";
}
```

**设计原因：** TypeScript 利用 JavaScript 运行时已有的 `typeof` 检查来推断类型，无需额外的运行时开销。

**注意事项：** `typeof null` 返回 `"object"`，这是 JavaScript 的历史遗留问题：

```typescript
function process(value: string | null) {
  if (typeof value === "object") {
    // 注意：这里 value 的类型是 null，不是 object
    // 因为 string 被排除后只剩 null
  }
}
```

### 1.2 instanceof 守卫

`instanceof` 用于判断对象是否为某个类的实例，适用于类层次结构的类型收窄。

```typescript
class HttpError extends Error {
  statusCode: number;
  constructor(message: string, statusCode: number) {
    super(message);
    this.statusCode = statusCode;
  }
}

class ValidationError extends Error {
  fields: string[];
  constructor(message: string, fields: string[]) {
    super(message);
    this.fields = fields;
  }
}

function handleError(error: HttpError | ValidationError) {
  if (error instanceof HttpError) {
    // error 被收窄为 HttpError
    console.log(`HTTP ${error.statusCode}: ${error.message}`);
  } else {
    // error 被收窄为 ValidationError
    console.log(`验证失败的字段: ${error.fields.join(", ")}`);
  }
}
```

**工程场景：** 在错误处理中间件中，根据不同的错误类型返回不同的 HTTP 响应。

### 1.3 in 操作符

`in` 操作符检查对象是否包含某个属性，适用于区分具有不同属性的对象类型。

```typescript
interface Fish {
  swim(): void;
  habitat: string;
}

interface Bird {
  fly(): void;
  wingspan: number;
}

function move(animal: Fish | Bird) {
  if ("swim" in animal) {
    // animal 被收窄为 Fish
    animal.swim();
    console.log(`栖息地: ${animal.habitat}`);
  } else {
    // animal 被收窄为 Bird
    animal.fly();
    console.log(`翼展: ${animal.wingspan}cm`);
  }
}
```

**设计原因：** 当无法使用 `instanceof`（例如接口没有运行时表示）时，`in` 操作符提供了基于结构的类型收窄方式。

### 1.4 自定义类型守卫（is 关键字）

当内置的类型收窄不够用时，可以定义自定义类型守卫函数。

```typescript
interface Cat {
  meow(): void;
  purr(): void;
}

interface Dog {
  bark(): void;
  fetch(): void;
}

// 返回类型 `pet is Cat` 是类型谓词
function isCat(pet: Cat | Dog): pet is Cat {
  return (pet as Cat).meow !== undefined;
}

function handlePet(pet: Cat | Dog) {
  if (isCat(pet)) {
    pet.meow(); // pet 被收窄为 Cat
    pet.purr();
  } else {
    pet.bark(); // pet 被收窄为 Dog
    pet.fetch();
  }
}
```

**高级用法 — 数组过滤中的类型守卫：**

```typescript
function isNonNull<T>(value: T | null | undefined): value is T {
  return value !== null && value !== undefined;
}

const mixedArray: (string | null | undefined)[] = ["hello", null, "world", undefined];
const cleanArray: string[] = mixedArray.filter(isNonNull);
// cleanArray 的类型是 string[]，而非 (string | null | undefined)[]
```

### 1.5 真值收窄

JavaScript 中的假值（`false`, `0`, `""`, `null`, `undefined`, `NaN`）可以用于类型收窄。

```typescript
function printName(name: string | null | undefined) {
  if (name) {
    // name 被收窄为 string（排除了 null、undefined 和空字符串）
    console.log(name.toUpperCase());
  }
}

// 使用 !! 进行布尔转换也能触发收窄
function getLength(arr: number[] | null): number {
  return arr?.length ?? 0;
}
```

**注意：** 真值收窄会同时排除空字符串 `""` 和数字 `0`，在某些场景下可能不是期望的行为。

### 1.6 相等性收窄

使用 `===`、`!==`、`==`、`!=` 进行比较时，TypeScript 会收窄类型。

```typescript
function compare(x: string | number, y: string | boolean) {
  if (x === y) {
    // x 和 y 都被收窄为 string（唯一可能相等的类型）
    console.log(x.toUpperCase());
    console.log(y.toUpperCase());
  }
}

// 利用与 null 的比较
function process(value: string | null | undefined) {
  if (value != null) {
    // 使用 != 同时排除 null 和 undefined
    // value 被收窄为 string
    console.log(value.trim());
  }
}
```

### 1.7 控制流分析

TypeScript 的控制流分析能够追踪变量在不同代码路径中的类型变化。

```typescript
function example(x: string | number | boolean) {
  // x: string | number | boolean

  if (typeof x === "string") {
    return x.toUpperCase(); // x: string，函数提前返回
  }

  // x: number | boolean（string 已被排除）

  if (typeof x === "number") {
    return x.toFixed(2); // x: number
  }

  // x: boolean（string 和 number 都已被排除）
  return x ? "真" : "假";
}
```

**穷尽性检查（Exhaustiveness Checking）：**

```typescript
type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "rectangle"; width: number; height: number }
  | { kind: "triangle"; base: number; height: number };

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "rectangle":
      return shape.width * shape.height;
    case "triangle":
      return (shape.base * shape.height) / 2;
    default:
      // 如果所有情况都已处理，shape 的类型为 never
      const _exhaustive: never = shape;
      return _exhaustive;
  }
}
```

**设计原因：** 穷尽性检查确保当联合类型新增成员时，所有 switch 语句都会产生编译错误，防止遗漏处理。

## 2. 映射类型 (Mapped Types)

映射类型允许你基于已有类型创建新类型，通过遍历键来转换每个属性。

### 2.1 基本语法

```typescript
// 基本映射类型语法：[K in keyof T]: 转换规则
type Readonly<T> = {
  readonly [K in keyof T]: T[K];
};

type Optional<T> = {
  [K in keyof T]?: T[K];
};

interface User {
  id: number;
  name: string;
  email: string;
}

// ReadonlyUser 的每个属性都变为只读
type ReadonlyUser = Readonly<User>;
// 等价于：
// {
//   readonly id: number;
//   readonly name: string;
//   readonly email: string;
// }
```

**工程场景：** 在 Redux/状态管理中，确保 state 对象不可变。

### 2.2 修饰符 readonly 和 ?

映射类型可以添加或移除 `readonly` 和 `?` 修饰符。

```typescript
// 移除 readonly（使用 -readonly）
type Mutable<T> = {
  -readonly [K in keyof T]: T[K];
};

// 移除可选性（使用 -?）
type Required<T> = {
  [K in keyof T]-?: T[K];
};

// 实际示例
interface Config {
  readonly host: string;
  readonly port: number;
  debug?: boolean;
}

type MutableConfig = Mutable<Config>;
// { host: string; port: number; debug?: boolean }

type StrictConfig = Required<Config>;
// { readonly host: string; readonly port: number; debug: boolean }
```

### 2.3 键重映射 (as)

TypeScript 4.1 引入了键重映射，允许在映射过程中转换键名。

```typescript
// 为所有属性添加 get 前缀
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

interface Person {
  name: string;
  age: number;
}

type PersonGetters = Getters<Person>;
// {
//   getName: () => string;
//   getAge: () => number;
// }

// 过滤特定类型的属性
type OnlyStringProperties<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K];
};

interface Mixed {
  id: number;
  name: string;
  email: string;
  active: boolean;
}

type StringProps = OnlyStringProperties<Mixed>;
// { name: string; email: string }
```

**设计原因：** 键重映射让映射类型具备了过滤和转换键名的能力，大幅扩展了类型编程的表达力。

### 2.4 实现自定义工具类型

```typescript
// 深度只读
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? T[K] extends Function
      ? T[K]
      : DeepReadonly<T[K]>
    : T[K];
};

// 选取部分属性并使其可选
type PartialPick<T, K extends keyof T> = Omit<T, K> & Partial<Pick<T, K>>;

interface Article {
  id: number;
  title: string;
  content: string;
  author: string;
  createdAt: Date;
}

// 创建文章时，id 和 createdAt 是可选的（由系统生成）
type CreateArticleDTO = PartialPick<Article, "id" | "createdAt">;

// 将对象类型转为联合类型
type ObjectToUnion<T> = {
  [K in keyof T]: { key: K; value: T[K] };
}[keyof T];

type UserUnion = ObjectToUnion<User>;
// { key: "id"; value: number } | { key: "name"; value: string } | { key: "email"; value: string }
```

## 3. 条件类型 (Conditional Types)

条件类型根据类型关系选择不同的类型分支，类似于类型层面的三元表达式。

### 3.1 基本语法

```typescript
// T extends U ? X : Y
// 如果 T 可以赋值给 U，则结果为 X，否则为 Y

type IsString<T> = T extends string ? true : false;

type A = IsString<string>;  // true
type B = IsString<number>;  // false
type C = IsString<"hello">; // true（字面量类型是 string 的子类型）
```

### 3.2 分布式条件类型

当条件类型作用于联合类型时，会自动分发到每个成员：

```typescript
type ToArray<T> = T extends any ? T[] : never;

// 联合类型会被分发
type Result = ToArray<string | number>;
// 等价于 ToArray<string> | ToArray<number>
// 即 string[] | number[]

// 如果不想分发，用方括号包裹
type ToArrayNoDistribute<T> = [T] extends [any] ? T[] : never;
type Result2 = ToArrayNoDistribute<string | number>;
// (string | number)[]  — 不分发
```

### 3.3 infer 关键字

`infer` 在条件类型中声明一个待推断的类型变量：

```typescript
// 提取函数返回类型
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type Fn = (x: number) => string;
type R = MyReturnType<Fn>; // string

// 提取 Promise 内部类型
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T;
type P = UnwrapPromise<Promise<number>>; // number

// 提取数组元素类型
type ElementOf<T> = T extends (infer E)[] ? E : never;
type E = ElementOf<string[]>; // string
```

## 4. 模板字面量类型 (Template Literal Types)

```typescript
// 基本语法：在类型层面做字符串拼接
type Greeting = `Hello, ${string}`;
let g: Greeting = "Hello, World"; // OK
// let g2: Greeting = "Hi, World"; // Error

// 与联合类型结合 — 生成所有组合
type Color = "red" | "blue" | "green";
type Size = "sm" | "md" | "lg";
type ClassName = `${Color}-${Size}`;
// "red-sm" | "red-md" | "red-lg" | "blue-sm" | ... 共 9 种

// 实际应用：CSS 属性类型
type CSSProperty = `${string}-${string}`;
type EventName = `on${Capitalize<string>}`;

// 内置字符串操作类型
type Upper = Uppercase<"hello">;     // "HELLO"
type Lower = Lowercase<"HELLO">;     // "hello"
type Cap = Capitalize<"hello">;      // "Hello"
type Uncap = Uncapitalize<"Hello">;  // "hello"

// 实际工程：从对象类型生成事件名
interface User {
  name: string;
  age: number;
  email: string;
}

type UserChangeEvent = `${keyof User}Changed`;
// "nameChanged" | "ageChanged" | "emailChanged"
```

## 5. 索引访问类型

```typescript
// T[K] 获取类型中某个属性的类型
interface User {
  id: number;
  name: string;
  address: {
    city: string;
    zip: string;
  };
}

type UserId = User["id"];           // number
type UserAddress = User["address"]; // { city: string; zip: string }
type City = User["address"]["city"]; // string

// 用联合类型获取多个属性
type IdOrName = User["id" | "name"]; // number | string

// 获取数组元素类型
const fruits = ["apple", "banana", "cherry"] as const;
type Fruit = typeof fruits[number]; // "apple" | "banana" | "cherry"
```

## 6. keyof 和 typeof

```typescript
// keyof：获取对象类型的所有键的联合类型
interface Config {
  host: string;
  port: number;
  debug: boolean;
}

type ConfigKey = keyof Config; // "host" | "port" | "debug"

// typeof：获取值的类型（类型层面的 typeof）
const config = {
  host: "localhost",
  port: 3000,
  debug: true,
} as const;

type ConfigType = typeof config;
// { readonly host: "localhost"; readonly port: 3000; readonly debug: true }

// 经典组合：从对象值提取类型
const ROUTES = {
  home: "/",
  about: "/about",
  users: "/users/:id",
} as const;

type RouteKey = keyof typeof ROUTES;     // "home" | "about" | "users"
type RoutePath = typeof ROUTES[RouteKey]; // "/" | "/about" | "/users/:id"

// 类型安全的属性访问函数
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { name: "Alice", age: 30 };
const name = getProperty(user, "name"); // string
const age = getProperty(user, "age");   // number
// getProperty(user, "email");          // Error: "email" 不在 keyof User 中
```

## 7. 实际工程案例

### 类型安全的事件系统

```typescript
type EventMap = {
  click: { x: number; y: number };
  keydown: { key: string; code: number };
  resize: { width: number; height: number };
};

class TypedEmitter<T extends Record<string, any>> {
  private handlers = new Map<keyof T, Set<Function>>();

  on<K extends keyof T>(event: K, handler: (payload: T[K]) => void) {
    if (!this.handlers.has(event)) this.handlers.set(event, new Set());
    this.handlers.get(event)!.add(handler);
    return this;
  }

  emit<K extends keyof T>(event: K, payload: T[K]) {
    this.handlers.get(event)?.forEach(h => h(payload));
  }
}

const emitter = new TypedEmitter<EventMap>();
emitter.on("click", (e) => console.log(e.x, e.y)); // e 自动推断
emitter.emit("click", { x: 10, y: 20 });           // payload 类型检查
```

### 深度 Partial 实现

```typescript
type DeepPartial<T> = T extends object
  ? { [K in keyof T]?: DeepPartial<T[K]> }
  : T;

interface NestedConfig {
  server: { host: string; port: number };
  database: { url: string; pool: { min: number; max: number } };
}

// 所有层级的属性都变为可选
type PartialConfig = DeepPartial<NestedConfig>;
const partial: PartialConfig = { server: { host: "localhost" } }; // OK
```
