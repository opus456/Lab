# TypeScript 函数

函数是 TypeScript 中的一等公民。TypeScript 在 JavaScript 函数的基础上增加了类型系统的支持，使得函数的输入输出更加明确、可预测，从而大幅减少运行时错误。

---

## 1. 函数类型定义

### 1.1 函数声明的类型

```typescript
// 函数声明 - 参数和返回值都有类型标注
function add(x: number, y: number): number {
  return x + y;
}

// TypeScript 会检查参数数量和类型
add(1, 2);       // OK
add(1, "2");     // Error: Argument of type 'string' is not assignable to parameter of type 'number'
add(1, 2, 3);    // Error: Expected 2 arguments, but got 3
```

### 1.2 函数表达式的类型

```typescript
// 方式一：只标注右侧（最常用）
const multiply = function(x: number, y: number): number {
  return x * y;
};

// 方式二：标注变量类型（完整写法）
const divide: (x: number, y: number) => number = function(x, y) {
  return x / y;
};

// 注意：这里的 => 不是箭头函数，而是 TypeScript 函数类型的返回值标记
```

### 1.3 箭头函数类型

```typescript
// 箭头函数 - 类型推断
const square = (x: number): number => x * x;

// 箭头函数 - 完整类型标注
const power: (base: number, exp: number) => number = (base, exp) => {
  return Math.pow(base, exp);
};
```

### 1.4 完整的函数类型写法

```typescript
// 使用 type 别名定义函数类型
type MathOperation = (a: number, b: number) => number;

const add: MathOperation = (a, b) => a + b;
const sub: MathOperation = (a, b) => a - b;

// 使用 interface 定义函数类型（调用签名）
interface SearchFunc {
  (source: string, subString: string): boolean;
}
const mySearch: SearchFunc = (src, sub) => {
  return src.indexOf(sub) > -1;
};

// 设计原因：函数类型别名让我们可以复用类型定义，
// 当多个函数具有相同签名时，避免重复书写类型标注
```

## 2. 参数类型

### 2.1 必选参数

```typescript
// TypeScript 中函数参数默认都是必选的
function buildName(firstName: string, lastName: string): string {
  return `${firstName} ${lastName}`;
}

buildName("Bob");              // Error: Expected 2 arguments, but got 1
buildName("Bob", "Adams");     // OK
```

### 2.2 可选参数

```typescript
// 使用 ? 标记可选参数
function buildName(firstName: string, lastName?: string): string {
  if (lastName) {
    return `${firstName} ${lastName}`;
  }
  return firstName;
}

buildName("Bob");              // OK
buildName("Bob", "Adams");     // OK
```

**为什么可选参数必须放在必选参数后面？**

```typescript
// 错误示例 - 可选参数不能在必选参数前面
function badFunc(x?: number, y: number): number {  // Error!
  return (x ?? 0) + y;
}

// 原因：JavaScript 函数调用是按位置传参的
// 如果可选参数在前面，调用 badFunc(5) 时无法确定 5 是给 x 还是 y
// TypeScript 遵循这一设计以保持与 JavaScript 的一致性
```
### 2.3 默认参数

```typescript
// 默认参数 - 有默认值的参数自动变为可选
function createUser(name: string, role: string = "user"): object {
  return { name, role };
}

createUser("Alice");            // { name: "Alice", role: "user" }
createUser("Bob", "admin");     // { name: "Bob", role: "admin" }

// 默认参数可以放在必选参数前面（但调用时需要传 undefined 来跳过）
function buildUrl(protocol: string = "https", host: string): string {
  return `${protocol}://${host}`;
}

buildUrl(undefined, "example.com");  // "https://example.com"
```

### 2.4 剩余参数 (Rest Parameters)

```typescript
// 使用 ... 收集剩余参数
function sum(first: number, ...rest: number[]): number {
  return rest.reduce((acc, val) => acc + val, first);
}

sum(1, 2, 3, 4);  // 10

// 元组类型的剩余参数 - 限制参数数量和类型
function createTuple(...args: [string, number, boolean]): [string, number, boolean] {
  return args;
}

createTuple("hello", 42, true);  // OK
createTuple("hello", 42);       // Error: Expected 3 arguments
```

### 2.5 参数解构的类型标注

```typescript
// 对象解构参数
function printUser({ name, age }: { name: string; age: number }): void {
  console.log(`${name} is ${age} years old`);
}

// 使用类型别名让解构更清晰
type UserConfig = {
  host: string;
  port: number;
  ssl?: boolean;
};

function connectDB({ host, port, ssl = false }: UserConfig): void {
  console.log(`Connecting to ${host}:${port}, SSL: ${ssl}`);
}

// 数组解构参数
function head([first, ...rest]: number[]): number | undefined {
  return first;
}

// 实际工程场景：React 组件 props 解构
// function Button({ label, onClick, disabled = false }: ButtonProps) { ... }
```

## 3. 返回值类型

### 3.1 显式标注 vs 类型推断

```typescript
// 显式标注返回值类型
function getAge(): number {
  return 25;
}

// 类型推断 - TypeScript 自动推断返回值类型
function getName() {
  return "Alice";  // 推断为 string
}

// 何时应该显式标注？
// 1. 公共 API / 库函数 - 明确契约，防止意外修改
// 2. 复杂函数 - 帮助开发者理解意图
// 3. 递归函数 - 有时 TypeScript 无法推断递归函数的返回类型

// 何时可以省略？
// 1. 简单的内部函数
// 2. 箭头函数回调
// 3. 返回值显而易见的情况
```

### 3.2 void vs undefined 的区别

```typescript
// void - 表示函数不返回有意义的值
function logMessage(msg: string): void {
  console.log(msg);
  // 可以 return; 或不写 return
}

// undefined - 表示函数明确返回 undefined
function returnUndefined(): undefined {
  return undefined;  // 必须显式 return undefined
}

// 关键区别：void 类型的回调允许返回任何值（会被忽略）
type VoidCallback = () => void;

const cb: VoidCallback = () => 42; // OK! 返回值被忽略
// 这个设计是为了兼容 Array.forEach 等场景
// [1,2,3].forEach(x => x * 2) 中回调有返回值但被忽略
```

### never 返回类型

```typescript
function fail(message: string): never {
  throw new Error(message);
}

// 实际用途：穷尽检查辅助函数
function assertNever(value: never): never {
  throw new Error(`Unexpected value: ${value}`);
}
```

## 4. 函数重载 (Overloads)

当函数根据不同参数类型返回不同类型时，联合类型无法精确表达这种对应关系。

```typescript
// 重载签名
function parse(input: string): number;
function parse(input: number): string;
// 实现签名
function parse(input: string | number): string | number {
  if (typeof input === "string") return parseInt(input);
  return input.toString();
}

const num = parse("42");  // 精确推断为 number
const str = parse(42);    // 精确推断为 string

// 实际工程：DOM createElement
function createElement(tag: "div"): HTMLDivElement;
function createElement(tag: "input"): HTMLInputElement;
function createElement(tag: string): HTMLElement;
function createElement(tag: string): HTMLElement {
  return document.createElement(tag);
}
```

**重载 vs 联合类型的选择**：输入输出有对应关系用重载，否则用联合类型。

## 5. this 类型

```typescript
// 链式调用中的 this 类型
class QueryBuilder {
  private conditions: string[] = [];

  where(condition: string): this {
    this.conditions.push(condition);
    return this;
  }

  orderBy(field: string): this {
    return this;
  }
}

class MySQLBuilder extends QueryBuilder {
  limit(n: number): this {
    return this;
  }
}

// 子类链式调用类型正确
new MySQLBuilder().where("age > 18").limit(10).orderBy("name");
```

## 6. 回调函数与高阶函数

```typescript
// 高阶函数类型
type Predicate<T> = (item: T) => boolean;
type Mapper<T, U> = (item: T) => U;

// 柯里化函数的类型
function curry<A, B, C>(fn: (a: A, b: B) => C): (a: A) => (b: B) => C {
  return (a) => (b) => fn(a, b);
}

const add = (a: number, b: number) => a + b;
const add5 = curry(add)(5); // (b: number) => number
```

## 7. 实际工程案例

### 防抖函数的完整类型

```typescript
function debounce<T extends (...args: any[]) => any>(
  fn: T,
  delay: number
): (...args: Parameters<T>) => void {
  let timer: ReturnType<typeof setTimeout>;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

function search(query: string, page: number) { /* ... */ }
const debouncedSearch = debounce(search, 300);
debouncedSearch("hello", 1); // 参数类型正确
debouncedSearch(42);          // Error!
```

### Express 中间件类型

```typescript
import { Request, Response, NextFunction } from "express";

type Middleware = (req: Request, res: Response, next: NextFunction) => void;

function asyncHandler(
  fn: (req: Request, res: Response, next: NextFunction) => Promise<void>
): Middleware {
  return (req, res, next) => fn(req, res, next).catch(next);
}
```
