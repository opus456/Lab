
 # 任务
  请在指定根目录下,为「<技术名称>」生成一套完整的、工程导向的中文学习文档。
  若该技术对应文件夹已存在,则在现有基础上补全和完善,不要推翻重写。

  # 目录与文件组织
  - 根目录下以技术名为名建立文件夹(已存在则沿用)
  - 文件夹内组织:
    ├── README.md                    总目录、学习路线、难度图例、前置知识
    ├── 00_xxx.md ~ NN_xxx.md        分章节核心知识体系(序号+主题)
    ├── 扩展专题/  或  NN_扩展库与框架.md   工业界主流生态库/框架专题
    ├── NN_综合实战项目.md           压轴的真实可用项目
    └── NN_开源项目推荐.md           单独一章的优质开源项目精读清单

  # 每章节内容必须满足
  1. 开头标注难度:⭐ 重点 / ⭐⭐ 核心难点 / 入门 / 中等 / 高级(不常用) / 实战
  2. 知识点讲解三件套(每个知识点都要):
     (a) 这是什么、为什么存在
     (b) 在真实工程/项目中的具体用法 + 可运行代码示例
     (c) **为什么这样设计、这样设计带来的好处**(强制项,不能省)
  3. 风格:深入浅出,先讲动机和直觉,再上代码;不要堆玩具示例,贴近主流工程实践
  4. 对每个知识点显式标注:核心高频 / 重要 / 了解即可 / 不常用
  5. 用对比表、内存示意图、调用流程图等可视化方式辅助理解
  6. 章末必须有「动手练习」:
     - 3–5 道题,难度递增
     - 必须是工程/项目中真实会遇到的问题,不要纯算法题
     - 综合本章所学知识点
     - **只给详细解题思路、不给最终答案代码**

  # 基础知识学完后:工程扩展专题
  - 选取该技术在工业界最主流的扩展库/框架(例如 Rust 的 tokio/serde/sqlx/tracing/clap)
  - 每个专题专门成章
  - 重点讲:在真实项目里它解决什么问题、怎么集成、典型使用模式、踩坑点
  - 不做 API 文档搬运,讲设计哲学和工程取舍

  # 综合实战项目(压轴章)
  - 必须是一个**有一定复杂度、贴近工程实践、做完后用户日常真的能用上**的完整工具
  - 默认偏好:CLI 工具 / 实用桌面或终端工具(除非用户另外指定)
  - 工程化要求完整:目录结构、模块划分、错误处理、日志、测试、文档、可选 CI
  - 文档形式:讲清楚需求分析 → 技术选型理由 → 分阶段实现指引 → 扩展方向
  - 目标:用户做完后有成就感,产物可以集成进日常使用

  # 开源项目推荐(单独一章)
  - 单开一章,不与实战项目混编
  - 推荐 5–10 个 GitHub 上真正高质量、可读源码的开源项目
  - 每个项目说明:它是什么、用了哪些核心特性、值得学什么、阅读源码切入点

  # 工作流
  1. 先对照需求核对 README,补全/修正章节目录与笔误
  2. 按顺序补齐缺失章节,每章独立成文
  3. 完成基础与扩展章后再写综合实战项目
  4. 最后写开源项目推荐
  5. 全程使用中文;代码块语言标注准确;链接使用相对路
