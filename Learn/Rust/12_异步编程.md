# 12 异步编程

> **难度：⭐⭐ 核心难点** | Rust 异步编程用于高并发 I/O 场景，例如 Web 服务、爬虫、消息消费者和网络代理。

---

## 1. async/await 是什么

**标注：核心高频**

`async fn` 返回一个 `Future`，只有被运行时轮询时才会真正执行。

```rust
async fn fetch_user(id: u64) -> String {
    format!("user-{}", id)
}

#[tokio::main]
async fn main() {
    let name = fetch_user(1).await;
    println!("{}", name);
}
```

**为什么这样设计、这样设计带来的好处：**

异步不是创建大量线程，而是在少量线程上切换等待 I/O 的任务。这样能支撑大量连接，适合网络密集型服务。

---

## 2. Future 与惰性执行

**标注：重要**

Future 是一个尚未完成的计算。创建 Future 不等于执行它。

```rust
async fn expensive() -> u64 {
    println!("run");
    42
}

#[tokio::main]
async fn main() {
    let fut = expensive();
    println!("created");
    let value = fut.await;
    println!("{}", value);
}
```

执行流程：

```text
创建 Future -> 注册到运行时 -> await 时被轮询 -> 遇到 I/O 让出 -> 就绪后继续
```

**为什么这样设计、这样设计带来的好处：**

惰性 Future 让调用者决定何时、在哪个运行时、以什么组合方式执行任务。

## 3. Tokio：事实标准运行时

**标注：核心高频**

Tokio 提供异步运行时、定时器、TCP、任务调度、同步原语。

```toml
[dependencies]
tokio = { version = "1", features = ["full"] }
```

```rust
use tokio::time::{sleep, Duration};

#[tokio::main]
async fn main() {
    let handle = tokio::spawn(async {
        sleep(Duration::from_millis(100)).await;
        "done"
    });

    let result = handle.await.unwrap();
    println!("{}", result);
}
```

**为什么这样设计、这样设计带来的好处：**

标准库不绑定具体运行时，生态可以按场景演进。Tokio 成为事实标准，是因为它覆盖了生产级网络服务需要的大部分基础设施。

## 4. 并发组合：`join!` 与 `select!`

**标注：重要**

`join!` 等待多个任务全部完成，`select!` 等待多个分支中最先完成的一个。

```rust
use tokio::time::{sleep, Duration};

async fn load_config() -> &'static str {
    sleep(Duration::from_millis(50)).await;
    "config"
}

async fn load_user() -> &'static str {
    sleep(Duration::from_millis(30)).await;
    "user"
}

#[tokio::main]
async fn main() {
    let (config, user) = tokio::join!(load_config(), load_user());
    println!("{} {}", config, user);
}
```

**为什么这样设计、这样设计带来的好处：**

工程中经常要并发请求多个下游服务。组合 Future 可以减少总等待时间，同时保持代码结构清晰。

## 5. 异步中的常见坑

**标注：核心高频**

| 问题 | 后果 | 建议 |
|------|------|------|
| 在 async 中执行阻塞 I/O | 卡住运行时线程 | 使用异步 API 或 `spawn_blocking` |
| 持有锁跨 `.await` | 容易死锁或性能下降 | await 前释放锁 |
| 过度 `tokio::spawn` | 任务难追踪 | 优先结构化并发 |

```rust
use tokio::task;

#[tokio::main]
async fn main() {
    let result = task::spawn_blocking(|| {
        std::fs::read_to_string("Cargo.toml")
    }).await.unwrap();

    println!("{:?}", result.ok());
}
```

**为什么这样设计、这样设计带来的好处：**

异步运行时假设任务会快速让出执行权。阻塞操作必须隔离到专门线程池，才能保护整体吞吐。
