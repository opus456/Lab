# Rust 命令行工具开发

> **难度标记：实战** | 需要前置知识：错误处理、trait、模块系统、文件 I/O

## 为什么用 Rust 写 CLI 工具？

- **零依赖分发**：编译为单个静态二进制文件，无需安装运行时
- **启动速度**：毫秒级启动，不像 Python/Node 需要解释器加载
- **跨平台**：一套代码编译到 Linux/macOS/Windows
- **性能**：处理大文件、大量 I/O 时优势明显

真实案例：ripgrep（比 grep 快 10x）、bat、fd、exa、starship 都是 Rust CLI 工具。

## 1. clap —— 参数解析

clap 是 Rust 生态中最流行的命令行参数解析库，推荐使用 **derive API**。

### 1.1 基础用法

```rust
use clap::Parser;

/// 一个简单的文件搜索工具
#[derive(Parser, Debug)]
#[command(name = "mini-grep")]
#[command(version = "1.0")]
#[command(about = "在文件中搜索模式", long_about = None)]
struct Args {
    /// 要搜索的模式
    pattern: String,

    /// 要搜索的文件路径
    path: String,

    /// 忽略大小写
    #[arg(short, long)]
    ignore_case: bool,

    /// 显示行号
    #[arg(short = 'n', long)]
    line_number: bool,

    /// 限制输出行数
    #[arg(short, long, default_value_t = 0)]
    max_count: usize,
}

fn main() {
    let args = Args::parse();
    println!("搜索 '{}' in {}", args.pattern, args.path);
}
```

运行效果：
```bash
$ mini-grep --help
在文件中搜索模式

Usage: mini-grep [OPTIONS] <PATTERN> <PATH>

Arguments:
  <PATTERN>  要搜索的模式
  <PATH>     要搜索的文件路径

Options:
  -i, --ignore-case    忽略大小写
  -n, --line-number    显示行号
  -m, --max-count <N>  限制输出行数 [default: 0]
  -h, --help           Print help
  -V, --version        Print version
```
