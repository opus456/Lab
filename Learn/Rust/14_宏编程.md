## 1. 宏的概述与分类

宏是在编译时展开的代码生成工具。与函数不同，宏操作的是 **语法树（AST）**，而非运行时的值。

### Rust 中的两大类宏

| 类型 | 语法 | 能力 |
|------|------|------|
| 声明式宏 | `macro_rules! name { ... }` | 模式匹配 + 代码替换 |
| 过程宏 | `#[proc_macro]` 等 | 任意代码生成，操作 TokenStream |

### 为什么这样设计？

- **减少样板代码**：Rust 没有继承和反射，宏填补了代码复用的空白
- **编译时执行**：零运行时开销，错误在编译期暴露
- **类型安全**：展开后的代码仍然经过完整的类型检查
- **与 C/C++ 宏的区别**：Rust 宏是"卫生的"（hygienic），不会意外捕获变量

```rust
// C 宏的问题（Rust 不会有）
// #define SQUARE(x) x * x
// SQUARE(1+2) => 1+2*1+2 = 5  (期望 9)

// Rust 宏是安全的
macro_rules! square {
    ($x:expr) => { $x * $x };  // $x 被正确地作为表达式处理
}

fn main() {
    assert_eq!(square!(1 + 2), 9);  // 正确！
}
```

## 2. 声明式宏 macro_rules!

### 2.1 基本语法

```rust
macro_rules! say_hello {
    () => {
        println!("Hello!");
    };
}

fn main() {
    say_hello!();  // 展开为 println!("Hello!");
}
```

### 2.2 模式匹配与片段类型

宏参数使用 `$name:fragment_type` 语法，常用片段类型：

| 片段类型 | 匹配内容 | 示例 |
|----------|----------|------|
| `expr` | 表达式 | `1 + 2`, `foo()` |
| `ty` | 类型 | `i32`, `Vec<String>` |
| `ident` | 标识符 | `foo`, `MyStruct` |
| `pat` | 模式 | `Some(x)`, `_` |
| `stmt` | 语句 | `let x = 1` |
| `block` | 代码块 | `{ ... }` |
| `item` | 项（函数、结构体等） | `fn foo() {}` |
| `tt` | 单个 token tree | 任何 token |

```rust
macro_rules! create_function {
    ($name:ident, $body:expr) => {
        fn $name() -> i32 {
            $body
        }
    };
}

create_function!(answer, 42);
create_function!(double_answer, 42 * 2);

fn main() {
    println!("{}", answer());        // 42
    println!("{}", double_answer()); // 84
}
```

### 2.3 重复（Repetition）

重复是声明式宏最强大的特性，语法为 `$( ... )sep*` 或 `$( ... )sep+`：

```rust
/// 类似 vec! 的宏实现
macro_rules! my_vec {
    // 空向量
    () => { Vec::new() };
    // 带初始元素：my_vec![1, 2, 3]
    ( $( $elem:expr ),+ $(,)? ) => {
        {
            let mut v = Vec::new();
            $( v.push($elem); )+
            v
        }
    };
    // 重复元素：my_vec![0; 5]
    ( $elem:expr ; $count:expr ) => {
        vec![$elem; $count]
    };
}
