# 11 并发编程

> **难度：⭐⭐ 核心难点** | Rust 的并发模型把数据竞争变成编译错误，是高可靠后端、CLI 批处理和基础设施工具的重要能力。

---

## 1. 线程与所有权转移

**标注：核心高频**

Rust 使用 `std::thread::spawn` 创建系统线程。闭包通常使用 `move` 把需要的数据移动进线程。

```rust
use std::thread;

fn main() {
    let files = vec!["a.log".to_string(), "b.log".to_string()];

    let handle = thread::spawn(move || {
        for file in files {
            println!("process {}", file);
        }
    });

    handle.join().unwrap();
}
```

**为什么这样设计、这样设计带来的好处：**

线程可能比创建它的函数活得更久。`move` 明确转移所有权，避免线程引用已经释放的栈变量。

## 2. 消息传递：用 channel 解耦任务

**标注：核心高频**

channel 适合“多个工作线程产生结果，主线程统一汇总”的模式。

```rust
use std::sync::mpsc;
use std::thread;

fn main() {
    let (tx, rx) = mpsc::channel();

    for id in 0..3 {
        let tx = tx.clone();
        thread::spawn(move || {
            tx.send(format!("job {} done", id)).unwrap();
        });
    }

    drop(tx);

    for msg in rx {
        println!("{}", msg);
    }
}
```

**为什么这样设计、这样设计带来的好处：**

消息传递避免共享可变状态，把线程间协作变成数据流，故障边界更清晰。

```text
worker 1 ─┐
worker 2 ─┼─> channel ─> collector
worker 3 ─┘
```

## 3. 共享状态：`Arc<Mutex<T>>`

**标注：核心高频**

当多个线程必须修改同一份状态时，使用 `Arc` 共享所有权，用 `Mutex` 保证互斥访问。

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        let counter = Arc::clone(&counter);
        handles.push(thread::spawn(move || {
            let mut value = counter.lock().unwrap();
            *value += 1;
        }));
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("{}", *counter.lock().unwrap());
}
```

**为什么这样设计、这样设计带来的好处：**

`Arc` 解决“谁拥有”，`Mutex` 解决“谁能改”。Rust 类型系统强制你显式处理共享可变状态。

## 4. `Send` 与 `Sync`

**标注：重要**

`Send` 表示类型可以在线程间转移所有权，`Sync` 表示多个线程可以安全共享引用。

| trait | 含义 | 示例 |
|------|------|------|
| `Send` | 可以移动到另一个线程 | `String`, `Vec<T>` |
| `Sync` | `&T` 可以在线程间共享 | `Arc<T>`, `Mutex<T>` |

**为什么这样设计、这样设计带来的好处：**

Rust 把线程安全编码进类型系统。大多数时候你不需要手写它们，但编译器会阻止不安全类型跨线程使用。

## 5. 线程池思路

**标注：重要**

生产项目很少无限制创建线程，通常使用线程池限制并发。

```rust
use std::sync::{mpsc, Arc, Mutex};
use std::thread;

type Job = Box<dyn FnOnce() + Send + 'static>;

struct Worker {
    handle: thread::JoinHandle<()>,
}

fn main() {
    let (tx, rx) = mpsc::channel::<Job>();
    let rx = Arc::new(Mutex::new(rx));

    let workers: Vec<_> = (0..2)
        .map(|_| {
            let rx = Arc::clone(&rx);
            Worker {
                handle: thread::spawn(move || loop {
                    let job = rx.lock().unwrap().recv();
                    match job {
                        Ok(job) => job(),
                        Err(_) => break,
                    }
                }),
            }
        })
        .collect();

    tx.send(Box::new(|| println!("run job"))).unwrap();
    drop(tx);

    for worker in workers {
        worker.handle.join().unwrap();
    }
}
```

**为什么这样设计、这样设计带来的好处：**

线程池把任务数量和线程数量解耦，避免高并发场景下线程爆炸。

## 动手练习

### 练习 1：并发处理文件名

启动多个线程处理文件名列表，并等待全部完成。

**思路提示：** 使用 `move` 把每个文件名交给线程。

### 练习 2：结果汇总器

多个线程通过 channel 发送处理结果，主线程统计成功和失败数量。

**思路提示：** 发送结构体而不是字符串，便于扩展字段。

### 练习 3：共享计数器

用 `Arc<Mutex<u64>>` 实现多线程计数。

**思路提示：** 尽量缩短锁持有时间。

### 练习 4：迷你线程池

实现固定数量 worker，从 channel 接收任务执行。

**思路提示：** 任务类型可以用 `Box<dyn FnOnce() + Send + 'static>`。
