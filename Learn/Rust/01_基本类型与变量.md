# 第一章：基本类型与变量

> **难度：入门** | 重点理解不可变性默认和变量遮蔽，这是 Rust 与其他语言最大的思维差异之一。

---

## 1. 变量绑定

### 不可变性是默认的

```rust
fn main() {
    let x = 5;
    // x = 6;  // Error: cannot assign twice to immutable variable
    println!("x = {}", x);
}
```

**为什么 Rust 默认不可变？**

大多数变量在初始化后不需要改变。默认不可变让编译器可以做更多优化，也让代码更容易推理——你看到一个 `let x = 5`，就知道 `x` 在整个作用域内都是 5，不需要追踪它在哪里被修改了。

这与函数式编程的理念一致：减少可变状态，减少 bug。

### 可变变量

```rust
fn main() {
    let mut count = 0;  // mut 显式声明可变
    count += 1;
    count += 1;
    println!("count = {}", count); // 2
}
```

`mut` 是一个信号，告诉读代码的人："注意，这个变量后面会被修改"。

### 变量遮蔽 (Shadowing)

```rust
fn main() {
    let x = 5;
    let x = x + 1;      // 新的 x 遮蔽了旧的 x
    let x = x * 2;      // 再次遮蔽
    println!("x = {}", x); // 12

    // 遮蔽可以改变类型！
    let spaces = "   ";          // &str
    let spaces = spaces.len();   // usize，完全不同的类型
    println!("spaces = {}", spaces); // 3
}
```

**为什么允许遮蔽？**

遮蔽不是重新赋值，而是创建了一个新变量恰好同名。它的好处：
1. 避免起 `spaces_str`、`spaces_num` 这样的名字
2. 数据经过转换后，旧值不再需要，遮蔽让旧值自然失效
3. 新变量仍然是不可变的

### 常量

```rust
// 常量：编译时确定，全局可用，必须标注类型
const MAX_POINTS: u32 = 100_000;
const PI: f64 = 3.14159265358979;

// 与 let 的区别：
// 1. const 必须标注类型
// 2. const 必须是编译时可计算的值
// 3. const 可以在任何作用域声明（包括全局）
// 4. const 不能用 mut
// 5. const 命名惯例是 SCREAMING_SNAKE_CASE
```

## 2. 基本数据类型

### 2.1 整数类型

| 长度 | 有符号 | 无符号 |
|------|--------|--------|
| 8-bit | i8 | u8 |
| 16-bit | i16 | u16 |
| 32-bit | i32 | u32 |
| 64-bit | i64 | u64 |
| 128-bit | i128 | u128 |
| 平台相关 | isize | usize |

```rust
fn main() {
    let a: i32 = 42;          // 默认整数类型是 i32
    let b: u8 = 255;          // 无符号 8 位，范围 0-255
    let c: isize = -100;      // 平台相关（64位系统上是 i64）

    // 数字字面量的写法
    let decimal = 98_222;     // 下划线分隔，提高可读性
    let hex = 0xff;           // 十六进制
    let octal = 0o77;         // 八进制
    let binary = 0b1111_0000; // 二进制
    let byte = b'A';          // 字节（u8）

    println!("decimal: {}, hex: {}, binary: {}", decimal, hex, binary);
}
```

**为什么有这么多整数类型？**

Rust 是系统级语言，需要精确控制内存布局。在网络协议解析、嵌入式开发中，你需要知道一个字段到底占几个字节。`u8` 就是一个字节，`u32` 就是四个字节，没有歧义。

**usize 是什么？**

`usize` 是指针大小的无符号整数（64位系统上是 64 位）。所有索引和长度都是 `usize`，因为它能表示内存中任何位置的偏移量。

### 整数溢出

```rust
fn main() {
    let x: u8 = 255;
    // let y = x + 1;  // debug 模式下 panic，release 模式下回绕为 0

    // 显式处理溢出的方法
    let a = x.wrapping_add(1);   // 回绕：256 → 0
    let b = x.checked_add(1);    // 返回 Option：None
    let c = x.saturating_add(1); // 饱和：保持 255
    let d = x.overflowing_add(1); // 返回 (值, 是否溢出)

    println!("wrapping: {}, checked: {:?}, saturating: {}, overflowing: {:?}",
             a, b, c, d);
}
```

**为什么 Rust 要区分 debug 和 release 的溢出行为？**

debug 模式 panic 是为了帮你发现 bug；release 模式回绕是为了性能（溢出检查有运行时开销）。如果你需要明确的溢出行为，使用上面的方法。

### 2.2 浮点数

```rust
fn main() {
    let x = 2.0;      // f64（默认）
    let y: f32 = 3.0; // f32

    // 浮点数运算
    let sum = x + y as f64;  // 需要显式类型转换
    println!("sum = {}", sum);

    // 浮点数比较的陷阱
    let a = 0.1 + 0.2;
    println!("0.1 + 0.2 = {}", a);           // 0.30000000000000004
    println!("等于 0.3? {}", a == 0.3);       // false!

    // 正确的浮点比较
    let epsilon = f64::EPSILON;
    println!("近似等于 0.3? {}", (a - 0.3).abs() < epsilon);
}
```

### 2.3 布尔类型

```rust
fn main() {
    let t: bool = true;
    let f = false; // 类型推断

    // 布尔值占 1 字节（不是 1 位），因为内存最小寻址单位是字节
    println!("bool size: {} byte", std::mem::size_of::<bool>());
}
```

### 2.4 字符类型

```rust
fn main() {
    let c = 'z';
    let emoji = '😀';
    let chinese = '中';

    // char 是 4 字节的 Unicode 标量值（不是 1 字节的 ASCII！）
    println!("char size: {} bytes", std::mem::size_of::<char>()); // 4

    // 这与 C 语言的 char（1字节）完全不同
    // Rust 的 char 可以表示任何 Unicode 字符
}
```

## 3. 复合类型预览

### 元组 (Tuple)

```rust
fn main() {
    // 元组：固定长度，不同类型
    let tup: (i32, f64, bool) = (500, 6.4, true);

    // 解构
    let (x, y, z) = tup;
    println!("x={}, y={}, z={}", x, y, z);

    // 索引访问
    let first = tup.0;
    let second = tup.1;
    println!("first={}, second={}", first, second);

    // 单元元组 () —— 空值/无返回值
    let unit: () = ();
    // 不返回值的函数实际上返回 ()
}
```

### 数组 (Array)

```rust
fn main() {
    // 数组：固定长度，相同类型，栈上分配
    let arr: [i32; 5] = [1, 2, 3, 4, 5];
    let zeros = [0; 10]; // 10 个 0

    let first = arr[0];
    let last = arr[4];
    // let out = arr[10]; // 编译时如果能确定越界会报错，运行时越界会 panic

    println!("array length: {}", arr.len());

    // 数组切片
    let slice = &arr[1..3]; // [2, 3]，类型是 &[i32]
    println!("slice: {:?}", slice);
}
```

**数组 vs Vec：**
- 数组 `[T; N]`：编译时确定大小，栈上分配，性能最好
- Vec `Vec<T>`：动态大小，堆上分配，最常用

实际工程中，大多数场景使用 `Vec`。数组用于大小固定且已知的场景（如 RGB 颜色 `[u8; 3]`）。

## 4. 类型转换

```rust
fn main() {
    // Rust 不做隐式类型转换！必须显式转换
    let x: i32 = 42;
    let y: i64 = x as i64;  // as 关键字

    let f: f64 = 3.99;
    let i: i32 = f as i32;  // 截断，不是四舍五入！结果是 3

    // 字符串与数字转换
    let num_str = "42";
    let num: i32 = num_str.parse().expect("Not a number");
    // 或者用 turbofish 语法
    let num2 = "42".parse::<i32>().unwrap();

    let s = num.to_string(); // 数字转字符串

    println!("num={}, s={}", num, s);
}
```

**为什么 Rust 不做隐式转换？**

隐式转换是 C/C++ 中大量 bug 的来源。例如 `int` 到 `unsigned int` 的隐式转换可能导致负数变成巨大的正数。Rust 要求你显式写出每次转换，确保你知道自己在做什么。

## 5. 类型推断

```rust
fn main() {
    // Rust 有强大的类型推断，大多数时候不需要标注类型
    let x = 5;           // 推断为 i32
    let y = 3.14;        // 推断为 f64
    let z = true;        // 推断为 bool
    let s = "hello";     // 推断为 &str

    // 根据使用方式推断
    let mut vec = Vec::new(); // 此时还不知道类型
    vec.push(1);              // 现在推断为 Vec<i32>

    // 需要标注的场景：
    // 1. 编译器无法推断时
    let parsed: i64 = "42".parse().unwrap();
    // 2. 函数签名（必须标注）
    // 3. 你想要非默认类型时
    let small: i8 = 42;
}
```

## 6. 打印与格式化

```rust
fn main() {
    let name = "Rust";
    let version = 2021;

    // 基本打印
    println!("Hello, {}!", name);           // 位置参数
    println!("{0} is {0}", name);           // 重复使用
    println!("{name} {version}");           // 命名参数（Rust 1.58+）

    // 格式化
    println!("{:>10}", "right");            // 右对齐，宽度10
    println!("{:<10}", "left");             // 左对齐
    println!("{:^10}", "center");           // 居中
    println!("{:0>5}", 42);                 // 零填充：00042
    println!("{:.2}", 3.14159);             // 小数点后2位：3.14
    println!("{:b}", 42);                   // 二进制：101010
    println!("{:x}", 255);                  // 十六进制：ff
    println!("{:#x}", 255);                 // 带前缀：0xff

    // Debug 打印（用于调试，需要 #[derive(Debug)]）
    println!("{:?}", (1, 2, 3));            // (1, 2, 3)
    println!("{:#?}", vec![1, 2, 3]);       // 美化打印
}
```

## 7. 所有权初探（预告）

> 这里只做概念预告，详细内容在第二章。

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1 的所有权转移给了 s2

    // println!("{}", s1);  // Error! s1 已经无效了
    println!("{}", s2);     // OK

    // 为什么？因为 String 在堆上分配内存
    // 如果 s1 和 s2 都有效，谁负责释放内存？
    // Rust 的答案：转移所有权，确保只有一个所有者
}
```

这是 Rust 最独特的设计，也是学习曲线最陡的地方。下一章会详细讲解。

## 练习题

### 练习 1（基础）：温度转换器

编写一个程序，实现华氏温度和摄氏温度的互相转换：
- 从命令行读取一个数字和单位（如 `100 C` 或 `212 F`）
- 输出转换后的温度，保留两位小数
- 处理无效输入（非数字、无效单位）

### 练习 2（融合）：数字统计工具

编写一个程序，从命令行读取一系列数字（以空行结束），然后输出：
- 最大值、最小值、平均值
- 数字个数
- 所有数字的和
- 使用适当的类型（考虑溢出问题）

### 练习 3（扩展）：简易密码生成器

编写一个密码生成器：
- 接受密码长度作为参数
- 生成包含大小写字母、数字、特殊字符的随机密码
- 添加 `rand` crate 作为依赖
- 输出密码强度评估（弱/中/强）

---

**思路提示：**
- 练习 1：用 `split_whitespace()` 分割输入，`match` 匹配单位，公式 `C = (F-32) * 5/9`
- 练习 2：用 `Vec<f64>` 存储数字，`iter()` 配合 `fold()`/`max()`/`min()` 计算统计值。注意 `f64` 没有实现 `Ord` trait（因为 NaN），需要用 `partial_cmp`
- 练习 3：`cargo add rand`，使用 `rand::Rng` trait 的 `gen_range()` 方法。密码强度可以根据长度和字符种类数判断
