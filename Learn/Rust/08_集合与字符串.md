# 08 集合与字符串（Collections & Strings） 难度：中等

> Rust 的集合类型提供了高效、安全的数据结构。理解 String 与 &str 的区别是 Rust 字符串处理的关键。

---

## 1. Vec<T> —— 动态数组

### 1.1 创建与初始化

```rust
// 方式一：Vec::new()
let mut v: Vec<i32> = Vec::new();

// 方式二：vec! 宏（最常用）
let v = vec![1, 2, 3, 4, 5];

// 方式三：指定容量（避免频繁扩容）
let mut v: Vec<i32> = Vec::with_capacity(100);

// 方式四：从迭代器收集
let v: Vec<i32> = (1..=10).collect();

// 方式五：重复值
let v = vec![0; 10]; // 10 个 0
```

### 1.2 基本操作

```rust
let mut v = vec![1, 2, 3];

// 添加元素
v.push(4);           // 尾部追加
v.insert(0, 0);      // 在索引 0 处插入

// 删除元素
v.pop();             // 移除并返回最后一个 Option<T>
v.remove(0);         // 移除索引 0 的元素（O(n)）
v.swap_remove(0);    // 用最后一个元素替换（O(1)，不保序）

// 访问元素
let third = &v[2];          // 直接索引（越界会 panic）
let third = v.get(2);       // 返回 Option<&T>（安全）

// 遍历
for item in &v { println!("{}", item); }       // 不可变遍历
for item in &mut v { *item += 10; }            // 可变遍历
for item in v { println!("{}", item); }        // 消费遍历（v 不再可用）
```

### 1.3 切片操作

```rust
let v = vec![10, 20, 30, 40, 50];

let slice: &[i32] = &v[1..4];  // [20, 30, 40]
let first_three = &v[..3];      // [10, 20, 30]
let last_two = &v[3..];         // [40, 50]

// 切片方法
println!("{}", slice.contains(&30)); // true
println!("{:?}", slice.windows(2).collect::<Vec<_>>()); // [[20,30],[30,40]]
println!("{:?}", slice.chunks(2).collect::<Vec<_>>());  // [[20,30],[40]]
```

### 1.4 常用方法速查

```rust
let mut v = vec![3, 1, 4, 1, 5, 9, 2, 6];

v.sort();                    // 排序 [1, 1, 2, 3, 4, 5, 6, 9]
v.dedup();                   // 去除连续重复 [1, 2, 3, 4, 5, 6, 9]
v.retain(|&x| x > 3);       // 保留满足条件的 [4, 5, 6, 9]
v.truncate(2);               // 截断到长度 2 [4, 5]

let v = vec![1, 2, 3, 4, 5];
let sum: i32 = v.iter().sum();
let max = v.iter().max();    // Some(&5)
```

### 1.5 为什么这样设计

Vec 在堆上分配连续内存，类似 C++ 的 `std::vector`。连续内存意味着：
- 缓存友好，遍历快
- 随机访问 O(1)
- 尾部插入均摊 O(1)
- 中间插入/删除 O(n)（需要移动元素）

## 2. HashMap<K, V>

### 2.1 创建与基本操作

```rust
use std::collections::HashMap;

// 创建
let mut scores: HashMap<String, i32> = HashMap::new();

// 插入
scores.insert(String::from("Alice"), 95);
scores.insert(String::from("Bob"), 87);

// 从元组迭代器创建
let teams = vec!["Blue", "Red"];
let initial_scores = vec![10, 50];
let scores: HashMap<_, _> = teams.into_iter().zip(initial_scores).collect();

// 访问
let score = scores.get("Blue");  // Option<&i32>
let score = scores["Blue"];       // 直接取值（不存在会 panic）
```

### 2.2 Entry API（核心特性）

Entry API 是 Rust HashMap 最强大的特性之一，解决了"检查是否存在然后插入"的常见模式。

```rust
use std::collections::HashMap;

let mut word_count: HashMap<&str, i32> = HashMap::new();
let text = "hello world hello rust world hello";

for word in text.split_whitespace() {
    // or_insert: 不存在则插入默认值，返回 &mut V
    let count = word_count.entry(word).or_insert(0);
    *count += 1;
}
// {"hello": 3, "world": 2, "rust": 1}

// 其他 Entry 方法
let mut map: HashMap<String, Vec<i32>> = HashMap::new();

// or_insert_with: 惰性计算默认值
map.entry("key".to_string()).or_insert_with(Vec::new).push(1);

// or_default: 使用 Default trait
map.entry("key2".to_string()).or_default().push(2);

// and_modify: 存在时修改
map.entry("key".to_string())
    .and_modify(|v| v.push(99))
    .or_insert_with(|| vec![99]);
```

### 2.3 自定义 Hash

```rust
use std::collections::HashMap;
use std::hash::{Hash, Hasher};

#[derive(Eq, PartialEq)]
struct Point {
    x: i32,
    y: i32,
}

impl Hash for Point {
    fn hash<H: Hasher>(&self, state: &mut H) {
        self.x.hash(state);
        self.y.hash(state);
    }
}

let mut locations: HashMap<Point, &str> = HashMap::new();
locations.insert(Point { x: 0, y: 0 }, "原点");
```

### 2.4 为什么这样设计

HashMap 使用 SipHash 作为默认哈希算法，它对 HashDoS 攻击有抵抗力。如果你确定不需要这种保护（如内部数据），可以使用更快的哈希：

```rust
use std::collections::HashMap;
use std::hash::BuildHasherDefault;
// 需要添加依赖：rustc-hash = "1"
// use rustc_hash::FxHasher;
// type FxHashMap<K, V> = HashMap<K, V, BuildHasherDefault<FxHasher>>;
```
