# 10 智能指针

> **难度：⭐ 重点** | 智能指针是 Rust 在所有权规则之外表达堆分配、共享所有权、内部可变性的重要工具。

---

## 1. `Box<T>`：把值放到堆上

**标注：核心高频**

`Box<T>` 表示拥有堆上数据的指针。它存在的原因是：有些数据大小在编译期无法直接确定，或者你希望避免在栈上移动大对象。

```rust
enum Expr {
    Number(i64),
    Add(Box<Expr>, Box<Expr>),
}

fn eval(expr: &Expr) -> i64 {
    match expr {
        Expr::Number(n) => *n,
        Expr::Add(left, right) => eval(left) + eval(right),
    }
}

fn main() {
    let expr = Expr::Add(Box::new(Expr::Number(1)), Box::new(Expr::Number(2)));
    println!("{}", eval(&expr));
}
```

**为什么这样设计、这样设计带来的好处：**

递归类型如果直接包含自身，编译器无法计算大小。`Box<T>` 的大小固定，内部指向堆上数据，因此可以打破无限递归大小问题。

```text
Expr::Add
├─ Box -> Expr::Number(1)
└─ Box -> Expr::Number(2)
```

## 2. `Rc<T>`：单线程共享所有权

**标注：重要**

`Rc<T>` 是引用计数指针，允许多个所有者共享同一份只读数据。

```rust
use std::rc::Rc;

#[derive(Debug)]
struct Config {
    app_name: String,
}

fn main() {
    let config = Rc::new(Config { app_name: "tool".to_string() });
    let a = Rc::clone(&config);
    let b = Rc::clone(&config);

    println!("{} {}", a.app_name, b.app_name);
    println!("count={}", Rc::strong_count(&config));
}
```

**为什么这样设计、这样设计带来的好处：**

Rust 默认单所有权，`Rc` 用运行时计数换取共享所有权，适合配置、语法树节点、UI 树等单线程共享场景。

## 3. `Arc<T>`：多线程共享所有权

**标注：核心高频**

`Arc<T>` 是线程安全的引用计数指针，常用于多线程共享配置、连接池、应用状态。

```rust
use std::sync::Arc;
use std::thread;

fn main() {
    let app_name = Arc::new(String::from("worker"));
    let mut handles = vec![];

    for _ in 0..3 {
        let app_name = Arc::clone(&app_name);
        handles.push(thread::spawn(move || {
            println!("{}", app_name);
        }));
    }

    for handle in handles {
        handle.join().unwrap();
    }
}
```

**为什么这样设计、这样设计带来的好处：**

`Rc` 的计数不是原子操作，不能跨线程。`Arc` 使用原子计数，成本略高，但能安全跨线程共享。

| 指针 | 场景 | 是否线程安全 |
|------|------|--------------|
| `Rc<T>` | 单线程共享 | 否 |
| `Arc<T>` | 多线程共享 | 是 |

## 4. `RefCell<T>`：内部可变性

**标注：重要**

`RefCell<T>` 把借用检查从编译期推迟到运行期，适合测试替身、树结构父子引用等场景。

```rust
use std::cell::RefCell;

struct Metrics {
    count: RefCell<u64>,
}

impl Metrics {
    fn incr(&self) {
        *self.count.borrow_mut() += 1;
    }
}

fn main() {
    let metrics = Metrics { count: RefCell::new(0) };
    metrics.incr();
    println!("{}", metrics.count.borrow());
}
```

**为什么这样设计、这样设计带来的好处：**

有些对象从外部看是不可变的，但内部需要维护缓存或统计信息。`RefCell` 允许这种模式，但违反借用规则会在运行时 panic。

## 5. `Cow<'a, T>`：写时克隆

**标注：了解即可**

`Cow` 可以在不需要修改时借用数据，需要修改时再克隆。

```rust
use std::borrow::Cow;

fn normalize(input: &str) -> Cow<'_, str> {
    if input.contains(' ') {
        Cow::Owned(input.replace(' ', "_"))
    } else {
        Cow::Borrowed(input)
    }
}

fn main() {
    println!("{}", normalize("hello"));
    println!("{}", normalize("hello world"));
}
```

**为什么这样设计、这样设计带来的好处：**

它避免无意义分配，适合文本清洗、路径处理、HTTP header 规范化等性能敏感场景。
