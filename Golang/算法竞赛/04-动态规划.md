# 动态规划

> **目标**：建立 DP 思维框架——"状态是什么、转移方程怎么列、初始条件是什么"，然后用大量经典题目打磨直觉。

---

## 一、什么是动态规划？

### 1.1 一句话定义

**将大问题拆成子问题，记录子问题的答案避免重复计算。**

### 1.2 和递归/暴力的关系

```
暴力递归：从上到下拆问题，但有重复计算
  ↓ 加上记忆化
记忆化搜索（自顶向下DP）：递归 + 缓存
  ↓ 改为迭代
迭代DP（自底向上DP）：从小问题算到大问题
```

**经典例子：斐波那契数列**

```go
// 暴力递归 O(2^N) — 极慢，大量重复
func fibSlow(n int) int {
    if n <= 1 { return n }
    return fibSlow(n-1) + fibSlow(n-2)
}

// 记忆化搜索 O(N) — 加个缓存就快了
memo := map[int]int{}
func fibMemo(n int) int {
    if n <= 1 { return n }
    if v, ok := memo[n]; ok { return v }
    memo[n] = fibMemo(n-1) + fibMemo(n-2)
    return memo[n]
}

// 迭代DP O(N) — 最干净
func fib(n int) int {
    if n <= 1 { return n }
    a, b := 0, 1
    for i := 2; i <= n; i++ {
        a, b = b, a+b
    }
    return b
}
```

### 1.3 DP 三要素

| 要素 | 含义 | 斐波那契中的体现 |
|------|------|-----------------|
| **状态** | 用什么描述子问题 | `dp[i]` = 第 i 个斐波那契数 |
| **转移方程** | 大问题如何由小问题组合 | `dp[i] = dp[i-1] + dp[i-2]` |
| **初始条件** | 最小的子问题的答案 | `dp[0]=0, dp[1]=1` |

### 1.4 如何判断一道题能用 DP？

两个特征：
1. **最优子结构**：大问题的最优解包含子问题的最优解
2. **重叠子问题**：不同的拆分路径会遇到相同的子问题

## 二、线性 DP

### 2.1 最长递增子序列（LIS）

**题目**：给定数组，找最长递增子序列的长度。

```
[10, 9, 2, 5, 3, 7, 101, 18]
→ LIS = [2, 3, 7, 101]，长度 4
```

**方法1：O(N²) DP**

`dp[i]` = 以 `a[i]` 结尾的 LIS 长度

```go
func lengthOfLIS(nums []int) int {
    n := len(nums)
    dp := make([]int, n)
    ans := 1

    for i := 0; i < n; i++ {
        dp[i] = 1 // 至少包含自己
        for j := 0; j < i; j++ {
            if nums[j] < nums[i] && dp[j]+1 > dp[i] {
                dp[i] = dp[j] + 1
            }
        }
        if dp[i] > ans {
            ans = dp[i]
        }
    }
    return ans
}
```

**方法2：O(N log N) 贪心+二分**

维护一个数组 `tails`，`tails[i]` = 长度为 i+1 的递增子序列的最小末尾。

```go
func lengthOfLIS(nums []int) int {
    tails := []int{}
    for _, v := range nums {
        pos := sort.SearchInts(tails, v)
        if pos == len(tails) {
            tails = append(tails, v)
        } else {
            tails[pos] = v
        }
    }
    return len(tails)
}
```

### 2.2 最长公共子序列（LCS）

**题目**：两个字符串的最长公共子序列。

`dp[i][j]` = `s1[:i]` 和 `s2[:j]` 的 LCS 长度

```go
func longestCommonSubsequence(s1, s2 string) int {
    m, n := len(s1), len(s2)
    dp := make([][]int, m+1)
    for i := range dp {
        dp[i] = make([]int, n+1)
    }

    for i := 1; i <= m; i++ {
        for j := 1; j <= n; j++ {
            if s1[i-1] == s2[j-1] {
                dp[i][j] = dp[i-1][j-1] + 1
            } else {
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
            }
        }
    }
    return dp[m][n]
}
```

### 2.3 编辑距离

**题目**：将 `word1` 变成 `word2` 的最少操作次数（插入/删除/替换）。

```go
func minDistance(word1, word2 string) int {
    m, n := len(word1), len(word2)
    dp := make([][]int, m+1)
    for i := range dp {
        dp[i] = make([]int, n+1)
        dp[i][0] = i
    }
    for j := 0; j <= n; j++ {
        dp[0][j] = j
    }

    for i := 1; i <= m; i++ {
        for j := 1; j <= n; j++ {
            if word1[i-1] == word2[j-1] {
                dp[i][j] = dp[i-1][j-1] // 不需要操作
            } else {
                dp[i][j] = 1 + min(
                    dp[i-1][j],   // 删除
                    dp[i][j-1],   // 插入
                    dp[i-1][j-1], // 替换
                )
            }
        }
    }
    return dp[m][n]
}
```

## 三、背包问题

### 3.1 0/1 背包

**问题**：有 N 件物品，重量 w[i]，价值 v[i]，背包容量 W。每件物品只能选一次，求最大价值。

`dp[i][j]` = 前 i 件物品、容量 j 时的最大价值

```
转移方程：
dp[i][j] = max(
    dp[i-1][j],           // 不选第 i 件
    dp[i-1][j-w[i]] + v[i] // 选第 i 件（如果放得下）
)
```

**空间优化（一维）**：

```go
func knapsack01(W int, weights, values []int) int {
    dp := make([]int, W+1)
    for i := 0; i < len(weights); i++ {
        // 逆序遍历！保证每件物品只用一次
        for j := W; j >= weights[i]; j-- {
            if dp[j-weights[i]]+values[i] > dp[j] {
                dp[j] = dp[j-weights[i]] + values[i]
            }
        }
    }
    return dp[W]
}
```

**为什么逆序？** 如果正序，`dp[j-w]` 可能已经被更新过（包含了第 i 件物品），等于同一件物品用了两次。逆序则 `dp[j-w]` 还是不含第 i 件的值。

### 3.2 完全背包

**区别**：每件物品可以无限次选取。

**只需把 0/1 背包的逆序改成正序！**

```go
func knapsackComplete(W int, weights, values []int) int {
    dp := make([]int, W+1)
    for i := 0; i < len(weights); i++ {
        // 正序遍历！允许重复选
        for j := weights[i]; j <= W; j++ {
            if dp[j-weights[i]]+values[i] > dp[j] {
                dp[j] = dp[j-weights[i]] + values[i]
            }
        }
    }
    return dp[W]
}
```

### 3.3 多重背包

每件物品有数量限制 cnt[i]。二进制拆分转化为 0/1 背包。

```go
func knapsackMultiple(W int, weights, values, counts []int) int {
    // 二进制拆分
    var ws, vs []int
    for i := range weights {
        c := counts[i]
        k := 1
        for k <= c {
            ws = append(ws, weights[i]*k)
            vs = append(vs, values[i]*k)
            c -= k
            k *= 2
        }
        if c > 0 {
            ws = append(ws, weights[i]*c)
            vs = append(vs, values[i]*c)
        }
    }
    // 转化为 0/1 背包
    return knapsack01(W, ws, vs)
}
```

### 3.4 背包问题变体速查

| 变体 | 特征 | 转移 |
|------|------|------|
| 0/1 背包 | 每件一个 | 逆序 |
| 完全背包 | 无限个 | 正序 |
| 多重背包 | 有限个 | 二进制拆分 → 0/1 |
| 分组背包 | 每组选一个 | 枚举组内选哪个 |
| 求方案数 | 不是求最值而是计数 | `dp[j] += dp[j-w]` |
| 恰好装满 | 初始化不同 | `dp[0]=0`, 其余 `-∞` |

## 四、区间 DP

### 4.1 石子合并

**问题**：N 堆石子排成一行，每次合并相邻两堆（代价 = 两堆石子数之和），求最小总代价。

`dp[i][j]` = 合并区间 [i,j] 的最小代价

```go
func mergeStones(stones []int) int {
    n := len(stones)
    // 前缀和
    prefix := make([]int, n+1)
    for i := 0; i < n; i++ {
        prefix[i+1] = prefix[i] + stones[i]
    }

    dp := make([][]int, n)
    for i := range dp {
        dp[i] = make([]int, n)
    }

    // 枚举区间长度
    for length := 2; length <= n; length++ {
        for i := 0; i+length-1 < n; i++ {
            j := i + length - 1
            dp[i][j] = math.MaxInt64
            sum := prefix[j+1] - prefix[i]
            for k := i; k < j; k++ {
                cost := dp[i][k] + dp[k+1][j] + sum
                if cost < dp[i][j] {
                    dp[i][j] = cost
                }
            }
        }
    }
    return dp[0][n-1]
}
```

### 4.2 最长回文子序列

```go
func longestPalindromeSubseq(s string) int {
    n := len(s)
    dp := make([][]int, n)
    for i := range dp {
        dp[i] = make([]int, n)
        dp[i][i] = 1
    }

    for length := 2; length <= n; length++ {
        for i := 0; i+length-1 < n; i++ {
            j := i + length - 1
            if s[i] == s[j] {
                dp[i][j] = dp[i+1][j-1] + 2
            } else {
                dp[i][j] = max(dp[i+1][j], dp[i][j-1])
            }
        }
    }
    return dp[0][n-1]
}
```

## 五、树形 DP

### 5.1 没有上司的舞会

**问题**：公司关系构成一棵树，每人有快乐值。上下级不能同时参加。求最大快乐值。

```go
var (
    happiness []int
    children  [][]int
    dp        [][2]int // dp[u][0]=不选u, dp[u][1]=选u
)

func dfs(u int) {
    dp[u][0] = 0
    dp[u][1] = happiness[u]
    for _, v := range children[u] {
        dfs(v)
        dp[u][0] += max(dp[v][0], dp[v][1]) // 不选u → 子可选可不选
        dp[u][1] += dp[v][0]                  // 选u → 子一定不选
    }
}
```

## 六、状压 DP

### 6.1 核心思想

用一个整数的二进制位表示集合中哪些元素被选取。N ≤ 20 时适用。

```
集合 {0, 2, 3} 用二进制表示: 0b1101 = 13
第 0 位=1, 第 1 位=0, 第 2 位=1, 第 3 位=1
```

### 6.2 TSP 旅行商问题

```go
func tsp(dist [][]int) int {
    n := len(dist)
    full := 1 << n
    dp := make([][]int, full)
    for i := range dp {
        dp[i] = make([]int, n)
        for j := range dp[i] {
            dp[i][j] = math.MaxInt64 / 2
        }
    }
    dp[1][0] = 0 // 从 0 出发，已访问 {0}

    for mask := 1; mask < full; mask++ {
        for u := 0; u < n; u++ {
            if mask>>u&1 == 0 || dp[mask][u] >= math.MaxInt64/2 {
                continue
            }
            for v := 0; v < n; v++ {
                if mask>>v&1 == 1 {
                    continue
                }
                newMask := mask | (1 << v)
                if dp[mask][u]+dist[u][v] < dp[newMask][v] {
                    dp[newMask][v] = dp[mask][u] + dist[u][v]
                }
            }
        }
    }

    ans := math.MaxInt64
    for u := 0; u < n; u++ {
        if dp[full-1][u]+dist[u][0] < ans {
            ans = dp[full-1][u] + dist[u][0]
        }
    }
    return ans
}
```

### 6.3 位运算速查

```go
1 << i           // 第 i 位为 1
mask >> i & 1    // 检查第 i 位
mask | (1 << i)  // 设置第 i 位为 1
mask & ^(1 << i) // 清除第 i 位
mask & (mask-1)  // 去掉最低位的 1
bits.OnesCount(uint(mask))  // 1 的个数
```

## 七、DP 解题思路总结

```
1. 确定状态：用什么变量描述子问题？
   → 一维数组下标？二维(行,列)？集合(状压)？
   
2. 写出转移方程：大状态如何由小状态算出？
   → 枚举"最后一步"是什么操作
   
3. 确定初始条件和边界
   → dp[0]=?, dp[...][0]=?
   
4. 确定遍历顺序
   → 确保计算 dp[i] 时，所依赖的 dp[j] (j<i) 已经算好
   
5. 优化空间（可选）
   → 滚动数组、一维数组
```

## 速查表

| 问题类型 | 状态设计 | 复杂度 |
|---------|---------|---------|
| LIS | `dp[i]` = 以 i 结尾的最长 | O(N²) 或 O(N log N) |
| LCS | `dp[i][j]` = 前缀匹配 | O(NM) |
| 编辑距离 | `dp[i][j]` = 前缀编辑 | O(NM) |
| 0/1 背包 | `dp[j]` = 容量 j 最大价值 | O(NW) |
| 完全背包 | `dp[j]` = 容量 j 最大价值 | O(NW) |
| 区间 DP | `dp[i][j]` = 区间最优 | O(N³) |
| 树形 DP | `dp[u][0/1]` = 选/不选 | O(N) |
| 状压 DP | `dp[mask][i]` = 集合状态 | O(2^N · N²) |
