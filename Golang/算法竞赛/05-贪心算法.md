# 贪心算法

> **目标**：理解贪心的核心思想——"每一步都选当前最优"，掌握证明贪心正确性的方法，积累经典贪心题目模式。

---

## 一、贪心 vs 动态规划

| 特征 | 贪心 | DP |
|------|------|-----|
| 决策方式 | 每步选当前最优，不回溯 | 枚举所有选择，取全局最优 |
| 正确性 | 需要证明（不一定对） | 只要状态转移正确就一定对 |
| 效率 | 通常更快 | 通常较慢 |
| 适用情况 | 有贪心选择性质 | 有最优子结构 + 重叠子问题 |

**直觉**：如果"局部最优 = 全局最优"，用贪心；否则用 DP。

---

## 二、贪心证明方法

### 2.1 交换论证法（最常用）

假设最优解不包含贪心选择，通过交换证明把非贪心选择换成贪心选择不会变差。

**模板**：
1. 假设存在一个最优解 OPT 不按贪心策略做
2. 找到 OPT 中第一个与贪心不同的位置
3. 把那个位置"交换"成贪心选择
4. 证明交换后的解不比 OPT 差
5. 矛盾（OPT 应该是最优的）or 得出贪心解也是最优

## 三、经典贪心问题

### 3.1 活动选择（区间调度）

**问题**：有 N 个活动，每个有开始和结束时间。同时只能参加一个活动，求最多参加几个。

**贪心策略**：按结束时间排序，能参加就参加。

```go
func maxActivities(intervals [][]int) int {
    sort.Slice(intervals, func(i, j int) bool {
        return intervals[i][1] < intervals[j][1] // 按结束时间排序
    })

    count := 0
    lastEnd := -1
    for _, itv := range intervals {
        if itv[0] >= lastEnd { // 不重叠就参加
            count++
            lastEnd = itv[1]
        }
    }
    return count
}
```

**为什么正确？** 越早结束的活动给后面留的时间越多。选一个晚结束的活动，最多也只是换了一个活动，不会更优。

### 3.2 跳跃游戏

**问题**：数组 nums[i] 表示从位置 i 最远能跳 nums[i] 步。能否到达最后一个位置？

```go
func canJump(nums []int) bool {
    maxReach := 0
    for i := 0; i < len(nums); i++ {
        if i > maxReach {
            return false // 当前位置超出了能到达的最远距离
        }
        if i+nums[i] > maxReach {
            maxReach = i + nums[i]
        }
    }
    return true
}
```

### 3.3 跳跃游戏 II（最少跳数）

```go
func jump(nums []int) int {
    jumps := 0
    curEnd := 0   // 当前跳跃能到的最远位置
    farthest := 0 // 在这一步内遍历到的最远位置

    for i := 0; i < len(nums)-1; i++ {
        if i+nums[i] > farthest {
            farthest = i + nums[i]
        }
        if i == curEnd { // 必须跳了
            jumps++
            curEnd = farthest
        }
    }
    return jumps
}
```

### 3.4 分发糖果

**问题**：N 个孩子站一排，每人有评分。给每人至少 1 颗糖，评分高的比邻居多。求最少总糖数。

```go
func candy(ratings []int) int {
    n := len(ratings)
    candies := make([]int, n)
    for i := range candies {
        candies[i] = 1
    }

    // 从左到右：右边比左边评分高则多一颗
    for i := 1; i < n; i++ {
        if ratings[i] > ratings[i-1] {
            candies[i] = candies[i-1] + 1
        }
    }
    // 从右到左：左边比右边评分高则取较大值
    for i := n - 2; i >= 0; i-- {
        if ratings[i] > ratings[i+1] && candies[i] <= candies[i+1] {
            candies[i] = candies[i+1] + 1
        }
    }

    total := 0
    for _, c := range candies {
        total += c
    }
    return total
}
```

### 3.5 加油站

**问题**：环形公路上有 N 个加油站，gas[i] 是加油量，cost[i] 是到下一站的油耗。找到一个出发站使得能走完一圈。

```go
func canCompleteCircuit(gas []int, cost []int) int {
    totalGas, totalCost := 0, 0
    tank := 0
    start := 0

    for i := 0; i < len(gas); i++ {
        totalGas += gas[i]
        totalCost += cost[i]
        tank += gas[i] - cost[i]
        if tank < 0 {
            start = i + 1 // 从 0~i 都不行，从 i+1 开始
            tank = 0
        }
    }

    if totalGas < totalCost {
        return -1 // 油总量不够
    }
    return start
}
```

### 3.6 合并区间

```go
func mergeIntervals(intervals [][]int) [][]int {
    sort.Slice(intervals, func(i, j int) bool {
        return intervals[i][0] < intervals[j][0]
    })

    result := [][]int{intervals[0]}
    for i := 1; i < len(intervals); i++ {
        last := result[len(result)-1]
        if intervals[i][0] <= last[1] {
            // 重叠，扩展
            if intervals[i][1] > last[1] {
                last[1] = intervals[i][1]
            }
        } else {
            result = append(result, intervals[i])
        }
    }
    return result
}
```

### 3.7 Huffman 编码（贪心 + 堆）

每次合并两个频率最小的节点，构建最优前缀编码树。

```go
func huffmanCost(freqs []int) int {
    h := &MinHeap{}
    for _, f := range freqs {
        *h = append(*h, f)
    }
    heap.Init(h)

    totalCost := 0
    for h.Len() > 1 {
        a := heap.Pop(h).(int)
        b := heap.Pop(h).(int)
        totalCost += a + b
        heap.Push(h, a+b)
    }
    return totalCost
}
```

## 四、贪心常见模式

| 模式 | 策略 | 典型题 |
|------|------|--------|
| 排序 + 顺序选 | 按某个维度排序后依次选择 | 活动选择、会议室 |
| 从两端向中间 | 同时处理左右约束 | 分发糖果 |
| 维护当前最优 | 遍历时更新一个全局变量 | 跳跃游戏、买卖股票 |
| 堆辅助 | 每步取最优用堆 | Huffman、任务调度器 |
| 反悔贪心 | 贪心选完后发现更优则替换 | 带权任务调度 |

---

## 五、贪心易错点

1. **不能贪就别贪**：贪心必须证明正确性。如果不确定，先用 DP 写个暴力验证。
2. **排序维度选错**：活动选择问题中，按开始时间排序是错误的，必须按结束时间。
3. **忘记特殊情况**：如空数组、只有一个元素等。

## 速查表

| 问题 | 贪心策略 | 时间复杂度 |
|------|---------|----------|
| 活动选择 | 按结束时间排序 | O(N log N) |
| 跳跃游戏 | 维护 maxReach | O(N) |
| 分发糖果 | 左右两遍扫描 | O(N) |
| 加油站 | 累计差值，负数则换起点 | O(N) |
| 合并区间 | 按左端点排序 | O(N log N) |
| Huffman | 堆取两最小合并 | O(N log N) |
