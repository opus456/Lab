## 一、排序算法总览

| 算法 | 平均 | 最坏 | 空间 | 稳定 | 特点 |
|------|------|------|------|------|------|
| 冒泡排序 | O(N²) | O(N²) | O(1) | ✅ | 最简单，几乎不用 |
| 选择排序 | O(N²) | O(N²) | O(1) | ❌ | 最简单，几乎不用 |
| 插入排序 | O(N²) | O(N²) | O(1) | ✅ | 小数据或近乎有序时极快 |
| 归并排序 | O(N log N) | O(N log N) | O(N) | ✅ | 稳定，适合链表/外部排序 |
| 快速排序 | O(N log N) | O(N²) | O(log N) | ❌ | 实践最快，Go 标准库用它 |
| 堆排序 | O(N log N) | O(N log N) | O(1) | ❌ | 原地、不稳定 |
| 计数排序 | O(N+K) | O(N+K) | O(K) | ✅ | 值域小时极快 |
| 基数排序 | O(d·N) | O(d·N) | O(N) | ✅ | 整数排序 |

**稳定性**：相等元素的相对顺序是否保持不变。

## 二、核心排序算法详解

### 2.1 快速排序

**核心思想**：选一个"基准"(pivot)，把数组分成 ≤pivot 和 >pivot 两部分，然后递归排序两部分。

```
原始: [3, 6, 8, 10, 1, 2, 1]
选 pivot = 3
分区: [1, 2, 1] | 3 | [6, 8, 10]
      ↑ ≤3 的      ↑ >3 的
递归排序两边...
结果: [1, 1, 2, 3, 6, 8, 10]
```

```go
func quickSort(a []int, lo, hi int) {
    if lo >= hi {
        return
    }
    pivot := a[lo+(hi-lo)/2] // 选中间元素做基准
    i, j := lo, hi
    for i <= j {
        for a[i] < pivot { i++ }
        for a[j] > pivot { j-- }
        if i <= j {
            a[i], a[j] = a[j], a[i]
            i++
            j--
        }
    }
    quickSort(a, lo, j)
    quickSort(a, i, hi)
}
```

**为什么实践中最快？**
- 缓存友好：数据连续访问
- 常数因子小
- 平均每次分区恰好把数据减半 → O(N log N)

**最坏情况**：每次 pivot 选到最大/最小值 → 退化为 O(N²)。解决方法：随机选 pivot 或三数取中。

### 2.2 归并排序

**核心思想**：把数组一直对半分，直到只剩一个元素，然后从底向上两两合并。

```
[3, 6, 8, 10, 1, 2, 1]
→ [3, 6, 8] | [10, 1, 2, 1]                拆
→ [3, 6] [8] | [10, 1] [2, 1]              拆
→ [3] [6] [8] | [10] [1] [2] [1]           拆到底
→ [3, 6] [8] | [1, 10] [1, 2]              合并
→ [3, 6, 8] | [1, 1, 2, 10]                合并
→ [1, 1, 2, 3, 6, 8, 10]                   合并
```

```go
func mergeSort(a []int) []int {
    if len(a) <= 1 {
        return a
    }
    mid := len(a) / 2
    left := mergeSort(a[:mid])
    right := mergeSort(a[mid:])
    return merge(left, right)
}

func merge(a, b []int) []int {
    result := make([]int, 0, len(a)+len(b))
    i, j := 0, 0
    for i < len(a) && j < len(b) {
        if a[i] <= b[j] {
            result = append(result, a[i])
            i++
        } else {
            result = append(result, b[j])
            j++
        }
    }
    result = append(result, a[i:]...)
    result = append(result, b[j:]...)
    return result
}
```

**归并排序的额外妙用**：求逆序对数量（在 merge 过程中统计）。

### 2.3 堆排序

利用堆的性质排序。先建最大堆，然后反复把堆顶（最大值）和最后一个交换，缩小堆的范围。

```go
func heapSort(a []int) {
    n := len(a)
    // 建堆（从最后一个非叶节点开始下沉）
    for i := n/2 - 1; i >= 0; i-- {
        siftDown(a, i, n)
    }
    // 排序
    for size := n - 1; size > 0; size-- {
        a[0], a[size] = a[size], a[0] // 最大值放到末尾
        siftDown(a, 0, size)           // 维护堆
    }
}

func siftDown(a []int, i, n int) {
    for {
        largest := i
        left, right := 2*i+1, 2*i+2
        if left < n && a[left] > a[largest] {
            largest = left
        }
        if right < n && a[right] > a[largest] {
            largest = right
        }
        if largest == i {
            break
        }
        a[i], a[largest] = a[largest], a[i]
        i = largest
    }
}
```

### 2.4 竞赛中直接用标准库

```go
import "sort"

a := []int{3, 1, 4, 1, 5, 9}

// 升序
sort.Ints(a)

// 降序
sort.Sort(sort.Reverse(sort.IntSlice(a)))

// 自定义排序
sort.Slice(a, func(i, j int) bool {
    return a[i] > a[j] // 降序
})

// 判断是否已排序
sort.IntsAreSorted(a)
```

## 三、二分搜索（最最最重要）

### 3.1 为什么二分搜索经常写错？

因为边界条件太多了！`lo <= hi` 还是 `lo < hi`？`hi = mid` 还是 `hi = mid - 1`？

**一劳永逸的方法**：记住一个万能模板，所有变体从它推导。

### 3.2 万能模板

```go
// 找第一个满足条件的位置
// check(mid) 在 [lo, hi] 上呈 FFFFFFTTTTTT 的形态
// 返回第一个 T 的位置
func lowerBound(lo, hi int, check func(int) bool) int {
    for lo < hi {
        mid := lo + (hi-lo)/2
        if check(mid) {
            hi = mid     // mid 满足，答案可能就是 mid 或更左
        } else {
            lo = mid + 1 // mid 不满足，答案一定在右边
        }
    }
    return lo // lo == hi，即为答案
}
```

**直觉**：想象一个数轴，左边全是 F，右边全是 T，二分找 F 和 T 的分界线。

```
索引:  0  1  2  3  4  5  6  7
check: F  F  F  T  T  T  T  T
                ↑
             答案 = 3
```

### 3.3 从万能模板推导所有变体

**变体1：查找 x 在有序数组中第一次出现的位置**

```go
func firstOccurrence(a []int, x int) int {
    lo, hi := 0, len(a)
    for lo < hi {
        mid := lo + (hi-lo)/2
        if a[mid] >= x { // 条件：≥ x
            hi = mid
        } else {
            lo = mid + 1
        }
    }
    if lo < len(a) && a[lo] == x {
        return lo
    }
    return -1
}
```

**变体2：查找 x 最后一次出现的位置**

```go
func lastOccurrence(a []int, x int) int {
    lo, hi := 0, len(a)
    for lo < hi {
        mid := lo + (hi-lo)/2
        if a[mid] > x { // 条件：> x（找第一个 > x 的位置，再 -1）
            hi = mid
        } else {
            lo = mid + 1
        }
    }
    if lo > 0 && a[lo-1] == x {
        return lo - 1
    }
    return -1
}
```

**变体3：查找第一个 ≥ x 的位置（lower_bound）**

```go
func lowerBound(a []int, x int) int {
    lo, hi := 0, len(a)
    for lo < hi {
        mid := lo + (hi-lo)/2
        if a[mid] >= x {
            hi = mid
        } else {
            lo = mid + 1
        }
    }
    return lo
}
```

**变体4：查找第一个 > x 的位置（upper_bound）**

```go
func upperBound(a []int, x int) int {
    lo, hi := 0, len(a)
    for lo < hi {
        mid := lo + (hi-lo)/2
        if a[mid] > x {
            hi = mid
        } else {
            lo = mid + 1
        }
    }
    return lo
}
```

**用标准库**：
```go
sort.SearchInts(a, x)   // 等价于 lowerBound
sort.Search(n, func(i int) bool { return a[i] > x }) // upperBound
```

### 3.4 二分答案

很多题目不是在数组上二分，而是**二分答案**。

**模式**：如果答案满足单调性（答案越大越容易满足/越不容易满足），就可以二分。

**例题：将数组分成 K 段，使最大段和最小**

```go
func splitArray(nums []int, k int) int {
    lo, hi := 0, 0
    for _, v := range nums {
        if v > lo { lo = v }  // 答案下界：最大的单个元素
        hi += v                // 答案上界：所有元素之和
    }

    for lo < hi {
        mid := lo + (hi-lo)/2
        if canSplit(nums, k, mid) {
            hi = mid    // mid 可行，尝试更小的
        } else {
            lo = mid + 1 // mid 不行，答案更大
        }
    }
    return lo
}

// 判断：每段和不超过 maxSum 时，能否分成 ≤ k 段？
func canSplit(nums []int, k, maxSum int) bool {
    count, curSum := 1, 0
    for _, v := range nums {
        if curSum+v > maxSum {
            count++
            curSum = v
        } else {
            curSum += v
        }
    }
    return count <= k
}
```

### 3.5 实数二分

答案是浮点数时，用精度控制循环次数：

```go
func sqrtFloat(x float64) float64 {
    lo, hi := 0.0, x
    if x < 1 { hi = 1 }

    for i := 0; i < 100; i++ { // 100 次迭代，精度约 10^-30
        mid := (lo + hi) / 2
        if mid*mid <= x {
            lo = mid
        } else {
            hi = mid
        }
    }
    return lo
}
```

**为什么用固定次数而不是 `hi - lo > eps`？** 因为浮点数精度问题——当 lo 和 hi 都很大时，`hi - lo` 可能永远不会小于 eps，导致死循环。固定 100 次一定终止。

## 四、特殊排序与相关技巧

### 4.1 计数排序

当值域 [0, K] 较小时，O(N+K)。

```go
func countingSort(a []int, maxVal int) []int {
    count := make([]int, maxVal+1)
    for _, v := range a {
        count[v]++
    }
    result := make([]int, 0, len(a))
    for val, cnt := range count {
        for i := 0; i < cnt; i++ {
            result = append(result, val)
        }
    }
    return result
}
```

### 4.2 逆序对（归并排序变体）

```go
var invCount int

func mergeSortCount(a []int) []int {
    if len(a) <= 1 {
        return a
    }
    mid := len(a) / 2
    left := mergeSortCount(a[:mid])
    right := mergeSortCount(a[mid:])
    return mergeCount(left, right)
}

func mergeCount(a, b []int) []int {
    result := make([]int, 0, len(a)+len(b))
    i, j := 0, 0
    for i < len(a) && j < len(b) {
        if a[i] <= b[j] {
            result = append(result, a[i])
            i++
        } else {
            result = append(result, b[j])
            invCount += len(a) - i // 关键：a[i:] 都比 b[j] 大
            j++
        }
    }
    result = append(result, a[i:]...)
    result = append(result, b[j:]...)
    return result
}
```
