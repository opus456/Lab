## 一、KMP 算法（模式匹配）

### 1.1 问题

在文本串 `text` 中查找模式串 `pattern` 出现的所有位置。

### 1.2 核心思想

暴力匹配在失配时回退到开头重新匹配，KMP 利用已匹配的信息"跳过"不必要的比较。

关键是构建 **next 数组**（又叫 fail/前缀函数）：`next[i]` = `pattern[:i+1]` 的最长相等前后缀长度。

```
pattern:  A B A B C
next:     0 0 1 2 0

解释: "ABAB" 的前缀 "AB" == 后缀 "AB"，长度 2
```

### 1.3 Go 实现

```go
// 构建 next 数组
func buildNext(pattern string) []int {
    n := len(pattern)
    next := make([]int, n)
    j := 0
    for i := 1; i < n; i++ {
        for j > 0 && pattern[i] != pattern[j] {
            j = next[j-1]
        }
        if pattern[i] == pattern[j] {
            j++
        }
        next[i] = j
    }
    return next
}

// KMP 搜索
func kmpSearch(text, pattern string) []int {
    next := buildNext(pattern)
    matches := []int{}
    j := 0
    for i := 0; i < len(text); i++ {
        for j > 0 && text[i] != pattern[j] {
            j = next[j-1]
        }
        if text[i] == pattern[j] {
            j++
        }
        if j == len(pattern) {
            matches = append(matches, i-j+1)
            j = next[j-1]
        }
    }
    return matches
}
```

**时间复杂度**：O(N+M)，N=文本长度，M=模式长度。

## 二、Trie（字典树/前缀树）

### 2.1 核心概念

Trie 是一棵多叉树，每条边代表一个字符，从根到某个节点的路径构成一个字符串前缀。

```
插入: "app", "apple", "bat"

         root
        /    \
       a      b
       |      |
       p      a
       |      |
       p*     t*
       |
       l
       |
       e*
(* 表示一个完整单词的结尾)
```

### 2.2 Go 实现

```go
type TrieNode struct {
    children [26]*TrieNode
    isEnd    bool
}

type Trie struct {
    root *TrieNode
}

func NewTrie() *Trie {
    return &Trie{root: &TrieNode{}}
}

func (t *Trie) Insert(word string) {
    node := t.root
    for _, ch := range word {
        idx := ch - 'a'
        if node.children[idx] == nil {
            node.children[idx] = &TrieNode{}
        }
        node = node.children[idx]
    }
    node.isEnd = true
}

func (t *Trie) Search(word string) bool {
    node := t.root
    for _, ch := range word {
        idx := ch - 'a'
        if node.children[idx] == nil {
            return false
        }
        node = node.children[idx]
    }
    return node.isEnd
}

func (t *Trie) StartsWith(prefix string) bool {
    node := t.root
    for _, ch := range prefix {
        idx := ch - 'a'
        if node.children[idx] == nil {
            return false
        }
        node = node.children[idx]
    }
    return true
}
```

**应用**：自动补全、拼写检查、IP 路由匹配、前缀统计。

## 三、字符串哈希

### 3.1 核心思想

把字符串映射为一个整数，用 O(1) 比较两个子串是否相同。

**公式**：$H(s) = s_0 \cdot p^{n-1} + s_1 \cdot p^{n-2} + ... + s_{n-1} \cdot p^0 \pmod{M}$

### 3.2 Go 实现

```go
const (
    hashBase = 131
    hashMod  = 1_000_000_007
)

// 预处理前缀哈希和幂次
func buildHash(s string) ([]int, []int) {
    n := len(s)
    h := make([]int, n+1)
    pw := make([]int, n+1)
    pw[0] = 1

    for i := 0; i < n; i++ {
        h[i+1] = (h[i]*hashBase + int(s[i])) % hashMod
        pw[i+1] = pw[i] * hashBase % hashMod
    }
    return h, pw
}

// 获取 s[l:r+1] 的哈希值 (0-indexed)
func getHash(h, pw []int, l, r int) int {
    return ((h[r+1] - h[l]*pw[r-l+1]%hashMod) % hashMod + hashMod) % hashMod
}
```

**应用**：O(1) 判断子串相等、Rabin-Karp 多模式匹配。

**防冲突**：使用双哈希（两个不同的 base 和 mod）。

## 四、其他常用字符串技巧

### 4.1 回文判断

```go
func isPalindrome(s string) bool {
    for i, j := 0, len(s)-1; i < j; i, j = i+1, j-1 {
        if s[i] != s[j] {
            return false
        }
    }
    return true
}
```

### 4.2 最长回文子串（中心扩展）

```go
func longestPalindrome(s string) string {
    best := ""

    expand := func(l, r int) string {
        for l >= 0 && r < len(s) && s[l] == s[r] {
            l--
            r++
        }
        return s[l+1 : r]
    }

    for i := 0; i < len(s); i++ {
        // 奇数长度
        p1 := expand(i, i)
        if len(p1) > len(best) { best = p1 }
        // 偶数长度
        p2 := expand(i, i+1)
        if len(p2) > len(best) { best = p2 }
    }
    return best
}
```

## 速查表

| 问题 | 算法 | 时间 |
|------|------|------|
| 单模式匹配 | KMP | O(N+M) |
| 前缀查询 | Trie | O(L)每次操作 |
| 子串比较 | 字符串哈希 | O(1)查询, O(N)预处理 |
| 最长回文子串 | 中心扩展 / Manacher | O(N²) / O(N) |
