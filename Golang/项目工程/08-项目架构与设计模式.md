## 一、Go Modules

### 1.1 初始化项目

```bash
mkdir myproject && cd myproject
go mod init github.com/yourname/myproject
```

### 1.2 go.mod 文件

```
module github.com/yourname/myproject

go 1.22

require (
    github.com/gin-gonic/gin v1.9.1
    gorm.io/gorm v1.25.7
)

require (
    // indirect: 间接依赖
    golang.org/x/sys v0.15.0 // indirect
)
```

### 1.3 常用命令

```bash
go mod tidy      # 整理依赖（添加缺少的，删除多余的）
go mod download  # 下载依赖到本地缓存
go mod vendor    # 将依赖复制到 vendor/ 目录
go mod graph     # 显示依赖图
go get -u ./...  # 升级所有直接依赖
```

## 二、项目结构

### 2.1 标准布局

```
myproject/
├── cmd/                    # 可执行程序入口
│   ├── server/
│   │   └── main.go         # HTTP 服务入口
│   └── worker/
│       └── main.go         # 后台任务入口
├── internal/               # 内部包（不可被外部导入）
│   ├── handler/            # HTTP 处理层
│   │   └── user.go
│   ├── service/            # 业务逻辑层
│   │   └── user.go
│   ├── repository/         # 数据访问层
│   │   └── user.go
│   ├── model/              # 数据模型
│   │   └── user.go
│   └── middleware/         # 中间件
│       └── auth.go
├── pkg/                    # 公共包（可被外部导入）
│   ├── logger/
│   └── response/
├── config/                 # 配置文件
│   └── config.yaml
├── migrations/             # 数据库迁移
├── docs/                   # 文档
├── go.mod
├── go.sum
├── Makefile
└── Dockerfile
```

### 2.2 关键目录说明

| 目录 | 作用 | 关键点 |
|------|------|--------|
| `cmd/` | 程序入口 | 尽量薄，只负责启动和组装 |
| `internal/` | 私有代码 | Go 编译器强制：外部包不能导入 |
| `pkg/` | 公共库 | 可以被其他项目导入 |
| `handler/` | HTTP 处理 | 参数解析、响应格式化 |
| `service/` | 业务逻辑 | 核心业务，不依赖 HTTP |
| `repository/` | 数据访问 | 封装数据库/缓存操作 |

### 2.3 分层架构

```
请求 → Handler → Service → Repository → 数据库
                    ↓
              Model (数据实体)

Handler:  解析参数，调用 Service，返回响应
Service:  业务规则，组合多个 Repository
Repository: SQL 查询，缓存操作
```

## 三、依赖注入

### 3.1 手动注入（推荐）

```go
// repository/user.go
type UserRepository struct{ db *gorm.DB }
func NewUserRepository(db *gorm.DB) *UserRepository {
    return &UserRepository{db: db}
}
func (r *UserRepository) GetByID(ctx context.Context, id int) (*model.User, error) {
    var user model.User
    err := r.db.WithContext(ctx).First(&user, id).Error
    return &user, err
}

// service/user.go
type UserService struct{ repo *repository.UserRepository }
func NewUserService(repo *repository.UserRepository) *UserService {
    return &UserService{repo: repo}
}
func (s *UserService) GetByID(ctx context.Context, id int) (*model.User, error) {
    return s.repo.GetByID(ctx, id)
}

// handler/user.go
type UserHandler struct{ svc *service.UserService }
func NewUserHandler(svc *service.UserService) *UserHandler {
    return &UserHandler{svc: svc}
}

// cmd/server/main.go — 在入口组装依赖
func main() {
    db := initDB()
    repo := repository.NewUserRepository(db)
    svc := service.NewUserService(repo)
    handler := handler.NewUserHandler(svc)

    r := gin.Default()
    r.GET("/users/:id", handler.GetByID)
    r.Run(":8080")
}
```

### 3.2 Wire（Google 依赖注入代码生成）

```go
// wire.go
//go:build wireinject

package main

import "github.com/google/wire"

func InitApp() (*App, error) {
    wire.Build(
        initDB,
        repository.NewUserRepository,
        service.NewUserService,
        handler.NewUserHandler,
        NewApp,
    )
    return nil, nil
}
```

```bash
go install github.com/google/wire/cmd/wire@latest
wire ./cmd/server/  # 生成 wire_gen.go
```

## 四、配置管理

### 4.1 Viper

```bash
go get github.com/spf13/viper
```

```yaml
# config/config.yaml
server:
  port: 8080
  mode: release
database:
  host: localhost
  port: 3306
  name: mydb
  user: root
  password: secret
redis:
  addr: localhost:6379
```

```go
type Config struct {
    Server   ServerConfig   `mapstructure:"server"`
    Database DatabaseConfig `mapstructure:"database"`
    Redis    RedisConfig    `mapstructure:"redis"`
}

type ServerConfig struct {
    Port int    `mapstructure:"port"`
    Mode string `mapstructure:"mode"`
}

func LoadConfig(path string) (*Config, error) {
    viper.SetConfigFile(path)
    viper.AutomaticEnv() // 环境变量覆盖

    if err := viper.ReadInConfig(); err != nil {
        return nil, err
    }
    var cfg Config
    if err := viper.Unmarshal(&cfg); err != nil {
        return nil, err
    }
    return &cfg, nil
}
```

## 五、常用设计模式

### 5.1 单例模式

```go
var (
    instance *Database
    once     sync.Once
)

func GetDB() *Database {
    once.Do(func() {
        instance = &Database{}
        instance.Connect()
    })
    return instance
}
```

### 5.2 工厂模式

```go
type Cache interface {
    Get(key string) (string, error)
    Set(key, value string, ttl time.Duration) error
}

type MemoryCache struct{ /* ... */ }
type RedisCache struct{ /* ... */ }

func NewCache(cacheType string) Cache {
    switch cacheType {
    case "redis":
        return &RedisCache{}
    default:
        return &MemoryCache{}
    }
}
```

### 5.3 策略模式

```go
type Notifier interface {
    Send(to, message string) error
}

type EmailNotifier struct{}
type SMSNotifier struct{}
type WeChatNotifier struct{}

type NotificationService struct {
    notifier Notifier
}

func (s *NotificationService) Notify(to, msg string) error {
    return s.notifier.Send(to, msg)
}

// 运行时切换策略
svc := &NotificationService{notifier: &EmailNotifier{}}
svc.Notify("user@example.com", "hello")
```

### 5.4 观察者模式

```go
type Event struct {
    Type    string
    Payload any
}

type Handler func(Event)

type EventBus struct {
    mu       sync.RWMutex
    handlers map[string][]Handler
}

func NewEventBus() *EventBus {
    return &EventBus{handlers: make(map[string][]Handler)}
}

func (eb *EventBus) Subscribe(eventType string, h Handler) {
    eb.mu.Lock()
    defer eb.mu.Unlock()
    eb.handlers[eventType] = append(eb.handlers[eventType], h)
}

func (eb *EventBus) Publish(e Event) {
    eb.mu.RLock()
    defer eb.mu.RUnlock()
    for _, h := range eb.handlers[e.Type] {
        go h(e) // 异步处理
    }
}
```

### 5.5 选项模式（Functional Options）

```go
type Server struct {
    addr     string
    timeout  time.Duration
    maxConns int
    tls      bool
}

type Option func(*Server)

func WithTimeout(d time.Duration) Option { return func(s *Server) { s.timeout = d } }
func WithMaxConns(n int) Option          { return func(s *Server) { s.maxConns = n } }
func WithTLS() Option                    { return func(s *Server) { s.tls = true } }

func NewServer(addr string, opts ...Option) *Server {
    s := &Server{
        addr:     addr,
        timeout:  30 * time.Second, // 默认值
        maxConns: 100,
    }
    for _, opt := range opts {
        opt(s)
    }
    return s
}

// 使用
srv := NewServer(":8080", WithTimeout(10*time.Second), WithTLS())
```

## 六、Makefile

```makefile
.PHONY: build run test lint clean

APP_NAME = myapp
BUILD_DIR = ./bin

build:
	go build -o $(BUILD_DIR)/$(APP_NAME) ./cmd/server/

run:
	go run ./cmd/server/

test:
	go test -race -cover ./...

lint:
	golangci-lint run ./...

migrate-up:
	migrate -database "$(DB_DSN)" -path migrations up

migrate-down:
	migrate -database "$(DB_DSN)" -path migrations down 1

docker-build:
	docker build -t $(APP_NAME) .

clean:
	rm -rf $(BUILD_DIR)
```

## 速查表

| 概念 | 要点 |
|------|------|
| `internal/` | 编译器强制私有，外部包不能导入 |
| 分层 | Handler → Service → Repository |
| 依赖注入 | 构造函数注入，入口组装 |
| go mod | `tidy` 整理 / `vendor` 本地化 |
| 单例 | `sync.Once` |
| 工厂 | 返回接口，隐藏实现 |
| 策略 | 接口字段，运行时可替换 |
| 选项模式 | `func(opts ...Option)` 灵活配置 |
| 配置 | Viper：YAML + 环境变量 |
