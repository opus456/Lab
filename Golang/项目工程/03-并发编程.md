## 一、Goroutine

### 1.1 启动 Goroutine

```go
go func() {
    fmt.Println("hello from goroutine")
}()
```

### 1.2 Goroutine 的本质

```
┌─────────────────────────────────┐
│           Go Runtime            │
│  ┌───────┐ ┌───────┐ ┌───────┐ │
│  │  G1   │ │  G2   │ │  G3   │ │  Goroutine（用户态协程）
│  └──┬────┘ └──┬────┘ └──┬────┘ │
│     │         │         │       │
│  ┌──▼────┐ ┌──▼────┐            │
│  │  M1   │ │  M2   │           │  系统线程
│  └──┬────┘ └──┬────┘            │
│     │         │                  │
│  ┌──▼────┐ ┌──▼────┐            │
│  │  P1   │ │  P2   │           │  逻辑处理器
│  └───────┘ └───────┘            │
└─────────────────────────────────┘
```

**GMP 模型**：
- **G**（Goroutine）：用户级协程，初始栈仅 2KB（线程栈通常 1-8MB）
- **M**（Machine）：操作系统线程
- **P**（Processor）：逻辑处理器，数量 = `GOMAXPROCS`（默认=CPU核心数）

**调度特点**：
- 协作式 + 抢占式（Go 1.14 起基于信号的异步抢占）
- 一个 M 必须绑定一个 P 才能运行 G
- G 阻塞时，M 会释放 P 给其他 M 使用

### 1.3 Goroutine 泄漏

```go
// ❌ 泄漏：channel 没人读，goroutine 永远阻塞
func leak() {
    ch := make(chan int)
    go func() {
        ch <- 42 // 永远阻塞，没人接收
    }()
    // 函数返回，goroutine 永远不会释放
}

// ✅ 修复方案 1：保证消费
func noLeak() {
    ch := make(chan int, 1) // 带缓冲
    go func() { ch <- 42 }()
    <-ch
}

// ✅ 修复方案 2：用 context 取消
func noLeak2(ctx context.Context) {
    ch := make(chan int)
    go func() {
        select {
        case ch <- 42:
        case <-ctx.Done():
            return // 优雅退出
        }
    }()
}
```

## 二、Channel

### 2.1 基础操作

```go
ch := make(chan int)    // 无缓冲
ch := make(chan int, 5) // 缓冲区大小 5

ch <- 42        // 发送
v := <-ch       // 接收
v, ok := <-ch   // 接收 + 检测是否关闭
close(ch)       // 关闭
```

### 2.2 无缓冲 vs 有缓冲

```
无缓冲 (make(chan int))
──────────────────────
发送方阻塞，直到接收方准备好。
像 "面对面交接物品"。

有缓冲 (make(chan int, N))
──────────────────────────
发送方阻塞，仅当缓冲区满。
接收方阻塞，仅当缓冲区空。
像 "把物品放到桌上，对方有空再拿"。
```

### 2.3 Channel 的方向

```go
func producer(out chan<- int) { out <- 42 }   // 只写
func consumer(in <-chan int)  { v := <-in }   // 只读
```

### 2.4 遍历 Channel

```go
ch := make(chan int, 3)
ch <- 1; ch <- 2; ch <- 3
close(ch)

for v := range ch {
    fmt.Println(v)  // 1, 2, 3
}
// range 在 channel 关闭后自动退出
```

### 2.5 select 多路复用

```go
select {
case v := <-ch1:
    fmt.Println("from ch1:", v)
case v := <-ch2:
    fmt.Println("from ch2:", v)
case ch3 <- 42:
    fmt.Println("sent to ch3")
default:
    fmt.Println("no channel ready") // 非阻塞
}
```

**select 规则**：
- 多个 case 就绪时**随机**选一个
- 没有 default 时阻塞到某个 case 就绪
- 有 default 时立即执行 default（非阻塞）

## 三、sync 包

### 3.1 sync.Mutex / RWMutex

```go
type SafeMap struct {
    mu sync.RWMutex
    m  map[string]int
}

func (s *SafeMap) Get(key string) int {
    s.mu.RLock()         // 读锁（多个读可并行）
    defer s.mu.RUnlock()
    return s.m[key]
}

func (s *SafeMap) Set(key string, val int) {
    s.mu.Lock()          // 写锁（独占）
    defer s.mu.Unlock()
    s.m[key] = val
}
```

### 3.2 sync.WaitGroup

```go
var wg sync.WaitGroup
for i := 0; i < 10; i++ {
    wg.Add(1)
    go func(id int) {
        defer wg.Done()
        // 做事...
    }(i)
}
wg.Wait() // 等待所有 goroutine 完成
```

**陷阱**：`wg.Add(1)` 必须在 `go func()` **之前**调用！

### 3.3 sync.Once

```go
var (
    instance *Database
    once     sync.Once
)

func GetDB() *Database {
    once.Do(func() {
        instance = connectDB()
    })
    return instance
}
// 无论多少 goroutine 调用，connectDB 只执行一次
```

### 3.4 sync.Map

```go
var m sync.Map

m.Store("key", "value")
v, ok := m.Load("key")
m.Delete("key")
m.Range(func(k, v any) bool {
    fmt.Println(k, v)
    return true // 继续遍历
})
```

**使用场景**：key 集合稳定（读多写少）或 key 独立（各 goroutine 访问不同 key）。

### 3.5 sync.Pool

```go
var bufPool = sync.Pool{
    New: func() any {
        return new(bytes.Buffer)
    },
}

func process() {
    buf := bufPool.Get().(*bytes.Buffer)
    buf.Reset()
    defer bufPool.Put(buf) // 用完归还
    // 使用 buf...
}
```

## 四、context 包

### 4.1 为什么需要 context

- 控制 goroutine 的生命周期（取消、超时）
- 在调用链中传递请求级别的元数据

### 4.2 创建 context

```go
ctx := context.Background()          // 根 context
ctx := context.TODO()                // 占位

ctx, cancel := context.WithCancel(parent)
ctx, cancel := context.WithTimeout(parent, 5*time.Second)
ctx, cancel := context.WithDeadline(parent, deadline)
ctx = context.WithValue(parent, key, value)
```

### 4.3 使用范式

```go
func doWork(ctx context.Context) error {
    for {
        select {
        case <-ctx.Done():
            return ctx.Err() // context.Canceled 或 DeadlineExceeded
        default:
            // 做一小块工作...
        }
    }
}

// 调用方
ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
defer cancel() // 必须调用 cancel 释放资源
go doWork(ctx)
```

### 4.4 context 使用规范

- `context.Context` 作为函数**第一个参数**
- **不要**把 context 存到结构体里
- **不要**传 nil context，用 `context.TODO()`
- WithValue 只传请求级数据（如 traceID），不传业务参数

## 五、常用并发模式

### 5.1 Worker Pool（工作池）

```go
func workerPool(jobs <-chan int, results chan<- int, numWorkers int) {
    var wg sync.WaitGroup
    for w := 0; w < numWorkers; w++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            for job := range jobs {
                results <- process(job)
            }
        }()
    }
    wg.Wait()
    close(results)
}

func main() {
    jobs := make(chan int, 100)
    results := make(chan int, 100)

    go workerPool(jobs, results, 5)

    // 发送任务
    for i := 0; i < 50; i++ {
        jobs <- i
    }
    close(jobs)

    // 收集结果
    for r := range results {
        fmt.Println(r)
    }
}
```

### 5.2 Fan-out / Fan-in

```go
// Fan-out: 一个 channel 分发给多个 goroutine
func fanOut(in <-chan int, n int) []<-chan int {
    outs := make([]<-chan int, n)
    for i := 0; i < n; i++ {
        ch := make(chan int)
        outs[i] = ch
        go func() {
            defer close(ch)
            for v := range in {
                ch <- v * v
            }
        }()
    }
    return outs
}

// Fan-in: 多个 channel 汇聚成一个
func fanIn(channels ...<-chan int) <-chan int {
    out := make(chan int)
    var wg sync.WaitGroup
    for _, ch := range channels {
        wg.Add(1)
        go func(c <-chan int) {
            defer wg.Done()
            for v := range c {
                out <- v
            }
        }(ch)
    }
    go func() { wg.Wait(); close(out) }()
    return out
}
```

### 5.3 Pipeline（管道）

```go
func generate(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for _, n := range nums { out <- n }
    }()
    return out
}

func square(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for n := range in { out <- n * n }
    }()
    return out
}

func filter(in <-chan int, pred func(int) bool) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for n := range in {
            if pred(n) { out <- n }
        }
    }()
    return out
}

// 使用：generate → square → filter
nums := generate(1, 2, 3, 4, 5)
squared := square(nums)
even := filter(squared, func(n int) bool { return n%2 == 0 })
for v := range even { fmt.Println(v) } // 4, 16
```

### 5.4 限流器（Rate Limiter）

```go
func rateLimiter(ctx context.Context) <-chan struct{} {
    ticker := time.NewTicker(100 * time.Millisecond) // 每秒 10 次
    permits := make(chan struct{})
    go func() {
        defer ticker.Stop()
        defer close(permits)
        for {
            select {
            case <-ctx.Done():
                return
            case <-ticker.C:
                permits <- struct{}{}
            }
        }
    }()
    return permits
}
```

### 5.5 超时控制

```go
func doWithTimeout() (string, error) {
    ctx, cancel := context.WithTimeout(context.Background(), 3*time.Second)
    defer cancel()

    ch := make(chan string, 1)
    go func() {
        ch <- slowOperation()
    }()

    select {
    case result := <-ch:
        return result, nil
    case <-ctx.Done():
        return "", fmt.Errorf("timeout: %w", ctx.Err())
    }
}
```

## 六、并发安全 checklist

| 问题 | 解决方案 |
|------|---------|
| 共享变量读写 | `sync.Mutex` / `sync.RWMutex` |
| 计数/等待 | `sync.WaitGroup` |
| 一次初始化 | `sync.Once` |
| 并发 map | `sync.Map` 或 mutex+map |
| 通信 | Channel |
| 生命周期控制 | `context.Context` |
| goroutine 泄漏 | 确保 channel 可关闭，用 context 取消 |
| 竞态检测 | `go run -race` / `go test -race` |

```go
// ❌ 竞态

var count int64
go func() { atomic.AddInt64(&count, 1) }()
```

## 速查表

| 概念 | 要点 |
|------|------|
| goroutine | `go func()`，2KB 栈，GMP 调度 |
| 无缓冲 channel | 同步交接，发送/接收互相等待 |
| 有缓冲 channel | 异步，满时发送阻塞，空时接收阻塞 |
| select | 多路复用，随机选就绪 case |
| WaitGroup | Add→go→Done→Wait |
| Mutex | Lock/Unlock，RLock/RUnlock |
| Once | 保证只执行一次 |
| context | 取消/超时/传值，第一个参数 |
| -race | 竞态检测，开发必开 |
