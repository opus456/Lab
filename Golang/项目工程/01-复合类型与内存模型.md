## 一、数组

### 1.1 基础

数组在 Go 中是**值类型**，长度是类型的一部分。

```go
var a [5]int                // [0, 0, 0, 0, 0]
b := [3]int{1, 2, 3}       // [1, 2, 3]
c := [...]int{1, 2, 3}     // 编译器推断长度

// [3]int 和 [5]int 是不同类型！不能互相赋值
```

### 1.2 数组是值传递

```go
func modify(arr [3]int) {
    arr[0] = 999 // 修改的是副本
}

a := [3]int{1, 2, 3}
modify(a)
fmt.Println(a) // [1, 2, 3] — 没变！
```

## 二、切片（Slice）——Go 最重要的数据结构

### 2.1 底层结构

切片本质是一个**三元组**：

```go
type slice struct {
    array unsafe.Pointer // 指向底层数组的指针
    len   int            // 当前长度
    cap   int            // 容量（底层数组的可用长度）
}
```

```
切片变量:  [ptr | len=3 | cap=5]
               |
               ↓
底层数组:  [1] [2] [3] [ ] [ ]
            ↑             ↑
          数据区        剩余容量
```

### 2.2 创建切片

```go
// 从数组切割
arr := [5]int{1, 2, 3, 4, 5}
s := arr[1:4] // [2, 3, 4], len=3, cap=4

// make 创建
s := make([]int, 3)       // len=3, cap=3, [0,0,0]
s := make([]int, 3, 10)   // len=3, cap=10

// 字面量
s := []int{1, 2, 3}
```

### 2.3 append 与扩容

```go
s := make([]int, 0, 3)
s = append(s, 1) // len=1, cap=3
s = append(s, 2) // len=2, cap=3
s = append(s, 3) // len=3, cap=3
s = append(s, 4) // len=4, cap=6 ← 触发扩容！

// 扩容策略（Go 1.18+）：
// cap < 256: 翻倍
// cap >= 256: 增长约 25% + 192
```

**关键理解**：`append` 可能返回一个指向新底层数组的切片。所以**必须**用 `s = append(s, x)` 接收返回值。

### 2.4 切片陷阱

**陷阱1：共享底层数组**

```go
a := []int{1, 2, 3, 4, 5}
b := a[1:3] // [2, 3]，但和 a 共享底层数组
b[0] = 99
fmt.Println(a) // [1, 99, 3, 4, 5] ← a 也变了！
```

**解决方案**：`copy` 创建独立副本

```go
b := make([]int, 2)
copy(b, a[1:3]) // b 是独立的
```

**陷阱2：append 可能影响原切片**

```go
a := []int{1, 2, 3, 4, 5}
b := a[:3]         // [1, 2, 3], cap=5
b = append(b, 99)  // [1, 2, 3, 99], 没超 cap，直接写底层数组
fmt.Println(a)     // [1, 2, 3, 99, 5] ← a[3] 被改了！
```

**解决方案**：用三下标切片限制 cap

```go
b := a[:3:3]       // len=3, cap=3, 这样 append 必定扩容，不影响 a
```

### 2.5 常用操作

```go
// 删除索引 i
s = append(s[:i], s[i+1:]...)

// 在索引 i 处插入 v
s = append(s[:i], append([]int{v}, s[i:]...)...)

// 反转
for i, j := 0, len(s)-1; i < j; i, j = i+1, j-1 {
    s[i], s[j] = s[j], s[i]
}
```

## 三、Map

### 3.1 底层结构（简化）

Go 的 map 是**哈希表**，实现为桶数组 + 溢出链表：

```
map[string]int

哈希函数 → 桶编号
桶0: [key1:val1] → [key5:val5] → nil
桶1: [key2:val2] → nil
桶2: [key3:val3] → [key4:val4] → nil
```

### 3.2 基本操作

```go
// 创建
m := make(map[string]int)
m := map[string]int{"a": 1, "b": 2}

// 增/改
m["key"] = 42

// 查
val, ok := m["key"]
if !ok {
    // 不存在
}

// 删
delete(m, "key")

// 遍历（顺序随机）
for k, v := range m {
    fmt.Println(k, v)
}

// 长度
len(m)
```

### 3.3 Map 注意事项

1. **Map 不是并发安全的**。多个 goroutine 同时读写会 panic。用 `sync.RWMutex` 或 `sync.Map`。
2. **Map 的值不可寻址**。不能直接 `m[key].field = x`，需要取出来改再放回去。
3. **nil map 可以读，不能写**。

```go
var m map[string]int // nil
_ = m["key"]        // OK，返回零值 0
m["key"] = 1        // panic!
```

## 四、结构体（Struct）

### 4.1 基础

```go
type User struct {
    Name string
    Age  int
    Email string
}

// 创建
u1 := User{"Alice", 30, "alice@example.com"} // 按顺序
u2 := User{Name: "Bob", Age: 25}             // 按字段名（推荐）
u3 := new(User)                                // 返回 *User，字段为零值
```

### 4.2 结构体方法

```go
// 值接收者（不修改原对象）
func (u User) FullInfo() string {
    return fmt.Sprintf("%s (%d)", u.Name, u.Age)
}

// 指针接收者（可以修改原对象）
func (u *User) SetAge(age int) {
    u.Age = age
}

// 调用时自动取地址/解引用
u := User{Name: "Alice", Age: 30}
u.SetAge(31)       // 编译器自动取 &u
fmt.Println(u.Age) // 31
```

**何时用指针接收者？**
- 需要修改接收者
- 结构体较大，避免拷贝
- 保持一致性：如果某个方法用了指针，其他方法也用指针

### 4.3 结构体嵌入（组合）

```go
type Address struct {
    City    string
    ZipCode string
}

type Employee struct {
    User            // 嵌入 User（匿名字段）
    Address         // 嵌入 Address
    Department string
}

emp := Employee{
    User:       User{Name: "Alice", Age: 30},
    Address:    Address{City: "Beijing"},
    Department: "Engineering",
}

// 可以直接访问嵌入字段的成员
fmt.Println(emp.Name)  // Alice（从 User 提升）
fmt.Println(emp.City)  // Beijing（从 Address 提升）
```

### 4.4 标签（Tag）

```go
type User struct {
    Name  string `json:"name" validate:"required"`
    Email string `json:"email,omitempty"`
    Age   int    `json:"age" db:"user_age"`
}
```

标签通过反射读取，用于 JSON 序列化、数据库映射、验证等。

## 五、指针

### 5.1 基础

```go
var x int = 42
var p *int = &x   // p 指向 x 的地址
fmt.Println(*p)   // 42（解引用）
*p = 100
fmt.Println(x)    // 100
```

### 5.2 Go 指针 vs C 指针

| 特性 | Go | C |
|------|-----|---|
| 指针运算 | ❌ 不允许 | ✅ |
| 空指针解引用 | panic（运行时检查） | 未定义行为 |
| 垃圾回收 | ✅ 自动 | ❌ 手动 |
| 指针能逃逸到堆 | ✅ 编译器自动决定 | ❌ 需要手动 malloc |

### 5.3 逃逸分析

```go
func newUser() *User {
    u := User{Name: "Alice"} // u 在栈上创建
    return &u                 // 返回了指针 → u 逃逸到堆上
}
```

编译器会自动判断变量应该分配在栈上还是堆上。查看逃逸分析：`go build -gcflags="-m"`

## 六、内存模型要点

### 6.1 值语义 vs 引用语义

| 类型 | 语义 | 赋值行为 | 函数传参行为 |
|------|------|---------|------------|
| 数组 | 值 | 完整拷贝 | 完整拷贝 |
| 结构体 | 值 | 完整拷贝 | 完整拷贝 |
| 切片 | 引用式 | 拷贝三元组（共享底层数组） | 共享底层 |
| Map | 引用 | 共享同一个 map | 共享 |
| Channel | 引用 | 共享 | 共享 |
| 指针 | 引用 | 拷贝地址值 | 共享指向的对象 |

### 6.2 对齐与大小

```go
import "unsafe"

fmt.Println(unsafe.Sizeof(bool(false)))  // 1
fmt.Println(unsafe.Sizeof(int64(0)))     // 8
fmt.Println(unsafe.Sizeof(""))           // 16 (ptr + len)
fmt.Println(unsafe.Sizeof([]int{}))      // 24 (ptr + len + cap)
fmt.Println(unsafe.Sizeof(map[int]int{}))// 8 (只是一个指针)
```

**结构体对齐**：字段按对齐要求排列，可能有内部填充。字段排列顺序影响结构体大小。

```go
// 差的排列：24 字节
type Bad struct {
    a bool    // 1 + 7 padding
    b int64   // 8
    c bool    // 1 + 7 padding
}

// 好的排列：16 字节
type Good struct {
    b int64   // 8
    a bool    // 1
    c bool    // 1 + 6 padding
}
```
