# 接口与面向对象设计

> **目标**：理解 Go 独特的接口机制（隐式实现）、组合优于继承的设计哲学，写出灵活可扩展的代码。

---

## 一、接口基础

### 1.1 定义接口

```go
type Reader interface {
    Read(p []byte) (n int, err error)
}

type Writer interface {
    Write(p []byte) (n int, err error)
}

// 接口组合
type ReadWriter interface {
    Reader
    Writer
}
```

### 1.2 隐式实现（Duck Typing）

Go 不需要 `implements` 关键字。只要一个类型实现了接口的所有方法，它就自动满足该接口。

```go
type MyFile struct{ data []byte }

func (f *MyFile) Read(p []byte) (int, error) {
    n := copy(p, f.data)
    return n, nil
}
// MyFile 自动满足 Reader 接口，不需要任何声明
```

**好处**：
- 接口的定义者和实现者可以完全独立
- 标准库定义接口，第三方库实现接口，无需互相知道
- 极大提高了代码的可组合性

### 1.3 空接口

```go
interface{} // Go 1.18 之前
any         // Go 1.18+ 的别名

// 可以存任何类型
func printAnything(v any) {
    fmt.Println(v)
}
```

### 1.4 类型断言

```go
var r Reader = &MyFile{}

// 断言为具体类型
f, ok := r.(*MyFile)
if ok {
    // f 是 *MyFile 类型
}

// type switch
switch v := r.(type) {
case *MyFile:
    fmt.Println("MyFile:", v)
case *os.File:
    fmt.Println("os.File:", v)
default:
    fmt.Println("unknown")
}
```

## 二、接口设计原则

### 2.1 小接口原则

Go 标准库中最强大的接口都只有 1-2 个方法：

```go
type Reader interface { Read([]byte) (int, error) }
type Writer interface { Write([]byte) (int, error) }
type Closer interface { Close() error }
type Stringer interface { String() string }
type Error interface { Error() string }
```

**为什么？** 接口越小，能满足它的类型越多，复用性越强。

### 2.2 接口由使用方定义

在 Go 中，接口通常在**使用它的包**中定义，而不是在实现它的包中：

```go
// ❌ 不好：实现方定义一堆接口
package storage
type UserStorage interface {
    GetUser(id int) (*User, error)
    SaveUser(u *User) error
    DeleteUser(id int) error
}

// ✅ 好：使用方定义自己需要的最小接口
package handler
type UserGetter interface {
    GetUser(id int) (*User, error)
}
```

### 2.3 接口的值

接口值 = (type, value) 的二元组：

```go
var w Writer
fmt.Println(w == nil)      // true

var f *os.File = nil
w = f                       // w 的 type=*os.File, value=nil
fmt.Println(w == nil)      // false! 接口值不为 nil，因为有类型信息
```

**这是一个经典陷阱**。判断接口内部值是否为 nil：

```go
import "reflect"
func isNilInterface(v any) bool {
    if v == nil { return true }
    rv := reflect.ValueOf(v)
    return rv.Kind() == reflect.Ptr && rv.IsNil()
}
```

## 三、组合优于继承

Go 没有类继承，用**组合和接口**实现代码复用。

### 3.1 结构体嵌入 = 组合

```go
type Logger struct{}
func (l *Logger) Log(msg string) { fmt.Println("[LOG]", msg) }

type Server struct {
    Logger           // 嵌入 Logger
    addr string
}

s := &Server{addr: ":8080"}
s.Log("started")  // 直接调用嵌入的方法
```

### 3.2 继承 vs 组合对比

```
Java (继承):
Animal → Dog → GuideDog

Go (组合):
type Walker struct { ... }
type Barker struct { ... }
type Dog struct { Walker; Barker }
type GuideDog struct { Dog; GuideSkill }
```

**组合的好处**：
- 没有"菱形继承"问题
- 每个功能模块独立可测试
- 更灵活，可以按需组合

## 四、常用接口模式

### 4.1 Stringer（自定义打印）

```go
type Point struct{ X, Y int }

func (p Point) String() string {
    return fmt.Sprintf("(%d, %d)", p.X, p.Y)
}

fmt.Println(Point{1, 2}) // (1, 2)
```

### 4.2 Sort 接口

```go
type ByAge []User

func (a ByAge) Len() int           { return len(a) }
func (a ByAge) Less(i, j int) bool { return a[i].Age < a[j].Age }
func (a ByAge) Swap(i, j int)      { a[i], a[j] = a[j], a[i] }

sort.Sort(ByAge(users))

// 更简洁：sort.Slice
sort.Slice(users, func(i, j int) bool {
    return users[i].Age < users[j].Age
})
```

### 4.3 io.Reader/Writer 链

```go
// 读取文件 → 解压 → 解码
f, _ := os.Open("data.gz")
defer f.Close()
gz, _ := gzip.NewReader(f)     // 包装：文件 → gzip 解压
decoder := json.NewDecoder(gz)  // 包装：解压 → JSON 解码
decoder.Decode(&result)
```

这就是**装饰器模式**：每一层都是 `io.Reader`，像俄罗斯套娃一样嵌套。

## 五、面向对象设计实战

### 5.1 工厂模式

```go
type Storage interface {
    Save(key string, data []byte) error
    Load(key string) ([]byte, error)
}

type FileStorage struct{ dir string }
type RedisStorage struct{ addr string }

func NewStorage(storageType string) Storage {
    switch storageType {
    case "file":
        return &FileStorage{dir: "./data"}
    case "redis":
        return &RedisStorage{addr: "localhost:6379"}
    default:
        return &FileStorage{dir: "./data"}
    }
}
```

### 5.2 策略模式

```go
type Compressor interface {
    Compress(data []byte) ([]byte, error)
}

type GzipCompressor struct{}
type ZstdCompressor struct{}

type Uploader struct {
    compressor Compressor // 注入策略
}

func (u *Uploader) Upload(data []byte) error {
    compressed, _ := u.compressor.Compress(data)
    // 上传 compressed...
    return nil
}
```

## 速查表

| 概念 | 要点 |
|------|------|
| 接口实现 | 隐式，无需 `implements` |
| 空接口 | `any` 或 `interface{}` |
| 类型断言 | `v, ok := i.(T)` |
| 小接口 | 1-2 个方法最佳 |
| 接口值 nil 陷阱 | `(type=*T, value=nil) != nil` |
| 组合 | 结构体嵌入 + 接口 |
| 常用接口 | Reader/Writer/Closer/Stringer/Error |
