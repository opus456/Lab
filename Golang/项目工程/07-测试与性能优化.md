## 一、单元测试

### 1.1 基本规则

- 文件名以 `_test.go` 结尾
- 函数签名 `func TestXxx(t *testing.T)`
- 运行：`go test ./...`

```go
// math.go
func Add(a, b int) int { return a + b }
func Divide(a, b int) (int, error) {
    if b == 0 { return 0, errors.New("division by zero") }
    return a / b, nil
}

// math_test.go
func TestAdd(t *testing.T) {
    got := Add(2, 3)
    if got != 5 {
        t.Errorf("Add(2,3) = %d, want 5", got)
    }
}
```

### 1.2 表驱动测试（Table-Driven）

这是 Go 最推荐的测试模式：

```go
func TestAdd(t *testing.T) {
    tests := []struct {
        name     string
        a, b     int
        expected int
    }{
        {"positive", 2, 3, 5},
        {"negative", -1, -2, -3},
        {"zero", 0, 0, 0},
        {"mixed", -1, 5, 4},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got := Add(tt.a, tt.b)
            if got != tt.expected {
                t.Errorf("Add(%d,%d) = %d, want %d", tt.a, tt.b, got, tt.expected)
            }
        })
    }
}
```

### 1.3 错误路径测试

```go
func TestDivide(t *testing.T) {
    tests := []struct {
        name    string
        a, b    int
        want    int
        wantErr bool
    }{
        {"normal", 10, 2, 5, false},
        {"div by zero", 10, 0, 0, true},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got, err := Divide(tt.a, tt.b)
            if (err != nil) != tt.wantErr {
                t.Errorf("error = %v, wantErr %v", err, tt.wantErr)
                return
            }
            if got != tt.want {
                t.Errorf("got %d, want %d", got, tt.want)
            }
        })
    }
}
```

### 1.4 测试辅助

```go
// t.Helper() 标记辅助函数，报错时显示调用方行号
func assertEqual(t *testing.T, got, want int) {
    t.Helper()
    if got != want {
        t.Errorf("got %d, want %d", got, want)
    }
}

// t.Cleanup() 注册清理函数
func TestWithDB(t *testing.T) {
    db := setupTestDB(t)
    t.Cleanup(func() { db.Close() })
    // 测试...
}

// t.Parallel() 并行执行
func TestParallel(t *testing.T) {
    t.Parallel()
    // 这个测试会和其他 Parallel 测试并行运行
}

// t.Skip() 跳过
func TestSlowIntegration(t *testing.T) {
    if testing.Short() {
        t.Skip("skipping in short mode")
    }
    // 慢速集成测试...
}
// go test -short 时跳过
```

## 二、HTTP 测试

### 2.1 httptest

```go
import "net/http/httptest"

func TestGetUserHandler(t *testing.T) {
    // 创建请求
    req := httptest.NewRequest("GET", "/users/1", nil)
    // 创建响应记录器
    w := httptest.NewRecorder()

    // 调用 handler
    getUserHandler(w, req)

    // 验证
    resp := w.Result()
    if resp.StatusCode != 200 {
        t.Errorf("status = %d, want 200", resp.StatusCode)
    }

    body, _ := io.ReadAll(resp.Body)
    var user User
    json.Unmarshal(body, &user)
    if user.Name != "Alice" {
        t.Errorf("name = %s, want Alice", user.Name)
    }
}
```

### 2.2 测试 Gin Handler

```go
func TestGinHandler(t *testing.T) {
    gin.SetMode(gin.TestMode)
    r := gin.New()
    r.GET("/ping", pingHandler)

    w := httptest.NewRecorder()
    req := httptest.NewRequest("GET", "/ping", nil)
    r.ServeHTTP(w, req)

    if w.Code != 200 {
        t.Errorf("status = %d, want 200", w.Code)
    }

    var resp map[string]string
    json.Unmarshal(w.Body.Bytes(), &resp)
    if resp["message"] != "pong" {
        t.Errorf("message = %s, want pong", resp["message"])
    }
}

// POST 请求 + JSON body
func TestCreateUser(t *testing.T) {
    gin.SetMode(gin.TestMode)
    r := gin.New()
    r.POST("/users", createUserHandler)

    body := `{"name":"Alice","email":"alice@example.com"}`
    w := httptest.NewRecorder()
    req := httptest.NewRequest("POST", "/users", strings.NewReader(body))
    req.Header.Set("Content-Type", "application/json")
    r.ServeHTTP(w, req)

    if w.Code != 201 {
        t.Errorf("status = %d, want 201", w.Code)
    }
}
```

## 三、Mock 与依赖注入

### 3.1 接口 Mock

```go
// 定义接口
type UserRepository interface {
    GetByID(ctx context.Context, id int) (*User, error)
    Create(ctx context.Context, u *User) error
}

// 真实实现
type mysqlUserRepo struct{ db *gorm.DB }

// Mock 实现
type mockUserRepo struct {
    users map[int]*User
}

func (m *mockUserRepo) GetByID(ctx context.Context, id int) (*User, error) {
    u, ok := m.users[id]
    if !ok {
        return nil, errors.New("not found")
    }
    return u, nil
}

func (m *mockUserRepo) Create(ctx context.Context, u *User) error {
    m.users[int(u.ID)] = u
    return nil
}

// 测试 Service 层
func TestUserService_GetByID(t *testing.T) {
    repo := &mockUserRepo{
        users: map[int]*User{1: {ID: 1, Name: "Alice"}},
    }
    svc := NewUserService(repo) // 注入 mock

    user, err := svc.GetByID(context.Background(), 1)
    if err != nil {
        t.Fatal(err)
    }
    if user.Name != "Alice" {
        t.Errorf("name = %s, want Alice", user.Name)
    }
}
```

## 四、基准测试

### 4.1 编写 Benchmark

```go
func BenchmarkAdd(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Add(100, 200)
    }
}

// 运行
// go test -bench=. -benchmem
// BenchmarkAdd-8    1000000000    0.25 ns/op    0 B/op    0 allocs/op
```

### 4.2 比较不同实现

```go
func BenchmarkConcatPlus(b *testing.B) {
    for i := 0; i < b.N; i++ {
        s := ""
        for j := 0; j < 100; j++ {
            s += "hello"
        }
    }
}

func BenchmarkConcatBuilder(b *testing.B) {
    for i := 0; i < b.N; i++ {
        var builder strings.Builder
        for j := 0; j < 100; j++ {
            builder.WriteString("hello")
        }
        _ = builder.String()
    }
}

// 结果对比:
// BenchmarkConcatPlus-8       12345   95000 ns/op   53000 B/op   99 allocs/op
// BenchmarkConcatBuilder-8  1000000    1200 ns/op     896 B/op    4 allocs/op
// → Builder 快 80 倍，内存分配少 24 倍
```

### 4.3 ResetTimer / StopTimer

## 五、pprof 性能分析

### 5.1 CPU 分析

```go
import "net/http/pprof"

// 方式 1：HTTP 端点（开发/测试环境）
func main() {
    go func() {
        log.Println(http.ListenAndServe(":6060", nil)) // 默认 mux 已注册 pprof
    }()
    // ... 启动主程序
}

// 方式 2：写文件
import "runtime/pprof"

f, _ := os.Create("cpu.prof")
pprof.StartCPUProfile(f)
defer pprof.StopCPUProfile()
// ... 执行要分析的代码
```

```bash
# 采集 30 秒 CPU 数据
go tool pprof http://localhost:6060/debug/pprof/profile?seconds=30

# 分析 prof 文件
go tool pprof cpu.prof

# 常用命令
(pprof) top 10        # 最耗时的 10 个函数
(pprof) list funcName # 查看函数逐行耗时
(pprof) web           # 生成调用图（需要 graphviz）
```

## 六、常见性能优化技巧

### 6.1 字符串拼接

```go
// ❌ 循环中用 + 拼接
s := ""
for _, item := range items { s += item }

// ✅ 使用 strings.Builder
var b strings.Builder
for _, item := range items { b.WriteString(item) }
result := b.String()
```

### 6.2 预分配切片

```go
// ❌ 不断 append 扩容
var result []int
for _, v := range data { result = append(result, v) }

// ✅ 预分配
result := make([]int, 0, len(data))
for _, v := range data { result = append(result, v) }
```

### 6.3 sync.Pool 复用对象

```go
var bufPool = &sync.Pool{
    New: func() any { return new(bytes.Buffer) },
}

func process(data []byte) string {
    buf := bufPool.Get().(*bytes.Buffer)
    buf.Reset()
    defer bufPool.Put(buf)
    buf.Write(data)
    return buf.String()
}
```

### 6.4 减少内存逃逸

```go
// ❌ 逃逸到堆上
func newUser() *User {
    u := User{Name: "Alice"} // 逃逸
    return &u
}

// 查看逃逸分析
// go build -gcflags='-m' .
```

## 七、测试命令速查

```bash
# 运行所有测试
go test ./...

# 运行特定测试
go test -run TestAdd ./math/

# 详细输出
go test -v ./...

# 短模式（跳过慢测试）
go test -short ./...

# 竞态检测
go test -race ./...

# 覆盖率
go test -cover ./...
go test -coverprofile=cover.out ./...
go tool cover -html=cover.out  # 生成 HTML 报告

# 基准测试
go test -bench=. -benchmem ./...
go test -bench=BenchmarkAdd -benchtime=5s ./...
go test -bench=. -count=5 ./...   # 运行 5 次取平均

# benchstat 比较（安装 golang.org/x/perf/cmd/benchstat）
go test -bench=. -count=5 > old.txt
# 修改代码后
go test -bench=. -count=5 > new.txt
benchstat old.txt new.txt
```

| 概念 | 要点 |
